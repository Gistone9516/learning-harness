# 학습 프레임워크 기획 v5 (Discord 네이티브 전환)

> buildflow ① 기획 산출물. 스무고개 6라운드 + 멀티에이전트 이데이션(5렌즈·2비평, harness 58파일 전수) 수렴.
> 상태: G1 통과(사용자 확정). 다음 = ② 명세(`규격/` 계약: 엔진계약·봇계약·학습타입규격·AI모드골격·주입 인터페이스·구동 skill).
> 전환: 정적 HTML 프레임워크(v4, `_archive_v4/`) 전면 폐기 → Discord 네이티브 학습 하네스.

---

## 1. 의도

**Discord 네이티브 개인 학습 하네스(범용 프레임워크).** 카드 인출·간격반복·진단·AI 학습을 Discord 위에서 돌린다.
멀티기기(폰·PC)와 UI는 Discord가 흡수한다(자동 동기화·네이티브 컴포넌트). 옆 폴더 `Discord Agents/harness`
카탈로그를 차용해 만든다.

이 리포는 **과목·콘텐츠가 없는 도구·계약 키트**다. 구체 과목의 성격부여는 추후 별도 프로젝트가 이 프레임워크를
소비하며 수행한다(§3).

## 2. 전환 배경 (v4 → v5)

v4(컴활1급용 정적 HTML + 바닐라 JS + localStorage)는 8사이클 누적으로 8플러그인·카드퀴즈 엔진·generate.py
빌드까지 비대해졌다. 진단: 복잡·비대, UI 빈약·낡음, 학습효과·검증 약함, 혼자지만 멀티기기.

**불변경계 전면 재검토 결과 — v4의 "정적 HTML·서버0·localStorage"를 폐기한다.**
- 폐기: 정적 HTML 더블클릭 실행, 서버 0, localStorage 단일기기.
- 신규 경계: **상주 파이썬 봇 + Discord(전송·UI·동기화) + 파일/JSON 상태(`store.py`)** + (AI 모드) Claude 브리지 패턴 복제.
- 멀티기기 = Discord 동기화로 자연 해결(별도 동기화 코드 불요).

## 3. 설계 원칙

1. **★과목 무관(엔진·하네스 무오염).** v4의 "엔진 무오염 + 과목 주입(config-driven)" 원칙 계승. 프레임워크는
   범용 학습 타입·범용 AI 모드 골격만 제공. 콘텐츠·과목·구체 AI 모드는 소비 프로젝트가 주입한다.
   영단어·문법·영작 등은 **예시일 뿐 설계에 박지 않는다**(이 리포엔 목업 예시만, 실제 과목 콘텐츠 없음).
2. **★복사·소비형 키트 + 구동 skill(discord-bridge 모델).** APP/DATA 분리 — 이 리포 = 고정 APP 키트
   (봇·엔진코어·harness·skill). 소비 프로젝트 = 콘텐츠·config(성격부여). "앱은 고정, 콘텐츠만 갈아끼움."
   global skill이 어떤 소비 폴더에서든 그 폴더 콘텐츠를 대상으로 봇을 구동한다(§8).
3. **로컬·개인용.** 1인 전용(allowlist 게이팅). 비밀값 비노출. 토큰당 봇 1인스턴스.
4. **결정적=토큰0, AI=선택적 토큰.** 일상 인출·채점·SRS는 토큰 0(결정적). AI 모드만 구독 토큰을 쓰며, 통제 장치 동반.
5. **빼기 미학(소비 앱 한정).** 프레임워크 카탈로그는 최대 망라(골라 쓰도록)하되, 실제 소비 과목 앱은 핵심만 남긴다.

## 4. 스코프 경계 (의식적 트레이드오프 — 구멍 아님)

- **v1 제외 = 연속 인터랙티브 실습**(슬라이더→3D 로봇팔 실시간, JSCAD 3D 모델링, 라이브 코드 에디터,
  인터랙티브 스프레드시트)과 **장문 개념 정독**. Discord 컴포넌트로 못 덮는 영역 → 웹(frontend-design) 후순위(§9 `웹/`).
- **voice(리스닝·발음, PyNaCl/ffmpeg heavy)·게이미피케이션(역할·스티커 등) 제외** — 핵심 학습에 집중.
- **실제 과목 콘텐츠 없음** — 개발·검증용 목업 예시만(`_예시/`).
- **합격확률 예측 없음**(v4 계승) — proxy 라벨(인출성취율)만, 착각확신 방지.
- **자동채점 경계** — 결정적 채점은 텍스트/선택 인출에 한정. 연속 조작 실습 측정은 웹(후순위).

## 5. 두 기둥 (범용 골격)

- **A. 결정적 학습 엔진 + Discord 학습 프리미티브** (토큰 0).
  - 채점·정규화·Leitner·출제큐·대시보드 집계를 v4 JS 코어에서 **파이썬으로 포팅**(§7).
  - 진도 영속 = `store.py`(JSON 파일, localStorage 대체). 봇 재시작 생존 = `persistviews`.
- **B. AI 학습 모드 골격** (선택적 토큰).
  - `claude -p --input-format stream-json --output-format stream-json` `_invoke` 어댑터 패턴을 bridge.py에서
    **복제**(~140줄, light 형태). 구체 AI 모드는 소비 프로젝트가 주입.
  - 토큰 통제: 짧은 preamble, `--effort low`, 저렴 모델, 조건 미충족 시 미호출(예: due=0 → 스킵).
  - 결정적 데이터(예: 사용자 큐레이션 목록)를 재료로 AI가 실습을 생성·채점하는 producer→consumer 패턴 지원.

## 6. 범용 학습 능력 카탈로그 (4계층, 망라)

> lean 추리기가 아니라 최대 커버리지(소비 과목이 골라 쓰도록). 상세 per-능력 매핑표(학습가치·프리미티브·
> 상호작용·tier)는 `_이데이션_능력카탈로그.md` + ② 규격에서 전개. 아래는 계층·능력명 요약.
> **★ 경계: 학습 알고리즘(인터리빙·큐·Leitner)은 엔진 코어(파이썬) 소관, harness는 Discord I/O·진도 저장만**
> (이데이션 비평이 잡은 false-positive — 인터리빙을 `store.py`에 오귀속 — 교정).

**(1) 엔진 코어 능력** (파이썬 포트, Discord 무관)
- 채점 4모드: exact / keyword / cloze / self (+ 순서 sequence). 전부 이진 정/오.
- Leitner 전이(cold 첫시도 정답만 승급·졸업), buildQueue(**인터리빙**·due-first·weight·D-day), 대시보드 집계
  (인출성취율·약점·합격경로·완주분포), `store` 영속, 마이그레이션 shim.

**(2) Discord 학습 능력** (harness 프리미티브)
- 제시·입력: `cards`/`embeds`(문제·해설 렌더) · `form`(단답·cloze 다칸·서술 모달) · `buttons`/`selects`(객관식) ·
  `reactions`(초저마찰 O/X) · `poll` · `confirm`(자가 O/X).
- 피드백: 즉시 카드 피드백 · `matched`/`missed` 하이라이트 · concept-ref 링크 · `paginator`(개념·오답 넘김).
- 회상 강화: 자신감 평정(metacognition, 정답공개 전) · 힌트 점진공개 · preview-then-test · 정교화질문 · 변형질문.
- SRS·세션: due 푸시 알림(통합: 채널 파라미터) · 세션 스레드 · 진행바 · 시험모드 지연피드백.
- 진단·대시보드: live 대시보드(통합) · box 분포 테이블 · 숙달 차트 이미지 · 주간 다이제스트 · `forum` 약점 위키.
- 콘텐츠: `filedrop` 핫리로드(콘텐츠 폴더 감시→덱 반영) · 우클릭 큐레이션(`contextmenu`).

**(3) AI 학습 능력** (`_invoke` 어댑터)
- 오픈엔드 채점(서술·번역·단답) · 소크라테스 대화 · 힌트 생성 · 항목/예문 생성(시드 리스트) ·
  개인화 피드백(오답 이력 기반) · 오개념 탐지 · 적응형 난이도 · AI 세션 요약 · 스트리밍 렌더 · 페르소나(`webhook`).
- 각 항목에 토큰 통제 명시(짧은 preamble · effort low · 저렴 모델 · 미충족 시 미호출).

**(4) 인프라 기반** (foundation, 학습코어 아님 — 정확성·생존·보안)
- `gating`(인증) · `heartbeat`(배치 드라이버 생존) · `ratelimit`(coalescing 공통 베이스) · `events`(트리거) ·
  `checklist`/`botinvite`(권한 사전검증) · `presence`(상태 신호) · `channels`/`categories`(과목 스캐폴드).

**dedup 반영(이데이션 비평):** 중복 6쌍 병합 — LiveCard↔MetricDashboard(코드 동일) · Due 알림 3종(→채널 파라미터) ·
자유회상↔서술(→입력경로 2종) · 분산복습 알림↔복습세션 · 자가갱신SRS↔복습세션 · 시험모드↔시간차 자기퀴즈.

## 7. 엔진 포팅 개요

- **순수함수 7개**(JS→Python): `score` · `normalize`(규칙 9종 + lower 직후 synonyms) · `leitnerTransition` ·
  `nextDueAt` · `isDue` · `buildQueue` · `getDashboardData`. 전부 결정적·부수효과 없음(~500–600줄).
- **localStorage → `store.py`**: `loadProgress`/`saveProgress`를 JSON 파일 I/O + `.bak` 백업으로 재구현.
- **회귀 ~77케이스**: 기존 `score.test.js` 52 포팅 + Leitner/큐/대시보드 신규 ~25(v4 미작성분, ② 검증 게이트).
- 포팅 소스 = `_archive_v4/엔진/app/app.js`(라인 인벤토리 확보). 테스트 주도 포팅.
- **★ 엔진 vs harness 경계**: 학습 알고리즘은 엔진 코어, harness는 Discord 표현·진도 파일저장. 혼동 금지.

## 8. 아키텍처

- **상주 파이썬 봇** = 자기 discord.py Client(자기 토큰). 이벤트 루프·핸들러는 봇이 자작하되 복사한 harness
  프리미티브로 조립. 진도 = `store`(JSON). 영구버튼 = `persistviews`. 푸시 = `scheduler`/`watcher` + `mention`.
  AI = `_invoke` 복제(driver 배치).
- **프레임워크 ↔ 소비 프로젝트 경계**: 봇·엔진·harness·skill = 고정 APP(이 리포). 콘텐츠·config·구체 AI 모드 =
  소비 프로젝트 주입(주입 인터페이스는 ② 규격). 봇은 콘텐츠 폴더만 바꿔 마운트.
- **배포·구동 skill 모델**: global skill이 어떤 소비 폴더에서든 그 폴더 콘텐츠를 대상으로 봇을 구동
  (discord-bridge가 cwd를 Discord에 올리듯). skill 출하 + install 스크립트(소비 프로젝트가 1회 설치).
- **★ 봇 아이덴티티 격리(설정 확정)**: discord-bridge와 **완전 분리** — 새 Discord 앱 + 새 봇 토큰 +
  학습 전용 **새 서버(길드)**. 자기 `.env`(4키: 토큰·길드ID·학습채널ID·허용사용자ID). "토큰당 1봇"이라 토큰을
  분리해야 브리지와 동시 구동 가능. 브리지 토큰·채널·서버 재사용 금지.
- **★ 복사 모델, 런타임 의존 0**: discord-bridge를 import·실행·호출하지 않는다. 복사만 — ①`harness/` 카탈로그를
  자기 폴더로 복사 ②AI용 `_invoke` 어댑터 패턴 발췌 복사. bridge.py 전체 이식·실행 없음. 형제 키트(코드 차용),
  런타임 링크 아님.

## 9. 파티션맵

```
학습 프레임워크 제작/        ← 범용 프레임워크 리포(과목·콘텐츠 없음)
├ 기획_v5_discord.md        ← 이 문서(지속상태)
├ _이데이션_능력카탈로그.md ← 4계층 능력 카탈로그(② 규격 입력)
├ _archive_v4/              ← 옛 HTML 프레임워크 통째(포팅·참고 소스)
├ harness/                  ← Discord Agents/harness 복사본(다음 사이클)
├ 봇/                       ← 파이썬 학습 봇 셸(다음 사이클)
├ 엔진코어/                 ← 포팅된 파이썬 채점·Leitner(다음 사이클)
├ 규격/                     ← 새 규격 + 주입 인터페이스 계약(② 단계)
├ skills/                   ← 구동 skill 소스 + install 스크립트(소비 프로젝트용, 다음 사이클)
├ _예시/                    ← 개발·검증용 목업 콘텐츠만(실제 과목 아님)
└ 웹/                       ← frontend-design 작업공간(자리 예약, 후순위)
```

- **`웹/` 예약**: Discord가 못 덮는 학습(3D·FK·라이브에디터·시트, 장문 정독)용 frontend-design 작업공간.
  v1 구현 범위 아님 — 위치·역할만 기획에 명시해 여지를 남긴다.

## 10. 얇은 슬라이스 (첫 증분)

**엔진·규격 먼저(과목 무관).** 파이썬 채점·Leitner 코어 + 회귀 테스트 → 최소 봇(범용 회상 카드 1종 +
`store` 진도 + `persistview` 영구버튼)을 **목업 예시 콘텐츠**로 동작 증명. 전 루프(제시→입력→채점→Leitner→
저장→재출제)와 Discord 표현을 동시 증명. 콘텐츠 대량·AI 모드는 그 후.

## 11. 백로그 / 후순위

- 웹 실습(3D 모델링·로봇팔 FK·라이브 코드 에디터·인터랙티브 시트) = `웹/` frontend-design(후순위).
- 시각·코드 문제(도형·수식·플롯 이미지 렌더, 코드 제출 채점) = 결정적 v1 이후.
- voice(리스닝·발음)·게이미피케이션(역할·스티커·레벨) = 범위 외(필요 시 소비 프로젝트가 확장).
- full 모드 멀티에이전트 오케스트레이션 = 학습 봇 비범위(브리지 영역).
