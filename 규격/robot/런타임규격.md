## 3. FK 채점 인터페이스

### 3-1. 채점 절차 (의사코드)

```
채점(activity, angles):
  links       ← activity.front.links          // [{length}, ...]
  target      ← activity.front.target         // {x, y}
  tolerance   ← activity.front.tolerance      // number > 0
  joint_limits← activity.front.joint_limits   // [[min,max], ...]

  // 관절한계 위반 검사 (각도 클램핑 아닌 검출, feedback에 명시)
  violations ← []
  for i in 0..angles.length-1:
    lim ← joint_limits[i]
    if angles[i] < lim[0] OR angles[i] > lim[1]:
      violations.push({ joint: i, angle: angles[i], min: lim[0], max: lim[1] })

  // FK 순방향 계산 (평면 2D, Z축 고정)
  ee ← fk_planar(links, angles)   // §3-2

  // 유클리드 거리
  dx       ← ee.x - target.x
  dy       ← ee.y - target.y
  distance ← sqrt(dx*dx + dy*dy)

  // verdict 결정
  // 관절한계 위반이 있어도 거리 기준으로 verdict 결정 (위반은 feedback에만 명시)
  if distance <= tolerance:
    verdict ← 'correct'
  else:
    verdict ← 'incorrect'

  // feedback 생성 (§3-3)
  feedback ← buildFeedback(ee, target, distance, dx, dy, violations)

  return ScoreResult(verdict, score_raw, feedback, grader_id='engine')
```

### 3-2. FK 순방향 계산 (평면 2D)

```
fk_planar(links, angles) → {x: number, y: number}

  x ← 0
  y ← 0
  cumAngle ← 0
  for i in 0..links.length-1:
    cumAngle += angles[i]          // 누적 관절각 (라디안)
    x        += links[i].length * cos(cumAngle)
    y        += links[i].length * sin(cumAngle)
  return {x, y}
```

- 기준 좌표계: 베이스(joint 0) = 원점 (0, 0).
- Y축 양의 방향 = 화면 위쪽 (three.js 기본 좌표계 — `renderer.domElement` 기준).
- 각도 단위: **라디안**. `score()` 입력 `userAnswer`(angles 배열)도 라디안.
- 수식: 끝점 = Σ(L_i · [cos(Σθ_0..i), sin(Σθ_0..i)])

### 3-3. feedback 구조 (기계검증 가능, 진단형)

```ts
// 플러그인계약 §4 ScoreResult.feedback 필드 — 이진 pass/fail 금지
interface RobotFKFeedback {
  end_effector:     [x: number, y: number];   // FK 계산 끝점 좌표
  target:           [x: number, y: number];   // 목표 좌표
  distance:         number;                   // 유클리드 거리 (≥ 0)
  dx:               number;                   // ee.x - target.x
  dy:               number;                   // ee.y - target.y
  hint:             string;                   // 방향 진단 메시지 (§3-4)
  joint_violation?: string;                   // 관절한계 위반 메시지 (위반 없으면 필드 없음)
}
```

**joint_violation 조건부 규칙**: 위반 관절이 1개 이상이면 반드시 포함. 없으면 필드 자체 생략(undefined).

### 3-4. hint 생성 규칙

hint는 학습자가 **어느 방향으로 슬라이더를 조정해야 하는지** 알 수 있는 진단 문자열.

| 조건 | hint 문자열 (예시) |
|---|---|
| distance ≤ tolerance | "목표에 도달했습니다!" |
| `dy < -tolerance * 0.3` (끝점이 목표보다 아래) | "끝점이 목표보다 y축 아래에 있습니다." |
| `dy >  tolerance * 0.3` (끝점이 목표보다 위) | "끝점이 목표보다 y축 위에 있습니다." |
| `dx < -tolerance * 0.3` (끝점이 목표보다 왼쪽) | "끝점이 목표보다 x축 왼쪽에 있습니다." |
| `dx >  tolerance * 0.3` (끝점이 목표보다 오른쪽) | "끝점이 목표보다 x축 오른쪽에 있습니다." |
| `|dx| ≤ tolerance * 0.3` AND `|dy| ≤ tolerance * 0.3` | "거의 도달했습니다. 미세 조정이 필요합니다." |
| 조건 복합(dx, dy 모두 큰 경우) | 주된 방향(절댓값 큰 쪽) 우선 명시 |

- 구현자는 위 예시를 **참고** 형식으로 사용. 정확한 문자열은 자유. **단**: 방향 정보(위/아래/왼쪽/오른쪽 중 하나 이상) 반드시 포함.
- 관절한계 위반 시: hint와 **별도로** `joint_violation` 필드에 메시지 추가. 예: `"관절 1번(현재 2.09rad)이 최대 한계(1.57rad)를 초과했습니다."`

### 3-5. ScoreResult 반환 형태

플러그인계약 §4 ScoreResult conform:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect";
  score_raw: number;              // correct → 1.0, incorrect → 거리 기반 0..1 (§3-6 참조)
  grader_id: "engine";
  feedback:  RobotFKFeedback;    // §3-3
}
```

### 3-6. score_raw 계산

```
if verdict === 'correct':
  score_raw ← 1.0
else:
  // 최대 도달 가능 거리 = Σ links[i].length (모든 링크 완전 뻗은 경우)
  max_dist ← sum(links[i].length)
  score_raw ← max(0, 1 - distance / max_dist)
```

- 정답(correct) 시 항상 1.0.
- 오답 시 목표에 가까울수록 높은 부분점수(학습 진도 파악용).
- `max_dist` 계산 실패(0 또는 NaN) 시 score_raw = 0 (graceful).

## 4. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. robot-arm 전용 사항:

### 4-1. mount(container, ctx)

```
1. practice-runner 인스턴스 생성 (실습러너계약 §4 패턴 준수):
   _state.runner = createPracticeRunner({
     container: container,
     onScore: ...,
     onActivityComplete: result => ctx.emit({type:'activity-completed', result})
   })

2. window.ACTIVITIES['robot-arm'] 에서 첫 번째 activity 로드:
   _state.runner.loadActivity(activities[0])

3. UI 구성 (practice-runner가 canvas 주입):
   - 관절각 슬라이더 N개 (N = front.links.length):
       type=range, min/max = joint_limits[i] (라디안), step=0.01
   - 끝점 좌표 표시 레이블: "끝점: (x.xx, y.xx)"
   - 목표점 마커: three.js 씬에 표시
   - "제출·채점" 버튼

4. 슬라이더 input 이벤트 → FK 실시간 계산 → three.js 암 렌더 갱신 + 끝점 좌표 레이블 갱신

5. 진도 복원: onProgressRestored()에서 저장된 angles 복원 → 슬라이더 초기값 설정

6. mount 반환: Promise<void> (즉시 resolve)
```

### 4-2. score(userAnswer)

```ts
// userAnswer: number[]  — 관절각 배열 (라디안), length = front.links.length
// 반환: Promise<ScoreResult>
score(userAnswer: number[]): Promise<ScoreResult>
```

- `userAnswer` = 슬라이더 현재 값 배열(라디안).
- §3-1 채점 절차 실행 → ScoreResult 반환.
- 동기 계산이지만 인터페이스 통일을 위해 `Promise.resolve()` 래핑.
- **throw 금지**: 입력 오류(배열 길이 불일치 등) 시 verdict='incorrect' + feedback.hint에 오류 내용.

#### 입력 방어

| 조건 | 처리 |
|---|---|
| `userAnswer` null / 비배열 | 전 관절 0으로 대체하여 계산 |
| 배열 길이 < links.length | 부족분 0 채워서 계산, feedback.hint에 명시 |
| 배열 길이 > links.length | 앞 links.length 개만 사용 |
| NaN 포함 요소 | 해당 요소 0으로 대체 |

## 5. three.js 씬 구성 (practice-runner 내부)

> 상세 구현은 실습러너계약 §6 참조. 여기서는 robot-arm 관련 씬 요소만 정의.

### 5-1. 씬 요소 목록

| 요소 | three.js 객체 | 설명 |
|---|---|---|
| 베이스 | `CircleGeometry(r=6)`, `MeshBasicMaterial({color:0x444444})` | 원점 표시 |
| 링크 i | `BoxGeometry(length_i, 8, 1)` | 링크 막대. 피벗 = 좌측 끝 기준 offset |
| 관절 i | `CircleGeometry(r=5)`, `MeshBasicMaterial({color:0x888888})` | 관절 원 |
| 끝점 | `CircleGeometry(r=5)`, `MeshBasicMaterial({color:0x1f6b4a})` | 현재 끝점 (초록) |
| 목표 마커 | `CircleGeometry(r=7)`, `MeshBasicMaterial({color:0xa8301f,transparent:true,opacity:0.85})` | 목표점 (빨강) |
| 허용 반경 | `RingGeometry(tolerance-1, tolerance+1)` 또는 `CircleGeometry` 반투명 | tolerance 시각화 |

### 5-2. 업데이트 루프

```
슬라이더 input 이벤트 발생 시:
  angles ← 슬라이더 현재값 배열 (라디안)
  ee     ← fk_planar(links, angles)         // §3-2
  링크/관절 three.js 메시 위치·회전 갱신
  끝점 메시 위치 = (ee.x, ee.y, 0)
  끝점 좌표 레이블 갱신: "끝점: (ee.x.toFixed(1), ee.y.toFixed(1))"
  renderer.render(scene, camera)
```

- 애니메이션 루프(`requestAnimationFrame`) **불필요** — 슬라이더 이벤트 드리븐으로 충분. 불필요한 배터리 소모 방지.
- `dispose()` 시 추가 `requestAnimationFrame` 취소 로직 불필요(루프 없음).

### 5-3. 카메라 설정

```
THREE.OrthographicCamera:
  left/right/top/bottom = container 크기 기반 자동 계산
  near = -100, far = 100
  zoom = 1 (ResizeObserver로 자동 갱신)
```

- 원점 = 베이스 관절 = 화면 중앙 하단 (또는 중앙).
- Y 좌표 = 화면 위쪽 양수 (수학 좌표계와 동일).
