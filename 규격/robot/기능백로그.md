## 1. core 기능 (MVP, 학습루프 완결에 필수)

| # | 기능 | 설명 | 근거 |
|---|---|---|---|
| C1 | FK 슬라이더 UI | N개 슬라이더 → 실시간 관절각 입력. 각 슬라이더: min/max = joint_limits (라디안), step=0.01, 레이블 = "관절 i (현재: X.XX rad)" | 원칙1 — 학습루프 진입점 |
| C2 | three.js 2D 암 실시간 렌더 | 슬라이더 input 이벤트 → fk_planar 계산 → 메시 갱신. 베이스/링크/관절/끝점/목표 마커 표시 (런타임규격 §5) | 원칙1 |
| C3 | 끝점 좌표 실시간 표시 | "끝점: (x.x, y.y)" 레이블 — 슬라이더 조작마다 갱신 | 원칙2 — 규격 정의됨 |
| C4 | 목표점 마커 표시 | three.js 씬에 빨간 원 + tolerance 반경 시각화 | 원칙1 |
| C5 | 제출·채점 버튼 | 클릭 시 `score(angles)` 호출 → 결과 표시 | 원칙1 |
| C6 | FK 채점 (auto) | 끝점-목표 유클리드 거리 ≤ tolerance → correct. 런타임규격 §3 완전 conform | 원칙1 |
| C7 | 진단형 feedback 렌더 | distance, dx, dy, hint, joint_violation 렌더. 이진 pass/fail 금지 (실습러너계약 §9 V9) | 원칙2 + Hattie 피드백모델 |
| C8 | 해설 패널 (back 렌더) | 채점 후 back.why + back.explanation 표시. example_solution_angles 있으면 함께 표시 | 원칙2 — 규격 정의됨, 미구현 방지 |
| C9 | 관절한계 위반 경고 | 위반 시 feedback.joint_violation 렌더 + 슬라이더 색상 변경(빨강) | 원칙2 |
| C10 | 진도 저장/복원 | getProgressSnapshot / onProgressRestored — last_angles로 슬라이더 복원 | 플러그인계약 §3·§5 |
| C11 | practice-runner 연동 | mount 패턴 §4 준수 (createPracticeRunner + ctx.emit). 실습러너계약 V5·V6 | 실습러너계약 §4 |
| C12 | activity nav (복수 문제) | activities 2개 이상 시 nav 버튼 표시. 문제 전환 시 씬·슬라이더 초기화 | 원칙1 (콘텐츠 우선) |
| C13 | dispose 멱등성 | `dispose()` 2회 연속 호출 → throw 없음 (실습러너계약 V3) | 계약 필수 |
| C14 | graceful score() | userAnswer null/NaN/길이 불일치 → throw 없이 incorrect 반환 (런타임규격 §4-2) | 원칙5 대비 안전망 |

## 2. mid 기능 (학습효과 보강, C1~C14 완료 후)

| # | 기능 | 설명 | 근거 |
|---|---|---|---|
| M1 | 끝점 궤적 표시 | 슬라이더 조작 중 끝점 이동 경로를 화면에 선으로 잔상 표시. 500ms 후 자동 소거 | 공간감 학습 보조 |
| M2 | 도달 가능 영역 표시 (워크스페이스) | Σlinks 반경 원을 반투명 overlay로 표시 — "이 범위 안에서만 목표 가능" | FK 개념 직관화 |
| M3 | activity 완료 뱃지 | 정답 달성 nav 버튼에 ✓ 뱃지 (coding 플러그인 패턴 미러) | 동기부여 |
| M4 | example_solution_angles 시각화 버튼 | "예시 풀이 보기" 버튼 클릭 시 example_solution_angles로 암 자동 이동 (슬라이더 갱신) | 원칙2 — back 필드 활용 |
| M5 | 각도 단위 토글 (rad/deg) | 슬라이더 레이블을 라디안 ↔ 도(degree) 전환. 내부 계산은 항상 라디안 | 비전공자 접근성 |
| M6 | 풀이 시간 수집 (solve_ms) | mount→score 경과 ms 기록. plugin_extra에 저장 (coding 플러그인 패턴 미러) | 학습 데이터 |

## 3. low 기능 (선택, 검토 필요)

| # | 기능 | 설명 | 제외/보류 사유 |
|---|---|---|---|

| L4 | 앞면 통계 배지 | 슬라이더 옆 시도/정답 통계 표시 | 원칙4 인출순도 침범 |

## 4. 하드플로어 (기능 작업 전 선수정)

| # | 사안 | 조치 |
|---|---|---|
| HF1 | `plugin_id = "robot-arm"` 정규식 준수 확인 | `^[a-z][a-z0-9-]{2,63}$` 통과 — 하이픈 허용, 이미 준수 |
| HF2 | `scoring_mode = "auto"` 셸 분기 확인 | shell.js에서 robot-arm의 auto 분기 정상 동작 확인 필요 |
| HF3 | `capabilities = ['practice']` 셸 부트 분기 없음 확인 | shell.js가 'practice' 토큰으로 하드분기 하는지 확인 (없으면 OK, 있으면 파킹 보고) |

## 5. 셸 백로그로 이관 (_방향성 원칙3)

아래 항목은 robot-arm 단독 구현 금지 → 공통셸 백로그 대상:

- 세션 목표·분량 설정 (`session-done` 이벤트는 셸 소비 계약)
- plugin_extra 크기 가드 (last_angles 누적 시 localStorage 5MB 초과 방지)
- 진도 내보내기/가져오기 UI
