## 1. Leitner 코어

### 1.1 박스·졸업
- `box ∈ {1..3}`. 신규 초기 box=BOX_MIN. 간격 = `LeitnerConfig.intervals_days` 또는 기본.
- **졸업**: `box==BOX_MAX`에서 다시 cold 정답 → `graduated=True`(box 유지, due 재설정). 졸업 카드도 due면 큐(영구 제외 아님).

### 1.2 시도 분류
- **cold**: 세션 첫 시도(card_id별 1회). **warm**: 재큐 2회차+. skip도 시도(분류·add).
- 판정 = 세션 휘발 `seen_card_ids: set[str]`. **시도 직전(score/verdict 호출 전)** 미존재→cold, 존재→warm 판정 **후 즉시 add**(봇계약 §4 정렬).

### 1.3 승급/강등 (전이표)
입력 `(state, attempt_kind, verdict, now, cfg?, dday_mode)` → 새 `CardProgress`(불변 복사).

| attempt_kind | verdict | box | due_at | graduated |
|---|---|---|---|---|
| cold | correct | `min(box+1, BOX_MAX)` | `now + interval[new_box]*MS_PER_DAY` | box==BOX_MAX였고 정답이면 True |
| cold | incorrect | `BOX_MIN` | `now + interval[BOX_MIN]*MS_PER_DAY` | False |
| warm | correct/incorrect | 변화 없음 | 변화 없음 | 변화 없음 |
| cold/warm | skip | 변화 없음 | 변화 없음 | 변화 없음 |

- box 이동 = cold 첫시도 정답만(불변). 이진(partial 없음).
- cold면 `cold_attempts += 1`(verdict 무관), cold+correct만 `cold_correct += 1`. skip은 cold_attempts 카운트, cold_correct 미증가.
- D-day 압축(정답): `due_at = now + min(원래간격, dday_compress_days)*MS_PER_DAY`. 오답=일반.
- 졸업 카드 재전이: cold 정답→box=MAX 유지·due 재설정·graduated 유지. cold 오답→일반 강등(graduated→False).

### 1.4 시그니처
```python
def leitner_transition(state, attempt_kind, verdict, now, cfg=None, dday_mode=False) -> CardProgress: ...
def next_due_at(box, now, cfg=None) -> int: ...
def is_due(state, now) -> bool: ...   # now >= state.due_at
```

## 2. 출제큐

### 2.1 소스
- `new` = `cold_attempts==0`(progress 미존재 포함). `review` = `cold_attempts>0` 그리고 `is_due`(졸업 카드도 due면 review). due 아닌 학습이력 = 제외(D-day 예외).

### 2.2 우선순위·weight
정렬키(상위→하위): 1) review > new 2) review 내부 `weight DESC` 3) new 내부 `weight DESC` 4) **인터리빙**(due group 내부 단원 라운드로빈, 우선순위 불파괴) 5) tie-break `card_id ASC`.
- **weight 해소**: `opts.weight_overrides[card_id]`(사이드카 주입, SoT §2.1) 있으면 그 값, 없으면 `CardDef.tags.weight`. 정상화: 범위밖/비정수→[1,10] 클램프, NaN/비숫자→5.
```python
def build_queue(cards, progress: ProgressStore, now, opts: QueueOptions) -> list[str]: ...   # card_id 순서
# progress = ProgressStore(get_dashboard_data와 동일 타입). card_id 조회는 progress.cards.get(id).
```
- 빈 결과=`[]`. 고아 progress(cards에 없음)=큐 제외, 삭제 안 함.

### 2.3 D-day 모드
- `opts.dday_mode==True`(봇 UI/명령이 판정 주입, 엔진은 판정 안 함). 전 박스+new 소환. 정렬 `box ASC → weight DESC → 단원 라운드로빈 → card_id ASC`. limit 각 그룹 정렬 후 절단(이월 없음).

## 3. 채점코어

### 3.1 4모드 (전부 이진)
| mode | 입력 | 로직 | 결과 |
|---|---|---|---|
| exact | str | 정규화 후 accepted 완전일치(OR) | correct/incorrect |
| keyword | str | 정규화 후 required_keywords 전 그룹 충족(그룹별 any-of) | correct/incorrect |
| cloze | list[str] | 빈칸별 일치, 전부 일치→correct | correct/incorrect |
| self | "correct"/"incorrect" | 그대로 채택 | correct/incorrect |
- 부분점수 없음. matched/missed=피드백용.
- **recall_seq**: CardType `recall_seq`는 **exact 모드로 채점**(answer_spec.sequence 순서포함 완전일치, 길이 불일치=incorrect). 별도 ScoreMode 아님(ScoreMode는 4종).

### 3.2 입력 계약 (위반 시 `ScoreInputError`)
- exact/keyword→str. cloze→list[str](len==len(blanks)). self→"correct"|"incorrect". recall_seq 채점은 exact 경로로 list 비교(sequence).
- 빈 입력(정규화 후 "")=incorrect(throw 아님). skip은 score() 호출 안 함.

### 3.3 정규화 카탈로그 (9종, 양변 동일 적용, 순서대로)
| id | 동작 |
|---|---|
| `nfkc` | `unicodedata.normalize("NFKC", s)` |
| `trim` | `s.strip()` |
| `collapse_space` | 모든 유니코드 공백류 → 단일 스페이스 |
| `strip_all_space` | 모든 공백류 제거 |
| `lower` | ASCII 영문 소문자화 |
| `fullwidth_to_halfwidth` | 전각 영숫자·기호 → 반각 |
| `unify_cell_dollar` | 셀참조 `$` 제거 |
| `unify_arg_sep` | `;`·전각`，` → `,` |
| `strip_trailing_paren` | 후행 `(...)` 제거 |

- **synonyms 치환 위치(계약, SoT §7.10)**: `lower` 직후·비교 직전. 정확일치(부분문자열 아님), 다어절 구는 v1 밖. (토큰화 방식은 구현 자유 — 계약은 위치·정확일치만.)

### 3.4 출력
```python
def score(inp: ScoreInput) -> ScoreResult: ...
def normalize(s, rules: list[str], synonyms: dict[str,str] | None = None) -> str: ...
```
- cloze matched/missed/highlight_missed=빈칸 인덱스 문자열. self: user_answer 그대로 verdict. 결정성 보장. 정답률·합격확률 산출 안 함.

## 4. 마이그레이션 (migrate, 엔진 순수)

> **파일 읽기·쓰기(load/save)는 엔진이 아니라 봇(harness store.py).** 엔진은 버전 정규화 `migrate`만(순수). SoT §2·§3·§7.4.

```python
def migrate(raw: dict) -> ProgressStore: ...   # v(n)->v(n+1) 단계 체인, 순수, 누락=기본값
def new_card_progress(card_id: str) -> CardProgress: ...   # 기본값 생성(순수)
```
- 신규 기본: `{box:BOX_MIN, due_at:0, graduated:False, cold_attempts:0, cold_correct:0, last_attempt_at:None, last_verdict:None}`(due_at:0=즉시 due).
- **v0(레거시) = v4 ProgressStore 형태**: 필드명은 v4도 snake_case(`card_id·box·due_at·graduated·cold_attempts·cold_correct·last_attempt_at·last_verdict`)라 **near-identity**. v0→v1 변환 = `schema_version` 필드 주입 + 누락 필드 기본값 채움. 미래 단계는 체인에 추가.
- 봇 흐름: harness store가 JSON 읽기 → `migrate(raw)`로 정규화 → ProgressStore. 저장 시 `schema_version=SCHEMA_VERSION` 강제. parse 실패·migrate 예외는 봇이 빈 스토어 폴백+`.bak`(SoT §2).

## 5. 대시보드 집계

```python
@dataclass
class ByAreaEntry:     area: str; subarea: str; retrieval_rate: float | None   # cold_correct/cold_attempts, cold0→None
@dataclass
class WeaknessEntry:   area: str; subarea: str; unit: str; wrong_rate: float    # cold오답/cold_attempts
@dataclass
class PassPathEntry:   area: str; subarea: str; target: int; coverage: float; mastery: float; progress: float; status: Literal["safe","watch","danger"]
@dataclass
class CompletionEntry: area: str; subarea: str; box_dist: dict; mastery_rate: float   # box_dist={box1,box2,box3}

def get_dashboard_data(deck: DeckData, progress: ProgressStore, now: int,
                       pass_targets: dict[str, int] | None = None) -> DashboardData: ...
```
순수 집계(파일 I/O 0). `enabled==True` 카드만. progress 미존재=미학습(cold_attempts 0, box1).
- **by_area**[(area,subarea)]: `retrieval_rate = cold_correct/cold_attempts`, cold0→`None`.
- **weakness**[(area,subarea,unit)]: `wrong_rate`, cold0 단원 제외. 정렬 `wrong_rate DESC → unit ASC`.
- **pass_path**[(area,subarea)]: **`target = pass_targets[subarea]`**(주입, 과목 어휘·임계 하드코딩 금지 — SoT §7.1). `pass_targets` 미주입 또는 해당 subarea 키 없으면 그 그룹 **pass_path 제외**(엔진은 합격선 모름). `coverage`=cold≥1 카드/전체, `mastery`=cold 시도 카드 중 정답률≥1 카드 비율(**분모=cold≥1 카드 수**, 0이면 0), `progress=coverage*mastery`(= 전체 중 정복 비율). status(t=target/100): `r≥t`safe / `r≥0.7t`watch / else danger. UI 렌더만.
- **completion**[(area,subarea)]: `box_dist`(미학습=box1, graduated=box3), `mastery_rate=box3/전체`.
- area/subarea 없는 카드: by_area·pass_path·completion 제외(weakness는 unit 기준 포함, 둘 다 없으면 제외). 결정성 보장.
- **`pass_targets` 출처**: 봇 부트가 `SubjectConfig.pass_targets`(주입인터페이스 §5)에서 구성해 주입. 엔진은 dict만 소비.
