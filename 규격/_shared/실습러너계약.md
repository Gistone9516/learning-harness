## 3. createPracticeRunner — 팩토리 시그니처

```js
/**
 * @param {object}   opts
 * @param {HTMLElement} opts.container        — 캔버스/에디터를 주입할 DOM 노드
 * @param {function} opts.onScore             — (ScoreResult) => void  채점 완료 콜백
 * @param {function} opts.onActivityComplete  — (ScoreResult) => void  activity-completed 이벤트 발화 콜백
 * @returns {PracticeRunnerInstance}
 */
function createPracticeRunner(opts)
```

### 반환 타입: PracticeRunnerInstance

```ts
interface PracticeRunnerInstance {
  /** ActivitySpec을 받아 캔버스/에디터 UI를 컨테이너에 렌더 */
  loadActivity(spec: ActivitySpec): void;

  /** 현재 사용자 입력(코드 문자열 또는 관절각 배열 등)을 반환 */
  getUserAnswer(): any;

  /** 캔버스/렌더러/이벤트리스너 정리. unmount() 에서 반드시 호출 */
  dispose(): void;
}
```

- `loadActivity` 반환값 없음 (void). 렌더링은 즉시 동기 시작.
- `getUserAnswer` 반환 타입은 플러그인별 `userAnswer` 형태와 일치:
  - cad-print  → `string` (JSCAD 코드)
  - robot-arm  → `number[]` (관절각 배열, 라디안)
- `dispose` 는 멱등(idempotent) — 여러 번 호출해도 안전해야 함.

## 4. mount 표준 패턴 (두 플러그인 공통)

실습 플러그인 `plugin.js`의 `mount(container, ctx)` 구현은 아래 구조를 따른다:

```js
function mount(container, ctx) {
  _state.runner = createPracticeRunner({
    container: container,
    onScore: function(result) {
      // 진도 캐시 업데이트 (플러그인 내부)
    },
    onActivityComplete: function(result) {
      ctx.emit({ type: 'activity-completed', result: result });
    }
  });
  var activities = (window.ACTIVITIES && window.ACTIVITIES[PLUGIN_ID]) || [];
  if (activities.length > 0) {
    _state.runner.loadActivity(activities[0]);
  }
  return Promise.resolve();
}
```

- `container` = 셸이 제공하는 `<main id="plugin-host">` (플러그인계약 §7).
- `ctx.emit({ type: 'activity-completed', result })` = 셸 진도 저장 트리거 (플러그인계약 §5).
- `mount` 반환값 = `Promise<void>` (계약 §3 준수).

## 5. unmount 표준 패턴

```js
function unmount() {
  if (_state.runner) {
    _state.runner.dispose();
    _state.runner = null;
  }
  if (_state.host) {
    _state.host.innerHTML = '';
    _state.host = null;
  }
  // 나머지 _state 필드 리셋
}
```

## 6. 3D 캔버스 라이프사이클 (practice-runner 내부 책임)

practice-runner가 **단독 소유** — 플러그인이 직접 THREE 객체를 생성·파괴하지 않는다.

| 단계 | 책임 |
|---|---|
| `createPracticeRunner` | `THREE.WebGLRenderer` 생성, `container`에 `<canvas>` append |
| `loadActivity` | 씬 초기화/재초기화, 리사이즈 옵저버 등록 |
| `dispose` | `renderer.dispose()`, `geometry.dispose()`, `material.dispose()`, ResizeObserver 해제, 애니메이션 루프 취소 |

- `ResizeObserver`로 `container` 크기 변화 감지 → `renderer.setSize` 자동 갱신.
- `dispose` 호출 후 renderer · scene · animationFrame 참조는 `null` 처리.

## 7. 채점 트리거 표준

실습 플러그인이 채점을 요청할 때:

```js
// 플러그인 plugin.js 내부 (submit 버튼 클릭 등)
var answer = _state.runner.getUserAnswer();
score(answer).then(function(result) {
  ctx.emit({ type: 'activity-completed', result: result });
});
```

- `score(userAnswer)` = 플러그인계약 §3의 `score()` 그대로.
- `ctx.emit` = 셸 진도 저장 트리거. **반드시 score() 완료 후 호출**.

## 10. feedback 구조 (플러그인별)

### cad-print
```ts
feedback: {
  volume_user:     number,       // 사용자 코드 실행 결과 부피 (cm³ 또는 JSCAD 단위)
  volume_target:   number,       // 목표 부피
  volume_err_pct:  number,       // |user-target|/target * 100
  bbox_diff:       [dx, dy, dz], // 바운딩박스 차이 (절댓값)
  manifold:        boolean,      // watertight 여부 (false → 매니폴드 깨짐)
  hint:            string        // 방향 진단 메시지
}
```
- `manifold=false` 시 `hint`에 "닫힌 솔리드 아님(매니폴드 깨짐)" 포함 필수.

### robot-arm
```ts
feedback: {
  end_effector: [x, y],   // 현재 끝점 좌표
  target:       [x, y],   // 목표 좌표
  distance:     number,   // 유클리드 거리
  dx:           number,
  dy:           number,
  hint:         string,   // 예: "끝점이 목표보다 y축 아래"
  joint_violation?: string  // 관절한계 위반 시 메시지
}
```
