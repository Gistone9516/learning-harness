# english 플러그인 런타임규격 (소비 계약)

> buildflow cycle-2 ②명세. **플러그인계약.md conform** — 충돌 시 플러그인계약 > 본 문서.
> 대상: english 플러그인(`plugin_id = "english"`) 구현자 + 공통셸.
> 범위: ActivitySpec 형태·Web Speech 인터페이스·정적 채점기·BYO-key graceful 처리.

## 1. PluginManifest 선언

```js
// 파일: plugins/english/manifest.js
window.MANIFEST_ENGLISH = {
  plugin_id:              "english",
  label:                  "영어 학습",
  version:                "1.0.0",
  infra:                  "hybrid",         // 정적 핵심 + writing/speaking = BYO-key API
  entry_url:              null,             // hybrid지만 외부 서버 없음; BYO-key API는 클라이언트 직접 호출
  byok: {
    keys: [
      {
        id:    "llm_api_key",
        label: "LLM API 키 (writing 채점용)",
        where: "header",                    // Authorization: Bearer <key>
        help:  "OpenAI / Anthropic 등 OpenAI-compatible 엔드포인트 키. writing 모달 llm-rubric 채점에 사용."
      },
      {
        id:    "azure_speech_key",
        label: "Azure Speech 키 (speaking 발음 채점용)",
        where: "header",                    // Ocp-Apim-Subscription-Key: <key>
        help:  "Azure Cognitive Services Speech API 키. speaking 모달 pronunciation 채점에 사용. 지역(region)은 설정 화면에서 별도 입력."
      }
    ]
  },
  capabilities:           ["language"],
  scoring_mode:           "auto",           // 정적 모달은 auto; writing/speaking은 키 없으면 pending 반환
  activity_type:          "lang-task",
  progress_schema_version: 1
};
```

**infra = hybrid 근거**:
- vocab / grammar / reading / listening → 전부 브라우저 내 JS 채점(키·서버 0). 정적.
- listening → `SpeechSynthesis`(브라우저 내장 TTS)로 `audio_text` 재생, 사용자 받아쓰기 → `dictation-diff` 채점. 외부 API 없음.
- writing → `llm-rubric`(LLM API). BYO-key 필요.
- speaking → `pronunciation`(Azure Speech API). BYO-key 필요.
- 키 없으면 mount는 됨, 채점만 graceful degraded.

## 2. ActivitySpec — `type: "lang-task"`

플러그인계약 §4 ActivitySpec 공통 껍질에 english 전용 `front`/`back`/`grading` 정의.

### 2-1. 완전 형태 (JSON Schema)

```jsonc
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "english/activity-spec/v1",
  "type": "object",
  "required": ["activity_id","plugin_id","type","front","grading","weight","tags","enabled"],
  "additionalProperties": false,
  "properties": {

    // ── 공통셸 공용 필드 (플러그인계약 §4) ──
    "activity_id": { "type": "string", "pattern": "^[a-z][a-z0-9-]{2,63}$" },
    "plugin_id":   { "type": "string", "const": "english" },
    "type":        { "type": "string", "const": "lang-task" },
    "weight":      { "type": "number", "minimum": 1, "maximum": 10 },
    "tags": {
      "type": "object",
      "required": ["area","subarea","unit"],
      "properties": {
        "area":    { "type": "string" },   // 예: "vocabulary", "grammar", "reading", "listening", "writing", "speaking"
        "subarea": { "type": "string" },   // 예: "phrasal-verb", "present-perfect"
        "unit":    { "type": "string" }    // 예: "unit-03"
      },
      "additionalProperties": false
    },
    "enabled": { "type": "boolean" },

    // ── english 전용: front ──
    "front": {
      "type": "object",
      "required": ["modality","prompt"],
      "additionalProperties": false,
      "properties": {
        "modality": {
          "type": "string",
          "enum": ["vocab","grammar","reading","listening","writing","speaking"]
        },
        "prompt":       { "type": "string", "minLength": 5 },
        "passage":      { "type": "string" },          // reading 모달 전용 지문
        "audio_text":   { "type": "string" },          // listening 모달: TTS로 재생할 원문
        "options":      {                              // 객관식 선택지 (있으면 UI가 라디오/버튼 렌더링)
          "type": "array",
          "items": { "type": "string" },
          "minItems": 2,
          "maxItems": 6
        }
      }
    },

    // ── english 전용: back (선택, 해설용) ──
    "back": {
      "type": "object",
      "additionalProperties": false,
      "properties": {
        "answer":      { "type": "string" },
        "explanation": { "type": "string" }
      }
    },

    // ── english 전용: grading ──
    "grading": {
      "type": "object",
      "required": ["mode"],
      "additionalProperties": false,
      "properties": {
        "mode": {
          "type": "string",
          "enum": ["exact","keyword","cloze","dictation-diff","llm-rubric","pronunciation"]
        },
        // exact: accept[] 중 하나와 일치
        "accept": {
          "type": "array",
          "items": { "type": "string" },
          "minItems": 1
        },
        // keyword: keywords[][] — 각 서브배열 = OR 그룹, 전체 AND
        "keywords": {
          "type": "array",
          "items": {
            "type": "array",
            "items": { "type": "string" },
            "minItems": 1
          },
          "minItems": 1
        },
        // cloze: 빈칸 채우기, accept[]와 동일 구조
        // (cloze와 exact는 accept[]를 공유)
        // dictation-diff: expected = audio_text 와 동일하거나 별도 명시
        "expected": { "type": "string" },
        // llm-rubric: LLM에 넘길 루브릭 문자열
        "rubric":   { "type": "string" },
        // dictation-diff: 활동별 정답 임계 재정의 (기본 0.9 — §4-3). cycle-3 core 승격
        "dictation_threshold": { "type": "number", "minimum": 0, "maximum": 1 }
      },
      // 모드별 필수 필드 제약 (구현자 주의)
      // exact | cloze → accept 필수
      // keyword       → keywords 필수
      // dictation-diff→ expected 필수 (없으면 front.audio_text 사용)
      // llm-rubric    → rubric 필수
      // pronunciation → rubric 선택(발음 기준 설명용)
      "if":   { "properties": { "mode": { "enum": ["exact","cloze"] } } },
      "then": { "required": ["accept"] },
      "else": {
        "if":   { "properties": { "mode": { "const": "keyword" } } },
        "then": { "required": ["keywords"] },
        "else": {
          "if":   { "properties": { "mode": { "const": "llm-rubric" } } },
          "then": { "required": ["rubric"] }
        }
      }
    }

  }
}
```

| grading.mode | grader_id | 인프라 | BYO-key |
|---|---|---|---|
| `exact` | `"engine"` | 정적 | 불요 |
| `keyword` | `"engine"` | 정적 | 불요 |

## 3. Web Speech API 인터페이스

### 3-1. SpeechSynthesis (TTS) — listening 모달

**브라우저 지원**: Chrome, Edge, Firefox, Safari, Opera (2026 기준 모든 현대 데스크톱 브라우저). Firefox Android 제외.
SpeechRecognition과 달리 오프라인 작동 가능(시스템 TTS 음성 사용).

```js
// listening 모달 TTS 재생
function speakText(text, opts = {}) {
  return new Promise((resolve, reject) => {
    if (!window.speechSynthesis) {
      reject(new Error("SpeechSynthesis not supported"));
      return;
    }
    const utt = new SpeechSynthesisUtterance(text);
    utt.lang  = opts.lang  ?? "en-US";
    utt.rate  = opts.rate  ?? 0.9;       // 학습용: 약간 느리게
    utt.pitch = opts.pitch ?? 1.0;
    utt.onend   = () => resolve();
    utt.onerror = (e) => reject(e);
    window.speechSynthesis.cancel();     // 이전 발화 중단
    window.speechSynthesis.speak(utt);
  });
}

// 플러그인 내 호출 패턴
async function playAudio(activity) {
  const text = activity.front.audio_text;
  if (!text) throw new Error("audio_text missing");
  await speakText(text, { lang: "en-US", rate: 0.85 });
}
```

**주의사항**:
- Chrome에서 `speechSynthesis.speak()` 호출 전 반드시 사용자 제스처(버튼 클릭 등) 필요. autoplay 불가.
- 음성 목록(`speechSynthesis.getVoices()`)은 비동기 로드. `voiceschanged` 이벤트 대기 후 en-US 음성 선택 권장.
- iOS Safari: 사용자 인터랙션 콜백 내 최초 1회 호출 필요(이후 programmatic 가능).

```js
// en-US 음성 우선 선택 헬퍼
function getEnglishVoice() {
  const voices = window.speechSynthesis.getVoices();
  return (
    voices.find(v => v.lang === "en-US" && v.localService) ||
    voices.find(v => v.lang.startsWith("en-US")) ||
    voices.find(v => v.lang.startsWith("en")) ||
    voices[0] || null
  );
}
```

### 3-2. SpeechRecognition (STT) — speaking 모달 (BYO-key 파킹 경로)

**브라우저 지원**: Chrome, Edge(Chromium) 전용. Firefox, Safari 미지원(2026 기준).
SpeechRecognition은 Google 서버로 오디오를 전송하므로 **인터넷 필수**.

```js
// speaking 모달: 브라우저 STT 보조 입력 (발음 채점은 Azure Speech BYO-key)
function getSpeechRecognition() {
  const SR = window.SpeechRecognition || window.webkitSpeechRecognition;
  if (!SR) return null;           // 미지원 브라우저 → graceful(텍스트 입력 폴백)
  return SR;
}

function startRecognition(onResult, onError) {
  const SR = getSpeechRecognition();
  if (!SR) { onError(new Error("STT not supported")); return null; }
  const rec = new SR();
  rec.lang           = "en-US";
  rec.interimResults = false;
  rec.maxAlternatives = 1;
  rec.onresult = (e) => {
    const transcript = e.results[0][0].transcript;
    const confidence = e.results[0][0].confidence;
    onResult({ transcript, confidence });
  };
  rec.onerror = (e) => onError(e);
  rec.start();
  return rec;   // 호출자가 rec.stop() 책임
}
```

**speaking 모달 폴백 정책**:
- STT(`webkitSpeechRecognition`) 미지원 브라우저 → 텍스트 입력 폼으로 대체(발음 채점은 Azure Speech로 별도 처리).
- `azure_speech_key` 없으면 `verdict: "pending"`, `feedback.message: "Azure Speech 키가 필요합니다. 설정에서 키를 입력하세요."` 반환.

## 4. 정적 채점기 (grader_id = "engine")

### 4-1. exact / cloze 채점

```
채점(activity, userAnswer: string):
  normalized_answer = normalize(userAnswer)         // §4-5
  for each accepted in activity.grading.accept:
    if normalize(accepted) === normalized_answer:
      return ScoreResult("correct", 1.0, "engine", {matched: accepted})
  return ScoreResult("incorrect", 0.0, "engine", {accepted: activity.grading.accept})
```

### 4-2. keyword 채점

```
채점(activity, userAnswer: string):
  norm = normalize(userAnswer)
  groups = activity.grading.keywords     // string[][]
  hit_count = 0
  missed = []
  for each group in groups:
    group_hit = group.some(kw => norm.includes(normalize(kw)))
    if group_hit: hit_count++
    else: missed.push(group)
  total = groups.length
  score_raw = hit_count / total
  verdict = (score_raw === 1.0) ? "correct" : "incorrect"
  return ScoreResult(verdict, score_raw, "engine", {hit: hit_count, total, missed})
```

키워드 채점은 부분점수(`score_raw`)를 지원. `verdict = "incorrect"`이어도 `score_raw > 0`일 수 있음.

### 4-3. dictation-diff 채점

listening 모달 전용. TTS가 재생한 `audio_text`를 사용자가 받아쓴 결과를 비교.

```
채점(activity, userAnswer: string):
  expected_raw = activity.grading.expected ?? activity.front.audio_text
  if !expected_raw: error("dictation-diff requires expected or audio_text")

  expected_tokens = tokenize(normalize(expected_raw))    // §4-5, §4-6
  actual_tokens   = tokenize(normalize(userAnswer))

  // 최장 공통 부분수열(LCS) 기반 단어 단위 diff
  lcs_len = LCS_length(expected_tokens, actual_tokens)
  correct_words = lcs_len
  total_words   = expected_tokens.length
  score_raw     = correct_words / total_words            // 0..1
  verdict       = (score_raw >= 0.9) ? "correct" : "incorrect"
    // 임계값 0.9: 전체 단어의 90% 이상 일치 시 correct
    // 생성규칙에서 grading.dictation_threshold로 재정의 가능 (기본 0.9)

  // diff 하이라이트: 각 expected 토큰의 상태 (match|miss|extra)
  diff_detail = compute_word_diff(expected_tokens, actual_tokens)

  return ScoreResult(verdict, score_raw, "engine", {
    correct_words, total_words,
    diff: diff_detail    // [{word, status: "match"|"miss"|"extra"}]
  })
```

**LCS 구현 상한**: `expected_tokens.length` 최대 200 단어(listening 지문 제한). 초과 시 grading 오류.

### 4-4. ScoreResult 형태 (english 플러그인)

플러그인계약 §4 ScoreResult conform:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect" | "pending";
  score_raw: number;                  // 0..1. pending이면 null
  grader_id: "engine" | "llm" | "external";
  feedback: {
    // exact/cloze
    matched?:   string;
    accepted?:  string[];
    // keyword
    hit?:       number;
    total?:     number;
    missed?:    string[][];
    // dictation-diff
    correct_words?: number;
    total_words?:   number;
    diff?:          Array<{word: string; status: "match"|"miss"|"extra"}>;
    // llm-rubric / pronunciation (pending)
    message?:   string;               // 키 없을 때 안내 문구
    // 공통 에러
    error?:     string;
  };
}
```

### 4-5. normalize() 규칙

```
normalize(s: string): string
  1. toLowerCase()
  2. 구두점 제거: /[.,!?;:'"()\[\]{}—\-]/g → ""
  3. 축약형 정규화: "don't"→"dont", "it's"→"its", "I'm"→"im" 등
     (축약형 리스트: 생성규칙이 관리 — 런타임은 단순 strip apostrophe)
  4. 앞뒤 공백 trim, 내부 연속 공백 → 단일 공백
```

대소문자 무구분, 구두점 무시. exact/cloze/keyword/dictation-diff 전부 이 normalize 적용.

### 4-6. tokenize() 규칙

```
tokenize(s: string): string[]
  normalize(s).split(" ").filter(t => t.length > 0)
```

## 6. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. english 전용 사항:

### 6-1. mount(container, ctx)

```
1. activity.front.modality 분기 → 모달별 UI 렌더링
   vocab/grammar : prompt + [options] or 단답 입력 + 제출 버튼
   reading       : passage 표시 → prompt + [options] or 단답 입력
   listening     : "재생" 버튼(SpeechSynthesis) + 받아쓰기 입력 + 제출
   writing       : prompt + textarea + 제출 (키 상태 표시)
   speaking      : prompt + 녹음 버튼(webkitSpeechRecognition or 텍스트 폴백) + 제출 (키 상태 표시)
2. onProgressRestored() 호출됐으면 저장된 last_answer 복원
3. listening: SpeechSynthesis 지원 여부 확인 → 미지원 시 "브라우저가 TTS를 지원하지 않습니다" 안내
              (audio_text를 텍스트로 표시하는 fallback은 금지 — 받아쓰기 무의미)
```

### 6-2. score(userAnswer)

```ts
// userAnswer: { text?: string; audio_blob?: Blob }
// 반환: Promise<ScoreResult>
```

- `grading.mode`에 따라 §4(정적) 또는 §5(BYO-key) 분기.
- 완료(pending 포함) 후 `emit('activity-completed', result)`.
