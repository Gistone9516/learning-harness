# 플러그인 계약 (프레임워크 코어 SoT)

> buildflow cycle-2 ②명세(opus-owned, SoT-first). **프레임워크의 최상위 계약** — 실습환경 플러그인 라이브러리.
> 충돌 시 우선순위: 본 문서 > `_인터페이스계약.md`(카드퀴즈 플러그인 내부 계약으로 격하) > 개별 규격.
> 원칙(메모리 [[learning-framework-design]]): 로컬 개인용·비서비스·무비용. 백엔드=로컬 프로세스, 유료=BYO-key(사용자 과금). 전부 통합(자체구현 X).

---

## 1. 개념 모델
- **공통셸(Common Shell)** = 환경 불문 1개. 네비·라우터·진도(localStorage)·대시보드 통합·키설정·콘텐츠 파이프라인 소유.
- **플러그인(실습환경)** = 셸에 꽂히는 어댑터. 각자 {문제규격·런타임 UI·채점기·진도기여}. 카드퀴즈=#1(지식 인출), 코딩=#2(실습) …
- 1 플러그인 = 1 `PluginInstance` + 1 `PluginManifest`.

## 2. PluginManifest (플러그인 자기선언)
| 필드 | 타입 | 비고 |
|---|---|---|
| `plugin_id` | string | 정규식 `^[a-z][a-z0-9-]{2,63}$` (card_id 동일). 전역 유일 |
| `label` | string | UI 표시명 |
| `version` | string | semver |
| `infra` | enum `static\|backend\|hybrid` | **자기 인프라 선언**(§8 분기) |
| `entry_url` | string? | backend/hybrid 시 로컬 엔드포인트(예: `http://localhost:PORT`). static이면 생략 |
| `byok` | object? | BYO-key 필요 시 `{keys: [{id, label, where: 'header'\|'query', help}]}`. 없으면 키 불요 |
| `capabilities` | string[] | `quiz\|code-exec\|language\|cloud-lab\|sheet` 중 |
| `scoring_mode` | enum `auto\|self\|external` | external=비동기 채점서버 |
| `activity_type` | string | 이 플러그인이 다루는 ActivitySpec.type |
| `progress_schema_version` | int | 진도 포맷 버전(마이그레이션) |

## 3. PluginInstance 인터페이스 (모든 플러그인 구현)
```
mount(container: HTMLElement, ctx: PluginContext): Promise<void>   // 라이프사이클 시작
unmount(): void
score(userAnswer): Promise<ScoreResult>     // scoring_mode=auto 필수, self/external은 stub
getProgressSnapshot(): PluginProgressSnapshot
onProgressRestored(snapshot): void
getDashboardContrib?(): DashboardContrib     // 선택(없으면 공통 기본집계)
```

## 4. 공통 데이터 타입
**ActivitySpec** (문제 공통 규격): `{activity_id: string, plugin_id, type, front: object, back?: object, weight: float[1,10], tags: {area, subarea, unit}, enabled: bool}`. `front`/`back` 내부구조는 type별 플러그인 자체 정의.

**ScoreResult**: `{verdict: 'correct'|'incorrect'|'skip'|'pending', score_raw?: float01(부분점수), feedback: object, grader_id: 'engine'|'pyodide'|'judge0'|'llm'|'self'|'external'}`. `pending`=비동기/self 대기.

**PluginContext** (셸→플러그인 주입): `{progressStore, emit(event), settings: {subject, dDayMode, ...}, getKey(keyId): string|null}`. `getKey`=BYO-key 조회(§6 키설정). ※`logger` 제거 — cycle-7에서 미사용 확인, 플러그인은 직접 `console.*` 사용.

**PluginEvent** (플러그인→셸, 단방향):
- `{type:'activity-completed', result: ScoreResult}`
- `{type:'session-done', total: number, correct: number, wrong: string[]}` — `wrong`은 오답 `card_id` 배열(공식 페이로드; 셸 `_renderSessionSummary`가 이 필드를 우선 소비)
- `{type:'navigation-request', target: string, conceptRef?: string}` — `conceptRef`=개념서 딥링크 타깃 ref_id(선택)

**PluginProgressSnapshot**: `{plugin_id, schema_version: int, activities: Record<activity_id, ActivityProgress>}`. `ActivityProgress`: `{box?, due_at?, cold_attempts, cold_correct, last_verdict, last_attempt_at?: number, plugin_extra?: object}`(plugin_extra=고유필드: 코딩 제출코드 등; `last_attempt_at`=app.js `leitnerTransition`이 이미 기록).

**DashboardContrib**: `{plugin_id, by_area: AreaStat[], weakness: WeaknessEntry[], pass_path: PassPathEntry[], completion: CompletionEntry[], extra_widgets?: object[]}` (필드형은 `_인터페이스계약.md §15` 재사용).

## 5. 공통셸 책임
- **레지스트리**: `window.PLUGIN_REGISTRY[plugin_id] = PluginInstance`. 부트 시 등록.
- **라우터**: hash 기반 `#<route>/<plugin_id>/<activity_id?>`(file:// 호환). route ∈ `learn|dashboard|settings`. 플러그인 전환 = unmount→mount.
- **진도**: `loadPluginProgress(plugin_id)`/`savePluginProgress(snapshot)` — 키 `clf:<plugin_id>:progress`. activity-completed 이벤트 수신→자동 저장.
- **대시보드**: 각 플러그인 `getDashboardContrib()` 합산→통합 위젯(retrieval/weakness/pass-path/completion) + 플러그인 extra_widgets 삽입.
- **키설정**(§6) · **네비**(plugin→area→unit 트리, REF 사이드바 재사용).

## 6. BYO-key 설정 (공통셸)
- 사용자가 settings 화면서 키 입력 → `localStorage['clf:keys:<keyId>']` 저장(로컬만, 외부전송 X).
- 플러그인은 `ctx.getKey(keyId)`로 조회. 키 없으면 플러그인이 **graceful**(무료/로컬 경로로 동작 or "키 필요" 안내, mount는 됨).
- manifest.byok가 키 목록·도움말 선언 → 셸이 설정 UI 자동 생성.

## 7. 런타임 와이어링 (유일 전역명·부트, SoT-first 핵심)
- **전역(유일)**: `window.PLUGIN_REGISTRY`(plugin_id→instance) · `window.ACTIVITIES[plugin_id]`(ActivitySpec[]) · `window.MANIFEST[subject]`(+ `.plugins: PluginManifest[]`) · `window.DECKS/SYNONYMS`(카드퀴즈 전용 잔존, `_인터페이스계약 §16`).
- **컨테이너**: 셸이 `<main id="plugin-host">` 1개 제공. 활성 플러그인이 `mount(host, ctx)`로 자기 UI 주입. 전환 시 이전 unmount.
- **부트 순서**: index.html이 [CDN(폰트/CodeMirror/Pyodide/KaTeX/marked) → 생성물 scripts(synonyms→manifest→decks→activities) → 플러그인 파일(manifest+plugin.js) → shell.js] 로드 → **`window.SHELL.init({subject})`** 호출. shell.init = MANIFEST[subject].plugins 순회 등록 → 라우트 복원 → 활성 플러그인 mount.
- **정적 데이터**: 플러그인 데이터는 `window.ACTIVITIES[plugin_id]` 전역 등록(file:// fetch 회피). backend면 `entry_url`서 fetch.

## 8. infra 분기 (정적/백엔드/하이브리드)
- **static**: ACTIVITIES 전역 + JS 순수 채점 + localStorage. entry_url 불요. (카드퀴즈·코딩 Pyodide·영어 일부)
- **backend**: `entry_url` 헬스체크 → 실패 시 `mountOfflineFallback`(disabled+안내). score()=fetch(entry_url+'/grade'). 진도는 셸 localStorage(서버 아님). (코딩 다언어 Judge0·엑셀 PySheetGrader)
- **hybrid**: 문제=정적, 채점=entry_url 또는 BYO-key API. (AWS·영어 발음)
- 셸 부트: `infra==='static'` → ACTIVITIES 확인 / else → entry_url `/health` 3s 타임아웃 → 실패시 fallback.

## 9. card-quiz = 플러그인 #1 재배치 (엔진 무변경)
- 신규 얇은 래퍼(~100줄): `엔진/plugins/card-quiz/{manifest.js, plugin.js}`. 기존 `app.js`(window.APP.*)·엔진코어·deck.js **수정 0**.
- `plugin.mount()` = 기존 퀴즈 UI를 host에 주입 + `window.APP.init({subject, onCard, onEmpty→emit('session-done'), onError})`.
- `score()` = `window.APP.score()` 위임 / `getProgressSnapshot/onProgressRestored` = 기존 loadProgress/saveProgress 래핑 / `getDashboardContrib` = 기존 getDashboardData 래핑.
- `generate.py`: 기존 deck.js 유지 + `window.ACTIVITIES['card-quiz']` 등록 블록 추가(deck.cards→ActivitySpec 매핑) + `MANIFEST[subject].plugins`에 card-quiz 추가.
- 기존 localStorage 키(`clf:comp1:progress`) 그대로 — 진도 데이터 보존, 마이그레이션 없음.

## 10. _인터페이스계약.md 의 지위
기존 §1~16(card_id·deck·normalize·getDashboardData·와이어링)은 **card-quiz 플러그인의 내부 계약**으로 격하. 프레임워크 레벨에선 본 문서가 우선. 단 카드퀴즈가 쓰는 `window.DECKS/SYNONYMS/getDashboardData` 등은 §16과 정합 유지.
