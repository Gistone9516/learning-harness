## 1. 셸 구조 (`봇/`)

```
봇/
├ main.py      ← 진입(.env·마운트·부트·Client.run)
├ boot.py      ← 콘텐츠 로드·검증·컴파일·레지스트리·진도(harness store + engine.migrate)
├ context.py   ← Ctx
├ session.py   ← 세션 상태·루프
├ dispatch.py  ← 라우팅 테이블 + capability_id->handler
├ handlers/    ← 능력별 핸들러(학습타입규격 capability_id 1:1)
├ commands.py  ← 슬래시·매직워드
├ ai.py        ← _invoke 어댑터(AI모드골격 복제, 참고 ~140줄)
└ harness/     ← 복사 카탈로그(루트 harness/ 복사)
```

## 2. 부트 (SoT §3 구현)

1. `.env` 필수 4키 검증(누락→구동 거부). 선택 키 USER_LANG(기본 ko)·MOUNT.
2. 마운트 = `argv[1] > .env MOUNT > cwd`.
3. `boot.load(mount)`: manifest→deck→config 로드, 주입 검증(`ContentInjectionError`→부트 차단). 컴파일:
   - 정규화 프로파일, synonyms 역인덱스(`dict[str,str]`), **실효 grade_mode 맵**(`card_id → grade_mode`, CardDef.grade_mode에 `SubjectConfig.scoring_overrides` 적용), **pass_targets**(엔진 §5용).
4. 진도: harness store로 `<mount>/_상태/progress-<deck>.json` 읽기 → `engine.migrate(raw)` → ProgressStore. 파일 없음/손상=빈 스토어 폴백+`.bak`.
5. 능력 레지스트리(config `capabilities.enabled`, 미지정→core 전체) + 의존 검증(누락=경고+비활성). layer 4는 상시 와이어.
6. discord.py Client(intents) → `on_ready`: persistviews 재등록(§7) + perm_preflight + presence.

## 3. 핸들러 디스패치 (라우팅 + 테이블, [OPEN-B] 해소)

**카드 → capability_id 라우팅(우선순위, 위→아래 첫 매치):**
| 조건 | capability_id |
|---|---|
| `grade_mode == self`(모든 type) | `recall_self` |
| `type == cloze` | `cloze_modal` |
| `type == recall_seq` | `seq_modal` |
| `type == judge` | `mcq_buttons`(options≤5) / `mcq_select`(>5) |
| `type ∈ {func, proc}` | `short_modal` |
- config가 대체 렌더러를 켜면 우선(예: `reaction_quick`→객관식 대신 이모지, `quiz_poll`→투표). 명시 opt-in.

```python
HANDLERS: dict[str, Handler]            # capability_id -> 핸들러
Handler = Callable[[Ctx, CardDef], Awaitable[HandlerResult]]   # HandlerResult = SoT §1
```
- layer 4(gating/event/heartbeat/coalesce/perm/presence/scaffold/dm)는 HANDLERS 아님 — 부트/미들웨어 상시 와이어.

**Ctx(핸들러 주입):**
```python
@dataclass
class Ctx:
    channel: Any; user_id: int
    store: ProgressStore; deck: DeckData
    synonyms: dict[str, str]                 # 역인덱스(SoT §1 conform)
    grade_mode_of: Callable[[str], ScoreMode]  # 실효 grade_mode(scoring_overrides 반영)
    leitner_cfg: LeitnerConfig | None
    ai_model: str | None; ai_effort: Effort   # SoT §1 Effort(Literal). invoke에 직접 전달
    sid: str | None
    session: "Session"
    async def emit(self, card_id: str, verdict: Verdict, now: int) -> None: ...  # 분류+전이+저장 공통
```

## 4. 세션 루프 (전 루프 계약)

세션 상태(메모리 휘발): `seen_card_ids: set[str]`, `queue`, `requeue`, `idx`.
1. 시작: `build_queue(cards, progress, now, opts)` → `queue`. `opts.weight_overrides`=사이드카 로드(있으면).
2. **카드 제시 + 분류**: 다음 card_id의 능력 핸들러(라우팅 §3) 호출 **직전**, `attempt_kind = "warm" if card_id in seen_card_ids else "cold"` 판정 **후 즉시 `seen_card_ids.add(card_id)`**(엔진계약 §1.2 정렬 — 채점 결과와 무관, 제시 시점 확정).
3. 응답·채점: 핸들러가 실효 grade_mode(`ctx.grade_mode_of(card_id)`)로 `score()`(또는 self/AI) → `HandlerResult`(verdict·requeue).
4. **전이+저장**: `ctx.emit(card_id, verdict, now)` = `leitner_transition(state, attempt_kind, verdict, now, cfg, dday)` → harness store로 `save`(deck별). skip=verdict "skip"(score 미호출).
5. **세션내 재큐**: `HandlerResult.requeue`(오답)면 `requeue`에 추가. `requeue` 먼저 소진 후 `queue`. warm 결과는 box 무관.
6. **종료**: `queue`·`requeue` 소진 → 요약(§아래) + 진단 갱신.

**종료 요약(비AI 기본):** 임베드/카드로 `총 시도·정답률·box 이동 수·다음 due 정보`(digest store 갱신) + 맺음 @멘션. `ai_session_summary` 활성 시 AI 일지로 대체(학습타입규격 layer3).

**D-day 활성화:** `/study --dday` 또는 `/dday` 토글로 `opts.dday_mode=True`(또는 config `capabilities.dday`). 엔진은 판정 안 함(봇이 주입, 엔진계약 §2.3).

## 7. harness 와이어링
- 복사 = `봇/harness/`(루트 harness/ 복사, discord-harness 절차). 플레이스홀더(CHANNEL/ALLOWED_USER_ID) 채움.
- **persistviews 재등록(필수)**: custom_id = `srs:<verdict>:<card_id>`. `on_ready`에서 `add_view(PersistentLearnView())` 1회 → 재시작 후 버튼 클릭이 `emit`으로. **봇 재시작 후 클릭은 seen 휘발이라 cold로 처리됨**(재시작=세션 초기화, SoT §2 정합 — 의도된 동작).
- 사이드카(SoT §2.1): confidence·read_pos·elaboration·alert_sent·adaptive_weight = harness store로 봇이 관리(엔진 무관).
- live/* 코얼레싱은 공통 `coalesce_base`(ratelimit) 위. 하드 제약: allowlist·비밀값·≤1편집/초·토큰당 1봇·권한 스코프.
