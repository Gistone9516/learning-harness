## 0. 입력 인터페이스

```ts
// 깊이스무고개.md STEP 2 출력 (플러그인계약 §4 confirm)
interface LearningProfile {
  subject:           string;                      // 예: "엑셀", "스프레드시트 실무"
  subject_structure: SubjectStructure;            // STEP0 웹조사 결과
  estimated_level:   string;                      // "입문" | "초급" | "중급" | "고급"
  goal:              "exam" | "practical" | "literacy" | "skill";
  scope:             string[];                    // 대상 토픽 목록
  depth_ratio:       { theory: number; practice: number };  // 합=1
  volume:            string;                      // 예: "20문항", "30분 분량"
  target_plugins:    string[];                    // "excel" 포함 확인 필수
}
```

## 1. 생성 전 웹서치 의무 (사실 그라운딩)

> 이 규칙을 따르는 모델(Claude)은 임베딩 지식으로 엑셀 함수·수식 사실을 단정하지 않는다.
> **모든 함수 사실(인수 순서, 반환 타입, 버전)은 생성 전 웹서치로 확인 후 사용.**

### 1-1. 서치 대상

| 조회 항목 | 서치 키워드 예시 |
|-----------|-----------------|
| 함수 인수·반환 타입 | `"Excel SUM function syntax arguments"` |
| 함수 호환성(버전) | `"Excel XLOOKUP compatibility 2019 2021 365"` |
| 수식 edge case | `"Excel VLOOKUP exact match vs approximate"` |
| Univer 지원 함수 여부 | `"Univer spreadsheet supported functions list"` |

### 1-2. 임베딩 단정 금지 사례

```
❌ 임베딩: "VLOOKUP 4번째 인수 TRUE=근사"
✓  서치 후: docs.microsoft.com 또는 support.microsoft.com 확인 후 작성
```

### 1-3. 불확실 함수 처리

- Univer 지원 여부 미확인 함수(XLOOKUP, LAMBDA, LET 등 최신함수) → 파킹 목록(§7)에 기재, 과제에서 제외.
- MVP 검증된 안전 함수 목록(§8)에서 우선 선택.

## 2. LearningProfile → 생성 파라미터 매핑

```
N_total = volume에서 숫자 추출 (없으면 기본 15)
N_practice = round(N_total × depth_ratio.practice)   // 최소 1
N_theory_card = N_total - N_practice                  // card-quiz 소관, excel 무관

difficulty_mix(estimated_level):
  "입문" → { easy: 0.7, medium: 0.3, hard: 0.0 }
  "초급" → { easy: 0.4, medium: 0.5, hard: 0.1 }
  "중급" → { easy: 0.2, medium: 0.5, hard: 0.3 }
  "고급" → { easy: 0.0, medium: 0.4, hard: 0.6 }

topic_distribution = scope를 N_practice개에 균등 분배(round-robin)
```

## 3. ActivitySpec 생성 템플릿 (sheet-task)

아래 템플릿을 각 과제마다 빈칸 채움식으로 적용. **빈칸 없이 전부 채워야 함**.

```jsonc
{
  "activity_id": "xls-<unit_slug>-<3자리 순번>",   // 예: xls-sum-001
  "plugin_id":   "excel",
  "type":        "sheet-task",
  "weight":      <1~10, 난이도 반영>,
  "tags": {
    "area":    "<scope 항목>",
    "subarea": "<함수 범주>",
    "unit":    "<핵심 함수/기능>"
  },
  "enabled": true,
  "front": {
    "prompt":       "<§3-1 지문 규칙 준수>",
    "initial_grid": <§3-2 initial_grid 규칙 준수>,
    "target_cells": <§3-3 target_cells 규칙 준수>
  },
  "back": {
    "solution":    "<모범 수식 또는 값>",
    "explanation": "<간결한 풀이 근거>"
  },
  "grading": {
    "expected": <§3-4 expected 규칙 준수>,
    "compare":  "value"
  }
}
```

### 3-1. prompt 작성 규칙

| 규칙 | 세부 |
|------|------|
| P1 | 10자 이상 (스키마 minLength:10) |
| P2 | 어느 셀에 무엇을 입력/작성하라는 지시 명시. 예: "B6 셀에 SUM 함수로 B2:B5 합계를 구하시오." |
| P3 | initial_grid에 이미 있는 데이터 구조 설명 포함(학습자 컨텍스트 제공) |
| P4 | 함수명 대문자 표기(Excel 관례). 예: SUM, VLOOKUP, IF |
| P5 | 모호한 지시 금지. "적절한 함수" → "SUM 함수" 처럼 구체 지정 |

### 3-2. initial_grid 작성 규칙

| 규칙 | 세부 |
|------|------|
| G1 | 2D 배열(행 우선). `initial_grid[rowIdx][colIdx]` |
| G2 | 셀 주소 A1 표기 → 인덱스 변환: 열 A=0, B=1, … / 행 1=0, 2=1, … |
| G3 | 빈 셀 = `""` (null/undefined 금지) |
| G4 | 숫자 데이터는 JSON number 타입(`1`, `2.5`). 문자는 string(`"항목"`) |
| G5 | 1행은 헤더(레이블) 권장(학습자 가독성) |
| G6 | target_cells의 최대 행+2 이상의 행을 포함할 것(여백 확보) |
| G7 | initial_grid 크기: 최소 2행×2열 / 최대 20행×10열 (MVP UI 한계) |
| G8 | 과제 과정에서 학습자가 채울 셀 = `""` 로 초기화 |
| G9 | 수식 초기값 금지 — initial_grid에 `=SUM(...)` 등 수식 문자열 넣지 말 것. 수식은 학습자가 입력. |

**initial_grid → cellData 변환 (구현자 참고)**

```
IWorkbookData.sheets.sheet1.cellData =
  initial_grid[row][col] !== "" →
    숫자: { v: value, t: 2 }
    문자: { v: value, t: 1 }
  initial_grid[row][col] === "" → 키 자체 생략(빈 셀)
```

### 3-3. target_cells 작성 규칙

| 규칙 | 세부 |
|------|------|
| T1 | A1 표기. 정규식 `^[A-Z]{1,3}[1-9][0-9]*$` 준수 |
| T2 | initial_grid에서 `""` 인 셀만 target(학습자가 채워야 할 셀) |
| T3 | MVP: 1~3개 권장. 5개 초과 금지(채점 피드백 가독성) |
| T4 | grading.expected와 동일 순서·동일 길이 필수 |
| T5 | 범위 주소 금지(예: "B2:B6" ❌ → "B6" ✓) |

### 3-4. expected / grading 작성 규칙

| 규칙 | 세부 |
|------|------|
| E1 | `cell` = target_cells의 원소와 정확히 일치 |
| E2 | `value` = 수식 계산 결과값(정적). 생성자가 수작업으로 계산 또는 실제 Excel/Sheets에서 검증 후 기재 |
| E3 | 정수 결과 → JSON number. 예: `15` (string `"15"` 금지) |
| E4 | 소수 결과 → 반올림 자릿수 명시. 예: AVERAGE(1,2,3)=2 → `2` / AVERAGE(1,2)=1.5 → `1.5` |
| E5 | 문자열 결과(IF, CONCATENATE 등) → string. 예: `"합격"` |
| E6 | 부동소수 오차 회피: 과제 수식에 ROUND(expr, n) 적용 후 expected도 반올림값으로 기재 |
| E7 | `compare` = `"value"` 고정(MVP). 수식 동치 채점은 v2(파킹) |

## 4. activity_id 명명 규칙

```
형식: xls-<unit_slug>-<NNN>
- "xls-" 고정 접두사
- <unit_slug>: tags.unit을 소문자·영문·하이픈으로 변환. 예: "SUM" → "sum", "VLOOKUP" → "vlookup", "조건부합산" → "sumif"
- <NNN>: 001부터 순번(동일 unit 내)
정규식: ^xls-[a-z][a-z0-9-]{1,20}-[0-9]{3}$
```

## 5. weight 배정 기준

| 난이도 | weight 범위 | 기준 |
|--------|-------------|------|
| easy   | 1~2 | 단일 함수, 단순 범위, 결과 자명 |
| medium | 3~5 | 함수 중첩 1단계, 조건 1개, 범위 추론 필요 |
| hard   | 6~8 | 함수 중첩 2단계 이상, 다중 조건, 배열수식 개념 |

## 6. 생성 예시 (LearningProfile → ActivitySpec 세트)

**입력 LearningProfile (예시)**:
```json
{
  "subject": "엑셀 실무",
  "estimated_level": "초급",
  "scope": ["집계함수", "논리함수", "조회함수"],
  "depth_ratio": { "theory": 0.3, "practice": 0.7 },
  "volume": "10문항"
}
```

**생성 파라미터**:
- N_total = 10 → N_practice = 7 (depth 0.7)
- difficulty_mix: easy 40%, medium 50%, hard 10% → 3 easy, 3~4 medium, 0~1 hard
- topic_distribution: 집계함수 3개, 논리함수 2개, 조회함수 2개 (round-robin)

**생성 예시 1 — SUM (easy)**:

```json
{
  "activity_id": "xls-sum-001",
  "plugin_id": "excel",
  "type": "sheet-task",
  "weight": 2,
  "tags": { "area": "집계함수", "subarea": "수학함수", "unit": "SUM" },
  "enabled": true,
  "front": {
    "prompt": "B2:B6 셀에 월별 판매량이 입력되어 있습니다. B7 셀에 SUM 함수를 사용하여 합계를 구하시오.",
    "initial_grid": [
      ["월",   "판매량"],
      ["1월",  120],
      ["2월",  95],
      ["3월",  140],
      ["4월",  88],
      ["5월",  112],
      ["합계", ""]
    ],
    "target_cells": ["B7"]
  },
  "back": {
    "solution": "=SUM(B2:B6)",
    "explanation": "SUM(범위)은 범위 내 모든 숫자를 더함. B2:B6 = 120+95+140+88+112 = 555."
  },
  "grading": {
    "expected": [{ "cell": "B7", "value": 555 }],
    "compare": "value"
  }
}
```

**생성 예시 2 — IF (medium)**:

```json
{
  "activity_id": "xls-if-001",
  "plugin_id": "excel",
  "type": "sheet-task",
  "weight": 4,
  "tags": { "area": "논리함수", "subarea": "조건함수", "unit": "IF" },
  "enabled": true,
  "front": {
    "prompt": "C2:C5 셀에 점수가 있습니다. D2:D5 셀 각각에 IF 함수를 사용하여 점수가 60 이상이면 \"합격\", 미만이면 \"불합격\"을 표시하시오. D2 셀만 채점합니다.",
    "initial_grid": [
      ["이름", "점수", "결과"],
      ["김민준", 85, ""],
      ["이서연", 52, ""],
      ["박지호", 73, ""],
      ["최수아", 40, ""]
    ],
    "target_cells": ["C2"]
  },
  "back": {
    "solution": "=IF(B2>=60,\"합격\",\"불합격\")",
    "explanation": "IF(조건, 참값, 거짓값). B2=85 ≥ 60 이므로 \"합격\" 반환."
  },
  "grading": {
    "expected": [{ "cell": "C2", "value": "합격" }],
    "compare": "value"
  }
}
```

> 주: 예시 2는 D2:D5를 설명하나 MVP target_cells=1개로 단순화. 실제 생성 시 D2~D5 전부 채점하려면 target_cells를 ["D2","D3","D4","D5"]로, expected도 4개로 작성.

## 7. 파킹 (생성 금지) 함수 목록 (v1)

Univer 지원 여부 미확인 또는 수식 동치 채점 필요 — v1 생성 과제에서 제외.

| 함수 | 이유 |
|------|------|
| XLOOKUP | Excel 365/2021+, Univer 지원 미확인 |
| LAMBDA / LET | 최신함수, Univer 지원 미확인 |
| FILTER / SORT / UNIQUE | 배열 스필(Spill) 함수, Univer 미지원 가능 |
| IFERROR(중첩) | compare=value만으로 수식 동치 판별 불가 에지케이스 |
| INDIRECT | 동적 참조, 계산 완료 타이밍 이슈 가능 |
| ARRAYFORMULA | Sheets 전용, Excel/Univer 비호환 |

## 9. 기계검증 체크리스트 (생성물)

생성된 모든 ActivitySpec에 대해 아래 전부 통과 확인:

| #   | 항목 | 검증 수단 |
|-----|------|-----------|
| G1  | JSON Schema §2-1(런타임규격) validate 통과 | ajv / jsonschema |
| G2  | `activity_id` 정규식 `^xls-[a-z][a-z0-9-]{1,20}-[0-9]{3}$` | regex test |
| G3  | `front.target_cells.length === grading.expected.length` | assert |
| G4  | `grading.expected[].cell` ∈ `front.target_cells` | set inclusion |
| G5  | `grading.compare === "value"` | strict eq |
| G6  | `initial_grid[row][col]` 타입 = string 또는 number만 | typeof check |
| G7  | `initial_grid` 내 수식 문자열 없음 (`=` 로 시작하는 값 금지) | regex `^=` |
| G8  | target_cells 정규식 `^[A-Z]{1,3}[1-9][0-9]*$` | regex test |
| G9  | target_cells의 각 셀이 initial_grid에서 `""` 인 위치와 일치 | 좌표 역산 |
| G10 | `front.prompt.length >= 10` | length check |
| G11 | `weight` ∈ [1,10] 정수 | range check |
| G12 | `tags.area` ∈ `scope` (LearningProfile) | set inclusion |
| G13 | `grading.expected[].value` 타입 = number 또는 string | typeof check |
| G14 | `activity_id` 전체 집합 내 중복 없음 | unique check |
| G15 | 파킹 함수(§7) `back.solution`에 포함 안 됨 | regex keyword check |

## 10. 생성 절차 요약 (모델 실행 순서)

```
STEP A — 웹서치 (§1 의무)
  1. 과제에 쓸 함수 각각 → 인수·반환타입·버전 서치 확인
  2. Univer 지원 여부 미확인 함수 → §7 파킹 처리
  3. expected.value 수작업 계산 → 실제 Excel/Sheets 검증 권장

STEP B — 파라미터 산출 (§2)
  1. volume → N_practice 계산
  2. estimated_level → difficulty_mix
  3. scope → topic_distribution (round-robin)

STEP C — 과제 생성 (§3 템플릿 전부 준수)
  for i in range(N_practice):
    1. 해당 토픽·난이도 선택
    2. prompt 작성 (P1~P5)
    3. initial_grid 설계 (G1~G9)
    4. target_cells 결정 (T1~T5)
    5. expected.value 계산·검증 (E1~E7)
    6. back.solution/explanation 작성

STEP D — 기계검증 (§9)
  생성된 모든 spec에 G1~G15 전부 pass 확인 → 실패 항목 재생성

STEP E — window.ACTIVITIES["excel"] 등록
  window.ACTIVITIES["excel"] = [spec1, spec2, ...]
```
