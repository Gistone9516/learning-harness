# 엑셀(excel) 기능 백로그 (cycle-3)

> 방향성 SoT(`_방향성.md`) 준수 — 충돌 시 방향성 > 본 문서. 7대 선별 원칙 전부 통과 기준. 코어 SoT는 `플러그인계약.md`.

## core

| 기능 | 동작 | infra | 공수 | 검토 반영·근거 |
|------|------|-------|------|----------------|
| 해설 패널 (back.solution/explanation) | 채점 후 또는 '해설 보기' 클릭 시 `back.solution`·`back.explanation` 접이식 패널 렌더. 현행 `_showResult()` back 필드 미렌더 수정 | static | S | 원칙2 — 규격 정의+데이터 존재+미구현. 학습루프 완성 핵심. 이데이션+검토 공통 core |
| 값 비교 float 허용오차 | `compareValue`에 epsilon 파라미터 추가. `grading.compare='value-tol'` 또는 `grading.tolerance` 필드 지원. AVERAGE 등 부동소수 오판 방지 | static | S | 원칙2 — 런타임규격 §4-2 exact-number 비교 한계. 컴활 1급 빈출 함수 오채점 방지 |
| 다중셀 결과표 피드백 | `target_cells` 3개 이상 과제에서 각 셀 pass/fail 행별 테이블 표시 + `score_raw` 진척바. 현행 `first_fail` 1건 → 전체 셀 목록 렌더 | static | S | C1 계약대기 — `ScoreResult.feedback` 다건 확장 승인 후 반영. 데이터(`passed/total`)는 이미 존재, 렌더만 미구현 |
| calculationEnd 타임아웃 방어 | `scoreAfterCalc` 내 5초 타임아웃 추가. 만료 시 '수식 계산 실패, 수식을 확인하세요' 안내 렌더. `state===3` 미발화 시 Promise 영원히 pending 상태 방어 | static | S | 검토 누락 발굴 — 런타임규격 §4-3 `calculationEnd` 무한 pending UX 중단 방어. static S규모 |
| 생성규칙 예시 오기 수정 + validate 자동화 | 생성규칙.md §6 예시 2 `target_cells: ["C2"]` → `["D2","D3","D4","D5"]` 수정. `front.target_cells` ↔ `grading.expected` 정합 기계검증(V5·V7) 자동 실행 스크립트 추가 | static | S | 검토 누락 발굴 — 오기 그대로 출제 시 오채점 직결. 방향성 원칙6 하드플로어 계열 |

> **다중셀 결과표**: C1(`ScoreResult.feedback` 케이스 배열 확장) 사용자 승인 전까지 UI만 선행 구현(passed/total + 셀별 상태), 계약 변경 부분은 승인 후 반영.

| 기능 | 동작 | infra | 공수 | 검토 반영·근거 |
|------|------|-------|------|----------------|
| 진도 셀 스냅샷 저장/복원 | `getProgressSnapshot plugin_extra`에 `univerAPI.getActiveWorkbook().getSnapshot()` 직렬화 저장. `onProgressRestored` 시 `createWorkbook`으로 재주입 | static | M | core→mid 하향 — `getSnapshot()` Univer 0.x 미검증 + 실제 마찰(수식 재입력 ≤30초) 낮음. 브라우저 실측 선행 권고 |

| 약점 영역 재도전 필터 | 네비게이션 배지에 weight 기반 난이도 아이콘. `getDashboardContrib weakness` 데이터 재활용해 '약점 영역만 보기' 필터 버튼 | static | S | 이데이션 mid 유지. `weakness` 배열 이미 계산됨 — 렌더만 추가 |

## 파킹 (실측 선행 포함)

| 기능 | 파킹 사유 | 재개 조건 |
|------|-----------|-----------|
| XLOOKUP / 동적배열 함수 커버리지 | Univer UMD에서 XLOOKUP·FILTER·SORT·UNIQUE 실제 동작 미검증. 사용자 브라우저 실측 선행 필요. FILTER/SORT/UNIQUE는 배열 스필 → 단일셀 getValue 방식과 구조 충돌 | 브라우저 실측 확인 후 |
| ko-KR 로케일 | `@univerjs/preset-sheets-core` ko-KR UMD 파일 존재 여부 미확인. 런타임규격 §3-1 en-US 고정. 자체 빌드는 원칙5 위반 | unpkg 실측 존재 확인 후 |
| 수식 동치 채점 재검토 | PySheetGrader HTTP API 출현 또는 동등 차용 도구 등장 시 재평가 대상 | 차용 가능 HTTP API 등장 시 |
