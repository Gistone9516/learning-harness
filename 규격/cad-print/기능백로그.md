## 우선순위 등급 정의

| 등급 | 의미 |
|---|---|
| **core** | MVP에 반드시 필요. 없으면 학습루프(문제→입력→채점→해설→재시도) 불완전. |
| **mid** | 학습 효과 있음. core 완성 후 다음 사이클. |
| **low** | 있으면 좋음. 빼기 미학 우선 — 근거 없으면 제외. |
| **파킹** | 사용자 재평가 전 구현 금지. 이유 명시 필수. |

## core — MVP 필수

| # | 기능 | 설명 | 근거 |
|---|---|---|---|
| C1 | **ActivitySpec 데이터 4~6개** | cad-box, cad-cylinder, cad-union, cad-subtract + manifold-check. 채점 인프라보다 콘텐츠 먼저. | 방향성 원칙1 |
| C2 | **JSCAD 코드 에디터** | `<textarea>` (monospace) + starter_code 초기값 주입. CodeMirror는 선택. 비전공자 빈칸채우기 형태. | 런타임규격 §4-1 |
| C3 | **"실행 미리보기" 버튼** | 코드 실행 → three.js 3D 렌더 갱신. 채점 없음. 즉각 시각 피드백. | 방향성 원칙1·2 |
| C4 | **JSCAD geometry 3D 렌더** | practice-runner + three.js BufferGeometry. 법선 재질(MeshNormalMaterial) MVP. | 실습러너계약 §6 |
| C5 | **"제출·채점" 버튼 + score()** | 런타임규격 §3-2 채점 절차 전체. manifold→부피→바운딩박스 판정. | 런타임규격 §3 |
| C6 | **매니폴드 graceful 판정** | measureVolume NaN/≤0 → verdict=incorrect, feedback.manifold=false, hint 진단. throw 금지. | 실습러너계약 V7 |
| C7 | **진단형 feedback 렌더** | volume_user, volume_err_pct, bbox_diff, manifold, hint를 화면에 표시. 이진 "맞음/틀림" 금지. | 런타임규격 §3-7·3-8 |
| C8 | **해설 패널 (back.why + back.explanation)** | 제출 후 또는 버튼 클릭 시 해설 표시. 정답 직행 방지: 오답 첫 제출에는 숨기기. | 방향성 원칙2 |
| C9 | **진도 저장 (getProgressSnapshot)** | last_code, last_score_raw, manifold_ok. localStorage. 재마운트 시 코드 복원. | 플러그인계약 §5 |
| C10 | **타임아웃 방어 (3초)** | Promise.race + 3초. 초과 시 verdict=incorrect, hint='실행 시간 초과'. throw 금지. | 런타임규격 §3-3 |
| C11 | **target_spec + hints 사이드 표시** | front.target_spec·hints[] 렌더. back.grading 수치는 비노출 (힌트만). | 방향성 원칙4(인출강도) |
| C12 | **practice-runner 표준 패턴 준수** | mount/unmount/dispose 실습러너계약 §4·§5·§6 패턴 그대로. ctx.emit 포함. | 실습러너계약 §4·V5·V6 |

## mid — core 완성 후 다음 사이클

| # | 기능 | 설명 | 선행 조건 |
|---|---|---|---|
| M1 | **실시간 3D 미리보기 (타이핑 중 자동갱신)** | 디바운스(500ms) 후 자동 컴파일·렌더. 성능 측정 후 도입. | C3·C4 완성 |
| M2 | **부분점수 진행바 UI** | score_raw(0/0.5/1)를 시각적 진행바로 표시. | C5·C7 완성 |
| M3 | **치수 오버레이 (렌더 위 바운딩박스 표시)** | three.js BoxHelper로 bbox 시각화. 오차 직관 인지. | C4 완성 |
| M4 | **솔루션 코드 diff 보기** | 사용자 코드 vs solution_code 라인 diff. 단, 자동완성 아님 — 제출 후에만. | C8 완성 |
| M5 | **문제 네비게이터** | activities 2개 이상일 때 nav 버튼. _applyActivity 패턴 (coding 플러그인 미러). | C1 콘텐츠 충분 |
| M6 | **코드 재시도 유도 UX** | 오답 시 힌트 순차 공개 (hints[1]→hints[2] …). 정답 전까지 solution 숨김. | C11 완성 |

## 파킹 (재평가 전 구현 금지)

| # | 기능 | 파킹 사유 | 재평가 조건 |
