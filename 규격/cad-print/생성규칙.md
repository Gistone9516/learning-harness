## 0. 전제 — 비용 원칙

- **비전공자 MVP 분량**: cad3dp subject 전체 실습 activity 4~6개. 과하게 만들지 말 것.
- **콘텐츠 우선**: solution_code + grading 기준치(부피·바운딩박스) 먼저 확정 → 그 다음 prompt 작성.
  역순(prompt 먼저, 기준치 나중) 금지.
- **solution_code 검증 의무**: 작성한 solution_code는 반드시 실제 JSCAD 실행으로 measureVolume·measureBoundingBox 검증 후 grading 수치 확정. 추정치 금지.
- **스캐폴딩 우선**: 스타터 코드에 `require('@jscad/modeling')` 심 사용법 + 함수 시그니처 + 빈칸(`// 여기에 작성`) 포함. 백지 시작 금지(비전공자 진입장벽 제거).
- **선수지식 명시 의무**: 각 activity의 `front.hints[]` 첫 원소는 "선수지식: ..." 형태로 필요 개념 명시.

## 2. ActivitySpec 생성 템플릿

### 2-1. 공통 필드 규칙

```
activity_id  = "cad-{형상약어}-{3자리 시퀀스}"  (예: cad-box-001)
plugin_id    = "cad-print"
type         = "cad-model"
weight       = 입문→2 / 기초→4 / 중급→6  (default 3)
tags.area    = "3D모델링"
tags.subarea = 단원 분류 문자열 (예: "기초형상", "파라메트릭", "CSG", "매니폴드")
tags.unit    = 단원 ID (§1-1)
enabled      = true
```

### 2-2. front 작성 규칙

**front.prompt 템플릿**:
```
⟨대상 형상⟩을 JSCAD 코드로 만드세요.
치수: ⟨구체적 치수 조건⟩.
(⟨비전공자 친화 부연⟩)
```
- 최대 3문장. 반드시 치수·형상 조건 명시.
- "적절히"/"올바르게"/"잘" 금지 → 수치 조건 대체.
- 한국어.

**front.starter_code 규칙**:

| 난이도 | 스캐폴딩 수준 |
|---|---|
| 입문 | require 심 + 단일 프리미티브 함수 이름 힌트 + `// 여기에 작성` |
| 기초 | require 심 + 2개 이상 프리미티브 + 불리언 연산 이름 힌트 |
| 중급 | require 심 + 함수 구조 + 파라미터 변수 선언 |

템플릿 기본:
```js
const { ⟨사용할 함수명들⟩ } = require('@jscad/modeling').primitives;
// 불리언 필요 시: const { union, subtract } = require('@jscad/modeling').booleans;

function main() {
  // 여기에 작성
}

module.exports = { main };
```

규칙:
- starter_code는 실행 가능한 JS여야 함 (SyntaxError 금지).
- `module.exports = { main }` 반드시 포함.
- 주석은 한국어.

**front.hints 규칙**:
- `hints[0]` = `"선수지식: ⟨필요 개념⟩"` (필수, 난이도 급경사 방지).
- `hints[1]` 이후 = API 사용법 힌트 1~2개 (선택).
- 정답 직접 제공 금지 (인출 강도 유지, 방향성.md 원칙4).

예시:
```json
"hints": [
  "선수지식: 3D 좌표계(x,y,z)와 mm 단위 이해 필요.",
  "cuboid({ size: [x, y, z] }) — size 배열 순서는 x, y, z 입니다."
]
```

### 2-3. back 작성 규칙

**back.solution_code**:
- 완전히 실행 가능한 JSCAD 코드.
- `module.exports = { main }` 포함.
- `main()` 반환값 = JSCAD geometry (단일 또는 배열).
- **검증 의무**: solution_code 실행 → measureVolume·measureBoundingBox 출력값 → grading 수치 확정.

**back.grading 채우기 절차**:
```
1. solution_code 실행 (로컬 Node.js + @jscad/modeling 또는 브라우저 콘솔)
2. vol    = jscadModeling.measurements.measureVolume(main())
3. bbox   = jscadModeling.measurements.measureBoundingBox(main())
4. 기록:
   volume_target        = round(vol, 2)
   volume_tolerance_pct = 5        // 기본값, 교수 검토 후 조정 가능
   bbox_target          = { x: bbox[1][0]-bbox[0][0], y: bbox[1][1]-bbox[0][1], z: bbox[1][2]-bbox[0][2] }
   bbox_tolerance       = 0.5      // 기본값 (mm), 조정 가능
```

**back.why**:
- 1~2문장. "왜 이 접근이 맞는가" (개념 연결).
- 예: `"cuboid size 배열이 [x, y, z] 순이며, 직육면체 부피 = x×y×z = 4000mm³."`

**back.explanation**:
- 2~3문장. 핵심 API + 비전공자 언어.
- 예: `"cuboid()는 직육면체 형상 프리미티브. size [20,20,10]이 x,y,z 각 치수(mm). JSCAD는 원점 중심 배치."`

## 3. 검증 체크리스트 (생성물 인수 기준)

| # | 항목 | 검증 방법 |
|---|---|---|
| G1 | 모든 ActivitySpec이 런타임규격 §2-1 스키마 통과 | ajv/jsonschema |
| G2 | `activity_id` 전체 유일, 정규식 `^[a-z][a-z0-9-]{2,63}$` | Set 크기 + regex |
| G3 | `back.solution_code` 실행 가능 (SyntaxError/ReferenceError 없음) | JS eval 또는 Node 실행 |
| G4 | `back.grading.volume_target` = solution_code measureVolume 실제값 (±1%) | 측정값 비교 |
| G5 | `back.grading.bbox_target` = solution_code measureBoundingBox 실제값 (±0.1mm) | 측정값 비교 |
| G6 | `front.starter_code` JS SyntaxError 없음 | `new Function` 파싱 |
| G7 | `front.hints[0]`가 "선수지식:" 으로 시작 | 문자열 검색 |
| G8 | `front.prompt`에 수치 조건 포함 ("적절히"/"올바르게"/"잘" 미포함) | 문자열 검색 |
| G9 | activity 수 ∈ [4, 6] (MVP 분량 범위) | 배열 길이 |
| G10 | `window.ACTIVITIES['cad-print']` 등록 확인 | JS 콘솔 |

## 5. 난이도별 형상 가이드 (템플릿 보조)

### 입문 (weight 2)

| 형상 | 사용 API | grading 기준치 예 |
|---|---|---|
| 직육면체 | `primitives.cuboid({ size: [x,y,z] })` | vol=x×y×z, bbox=[x,y,z] |
| 구 | `primitives.sphere({ radius: r })` | vol=4/3×π×r³, bbox=[2r,2r,2r] |
| 원기둥 | `primitives.cylinder({ radius, height })` | vol=π×r²×h, bbox=[2r,2r,h] |

### 기초 (weight 4)

| 형상 | 사용 API | 주의 |
|---|---|---|
| 두 솔리드 union | `booleans.union(a, b)` | 겹침 있을 때 부피≠합산 주의 |
| 구멍 뚫기(subtract) | `booleans.subtract(base, hole)` | 구멍이 base를 완전히 관통해야 매니폴드 |
| 닫힌 솔리드 검증 | 의도적 열린 starter + 수정 실습 | manifold 체크가 핵심 학습 목표 |

### 중급 (weight 6, MVP 이후 추가 고려)

- 복합 불리언 연산(3개 이상 오브젝트)
- 파라메트릭 변수화 (`const size = 20; cuboid({ size: [size, size, size/2] })`)
