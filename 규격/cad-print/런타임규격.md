## 2. ActivitySpec — `type: "cad-model"`

플러그인계약 §4의 ActivitySpec 공통 껍질에 cad-print 전용 `front`/`back`을 정의한다.

### 2-1. 완전 형태 (JSON Schema)

```jsonc
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id":     "cad-print/activity-spec/v1",
  "type":    "object",
  "required": ["activity_id","plugin_id","type","front","back","weight","tags","enabled"],
  "additionalProperties": false,
  "properties": {

    // ── 공통셸 공용 필드 (플러그인계약 §4) ──
    "activity_id": { "type": "string", "pattern": "^[a-z][a-z0-9-]{2,63}$" },
    "plugin_id":   { "type": "string", "const": "cad-print" },
    "type":        { "type": "string", "const": "cad-model" },
    "weight":      { "type": "number", "minimum": 1, "maximum": 10 },
    "tags": {
      "type": "object",
      "required": ["area","subarea","unit"],
      "properties": {
        "area":    { "type": "string" },
        "subarea": { "type": "string" },
        "unit":    { "type": "string" }
      }
    },
    "enabled": { "type": "boolean" },

    // ── cad-print 전용: front (사용자에게 보이는 쪽) ──
    "front": {
      "type": "object",
      "required": ["prompt","target_spec","hints"],
      "additionalProperties": false,
      "properties": {
        "prompt":        { "type": "string", "minLength": 10 },
        "starter_code":  { "type": "string" },
        "target_spec":   { "type": "string", "minLength": 5,
                           "description": "목표 형상 설명 (예: '20×20×10mm 직육면체')" },
        "hints":         { "type": "array", "items": { "type": "string" } }
      }
    },

    // ── cad-print 전용: back (해설용, 채점 기준 포함) ──
    "back": {
      "type": "object",
      "required": ["solution_code","grading","why","explanation"],
      "additionalProperties": false,
      "properties": {
        "solution_code": { "type": "string", "minLength": 10,
                           "description": "정답 JSCAD 코드 — 채점기가 grading 기준치를 이 코드로도 검증 가능" },
        "grading": {
          "type":     "object",
          "required": ["volume_target","volume_tolerance_pct","bbox_target","bbox_tolerance"],
          "additionalProperties": false,
          "properties": {
            "volume_target":       { "type": "number", "exclusiveMinimum": 0,
                                     "description": "목표 부피 (JSCAD 단위, mm³ 권장)" },
            "volume_tolerance_pct":{ "type": "number", "minimum": 0, "maximum": 100,
                                     "description": "허용 오차 %  (예: 5 → ±5%)" },
            "bbox_target":         {
              "type": "object",
              "required": ["x","y","z"],
              "additionalProperties": false,
              "properties": {
                "x": { "type": "number", "exclusiveMinimum": 0 },
                "y": { "type": "number", "exclusiveMinimum": 0 },
                "z": { "type": "number", "exclusiveMinimum": 0 }
              },
              "description": "목표 바운딩박스 [x,y,z] (mm)"
            },
            "bbox_tolerance":      { "type": "number", "minimum": 0,
                                     "description": "바운딩박스 각 축 허용 절대 오차 (mm, 예: 0.5)" }
          }
        },
        "why":           { "type": "string" },
        "explanation":   { "type": "string" }
      }
    }

  }
}
```

**설계 결정**: `grading` 채점 기준치는 `back` 오브젝트 내부에 포함한다. 채점 기준이
`back`에 있어야 `solution_code`와 나란히 관리되고, `front`는 학습자에게 보이는
`prompt·target_spec·hints`만 담아 두 관심사가 명확히 분리된다.

## 3. JSCAD 채점 인터페이스

### 3-1. 의존 라이브러리 (CDN, file:// 안전)

| 라이브러리 | CDN URL | 용도 |
|---|---|---|
| `@jscad/modeling` | `https://unpkg.com/@jscad/modeling@2/dist/jscad-modeling.min.js` | 기하 생성·측정 |
| `three.js` | `https://unpkg.com/three@0.160.0/build/three.module.js` | 3D 미리보기 렌더 |

- `@jscad/modeling`은 UMD 번들로 `window.jscadModeling` 전역 노출.
- three.js는 실습러너계약 §2에 고정된 버전(0.160.0) 단일 사용. 중복 로드 금지.
- WASM·worker·iframe **의존 없음**. 순수 JS 실행.

### 3-2. 채점 절차 (score 함수 의사코드)

```
채점(activity, userCode):
  [1] JSCAD 코드 실행
      geometry = jscad_run(userCode)          // § 3-3
      if geometry is ERROR:
          return ScoreResult(
            verdict   = 'incorrect',
            score_raw = 0,
            grader_id = 'jscad',
            feedback  = { ..., hint: '코드 실행 오류: ' + error_msg }
          )

  [2] 매니폴드(watertight) 검사
      manifold = is_manifold(geometry)        // § 3-4
      if not manifold:
          return ScoreResult(
            verdict   = 'incorrect',
            score_raw = 0,
            grader_id = 'jscad',
            feedback  = {
              volume_user:    NaN,
              volume_target:  grading.volume_target,
              volume_err_pct: Infinity,
              bbox_diff:      [Infinity, Infinity, Infinity],
              manifold:       false,
              hint:           '닫힌 솔리드 아님(매니폴드 깨짐) — CSG 불리언 연산 후 열린 면 확인'
            }
          )

  [3] 측정
      vol_user = measureVolume(geometry)      // § 3-5
      bbox     = measureBoundingBox(geometry) // § 3-5
      bbox_user = {
        x: bbox.max[0] - bbox.min[0],
        y: bbox.max[1] - bbox.min[1],
        z: bbox.max[2] - bbox.min[2]
      }

  [4] 오차 계산
      vol_err_pct = |vol_user - vol_target| / vol_target * 100
      bbox_diff   = [
        |bbox_user.x - bbox_target.x|,
        |bbox_user.y - bbox_target.y|,
        |bbox_user.z - bbox_target.z|
      ]

  [5] 판정
      vol_pass  = vol_err_pct  <= grading.volume_tolerance_pct
      bbox_pass = bbox_diff[0] <= grading.bbox_tolerance
                  AND bbox_diff[1] <= grading.bbox_tolerance
                  AND bbox_diff[2] <= grading.bbox_tolerance
      verdict   = (manifold AND vol_pass AND bbox_pass) ? 'correct' : 'incorrect'
      score_raw = _calc_score_raw(manifold, vol_pass, bbox_pass)   // § 3-6

  [6] feedback 조립 (진단형, § 3-7)
      hint = _diagnose_hint(vol_err_pct, vol_pass, bbox_diff, bbox_pass)
      return ScoreResult(
        verdict   = verdict,
        score_raw = score_raw,
        grader_id = 'jscad',
        feedback  = {
          volume_user:    vol_user,
          volume_target:  vol_target,
          volume_err_pct: vol_err_pct,
          bbox_diff:      bbox_diff,
          manifold:       true,
          hint:           hint
        }
      )
```

### 3-3. JSCAD 코드 실행 (`jscad_run`)

```
jscad_run(userCode):
  try:
    // @jscad/modeling UMD로 require 심 주입
    shimmedCode = injectRequireShim(userCode)
    // Function() 생성자로 실행 (new Function은 전역 오염 최소화)
    fn = new Function('require', 'module', 'exports', shimmedCode)
    moduleObj = { exports: {} }
    fn(requireShim, moduleObj, moduleObj.exports)
    // module.exports.main() 또는 직접 exports.main() 호출
    main = moduleObj.exports.main || moduleObj.exports
    geometry = main()
    return { ok: true, geometry }
  catch e:
    return { ok: false, error: e.message }
```

**require 심(shim) 규칙**:
- `require('@jscad/modeling')` → `window.jscadModeling` 반환.
- `require('@jscad/modeling').primitives` → `window.jscadModeling.primitives` 등 서브경로 지원.
- 미등록 모듈명 요청 시 `throw new Error('모듈 미지원: ' + id)` (graceful 오류 메시지).

**실행 격리**:
- `new Function` 실행이므로 `window` 직접 쓰기는 가능하지만 의도치 않은 전역 오염은
  `use strict` + 심 화이트리스트로 제한.
- 실행 시간 제한: `Promise.race` + 3초 타임아웃. 초과 시 `verdict='incorrect'`,
  `feedback.hint = '코드 실행 시간 초과(3초) — 무한 루프 가능'`.

### 3-4. 매니폴드 검사 (`is_manifold`)

```
is_manifold(geometry):
  // JSCAD @jscad/modeling measureVolume 활용
  // measureVolume이 NaN 또는 음수(≤0)면 비매니폴드 판정
  vol = jscadModeling.measurements.measureVolume(geometry)
  if isNaN(vol) OR vol <= 0:
    return false
  return true
```

- `measureVolume` 내장 함수가 비매니폴드(열린 면, 자기교차) 시 `NaN` 또는 음수를 반환하는
  성질을 이용한다.
- `geometry`가 `null` / `undefined` / 배열이지만 길이 0인 경우도 `false` 반환 (graceful).
- **throw 절대 금지** — 모든 예외는 try/catch로 잡아 `false` 반환.

### 3-5. 측정 함수 (`measureVolume`, `measureBoundingBox`)

| 함수 | JSCAD API | 반환 |
|---|---|---|
| `measureVolume(geom)` | `jscadModeling.measurements.measureVolume(geom)` | `number` (mm³, NaN 가능) |
| `measureBoundingBox(geom)` | `jscadModeling.measurements.measureBoundingBox(geom)` | `[[minX,minY,minZ],[maxX,maxY,maxZ]]` |

- `measureBoundingBox` 반환값을 `bbox.min = [0]`, `bbox.max = [1]` 인덱스로 접근.
- 각 축 치수 = `max[i] - min[i]` (항상 ≥ 0).

### 3-6. score_raw 계산

```
_calc_score_raw(manifold, vol_pass, bbox_pass):
  if not manifold:    return 0
  if vol_pass AND bbox_pass: return 1
  if vol_pass XOR bbox_pass: return 0.5
  return 0
```

- 부분점수(0.5)는 부피·바운딩박스 중 하나만 통과할 때 부여.
- 플러그인계약 §4 `score_raw: float01` conform.

### 3-7. ScoreResult 반환 형태

플러그인계약 §4 ScoreResult conform:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect";   // §3-2 판정 결과
  score_raw: number;                    // §3-6, 0 | 0.5 | 1
  grader_id: "jscad";
  feedback: {
    // ── 측정치 ──
    volume_user:     number;   // 사용자 코드 실행 결과 부피 (mm³, NaN = 비매니폴드)
    volume_target:   number;   // 목표 부피 (grading.volume_target)
    volume_err_pct:  number;   // |user-target|/target*100 (Infinity = NaN 입력)
    bbox_diff:      [dx: number, dy: number, dz: number];  // 축별 절댓값 차이 (mm)
    manifold:        boolean;  // watertight 여부
    // ── 진단 메시지 ──
    hint:            string;   // 방향 진단 (§3-8)
    // ── 실행 오류 시만 존재 ──
    error?:          string;   // 코드 실행 예외 메시지
  };
}
```

### 3-8. hint 진단 메시지 규칙 (Hattie 피드백 모델)

| 조건 | hint 내용 |
|---|---|
| 코드 실행 오류 | `'코드 실행 오류: ' + error_msg` |
| `manifold = false` | `'닫힌 솔리드 아님(매니폴드 깨짐) — CSG 불리언 연산 후 열린 면 확인'` |
| 코드 타임아웃 | `'코드 실행 시간 초과(3초) — 무한 루프 가능'` |
| 부피 초과 (vol_err_pct > tol) | `'부피가 목표보다 ' + vol_err_pct.toFixed(1) + '% 큼'` |
| 부피 부족 (vol_err_pct > tol) | `'부피가 목표보다 ' + vol_err_pct.toFixed(1) + '% 작음'` |
| 바운딩박스 오차 (bbox 통과, 부피 실패) | `'전체 크기는 맞으나 내부 형상 오류(부피 불일치)'` |
| 부피 통과, bbox 실패 | `'부피는 맞으나 ' + 오류축 + '축 크기가 ' + diff.toFixed(1) + 'mm 차이'` |
| 전체 통과 | `'목표 형상과 일치합니다.'` |

- hint는 **항상 한국어**, 방향 진단 포함 (무엇이 얼마나 틀렸는지).
- 이진 "맞음/틀림" 금지 → 수치 포함 진단 (Hattie 즉각 피드백 모델).

## 4. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. cad-print 전용 사항:

### 4-1. mount(container, ctx)

```
1. practice-runner 생성 (실습러너계약 §4 표준 패턴 준수)
   runner = createPracticeRunner({
     container,
     onScore: (result) => { /* 진도 캐시 업데이트 */ },
     onActivityComplete: (result) => { ctx.emit({ type: 'activity-completed', result }) }
   })
2. 코드 에디터 렌더 (plain <textarea> + monospace — CodeMirror 선택적)
   - 초기값: activity.front.starter_code (없으면 빈 JSCAD 템플릿)
   - 비전공자 친화: starter_code에 require 심 샘플 + 빈칸(// 여기에 작성) 포함
3. 3D 미리보기 캔버스 렌더 (three.js, practice-runner 소유)
4. "실행 미리보기" 버튼: 코드 실행 → 3D 렌더 갱신 (채점 없음)
5. "제출·채점" 버튼: score(userAnswer) 호출 → 결과 렌더
6. onProgressRestored() 호출됐으면 저장된 코드 복원(plugin_extra.last_code)
7. target_spec + hints를 사이드패널에 표시 (back.grading 수치 비노출 — 힌트만)
```

### 4-2. score(userAnswer)

```ts
// userAnswer: string  (사용자 JSCAD 코드)
// 반환: Promise<ScoreResult>
score(userAnswer: string): Promise<ScoreResult>
```

- `userAnswer` = 에디터 현재 코드 (string).
- §3-2 채점 절차 실행 → ScoreResult 반환.
- **throw/crash 절대 금지** — 모든 예외(코드 실행·측정·JSON 오류)를 잡아
  `verdict='incorrect' + feedback.error = 예외 메시지 첫 줄` 반환.
- 완료 후 셸에 `ctx.emit({ type: 'activity-completed', result })` 방출.

## 6. 3D 미리보기 렌더 (practice-runner 위임)

- practice-runner가 three.js WebGLRenderer를 생성하고 컨테이너에 `<canvas>` 주입.
- cad-print 플러그인은 JSCAD geometry를 three.js `BufferGeometry`로 변환해 씬에 추가.
- **변환 규칙**:
  ```
  jscad → polygons = geom3.toPolygons(geometry)
  polygons → THREE.BufferGeometry (vertices + indices)
  재질 = THREE.MeshNormalMaterial (법선 기반 컬러, MVP 간단)
  ```
- 미리보기 전용 ("실행 미리보기" 클릭 시만 갱신) — 코드 타이핑 시 자동 갱신 금지
  (무거운 리컴파일 방지).
- practice-runner의 dispose()가 renderer·geometry·material 일괄 정리.
