## 계약/wiring 주의

### killer_point — judge 선택지 버튼 wiring

- **증상**: options[i] 텍스트 대신 인덱스(0/1/2)를 answer로 전달 시 ScoreInputError 없이 incorrect 오판정(silent wrong behavior).
- **올바른 구현 경로**: 버튼 클릭 → `options[i]` 텍스트값을 collectAnswer() 반환값으로 사용 → exact 비교 → 엔진 무수정.
- **근거**: 엔진규격 §3.2 ScoreInput.mode=exact는 userAnswer:string(선택지 텍스트) 수용. 인덱스는 contract 외.
- [검토 killer_point 원문, 방향성 §4]

### self 카드 O/X → correct/incorrect 변환 책임

- **변환 주체 = UI 레이어** (SoT §5·§10). 엔진 grade()는 'correct'|'incorrect'만 수용.
- plugin.js collectAnswer()에서 'o'→'correct', 'x'→'incorrect' 변환 후 grade() 전달 필수.
- 'o'/'x' 직접 전달 시 ScoreInputError — 런타임 guard 없으면 오판정.
- [카드규격 §3.4, 엔진규격 §3.2]

### cloze blanks[] 0-base 정합

- UI의 빈칸 입력 맵 `{n:string}` → 마커 순서(0, 1, …) 기준 string[] 변환 후 grade() 전달.
- 순서 오배열 시 silently incorrect — 빌드타임 §8.3 검증은 런타임 오배열을 잡지 못함.
- [카드규격 §2.4, 엔진규격 §3.2, SoT §10]

### 단원 필터 — 집계 무결성 경계

- cards 배열 사전필터 후 buildQueue 호출 시 getDashboardContrib의 집계 대상과 출제 대상 불일치 가능성.
- getDashboardData는 'enabled==true 카드만 집계'(엔진규격 §7.3) — unit 필터는 이 경계 밖.
- mid 구현 전 엔진규격 §7.3·§2.5 확인 필수.

### D-day 토글 — 간격압축 동작 확인

- 현재 force-review-all 버튼은 dDayMode=true만 주입, Leitner 간격압축 여부 미명시.
- 엔진규격 §2.4 dDayMode 경로(due 무시·box ASC·간격압축) 실제 동작 확인 후 토글 UI 구현.
