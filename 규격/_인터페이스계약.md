# v5 인터페이스 계약 (코어 SoT)

> buildflow ② SoT-first 산출. **프레임워크 최상위 공유 계약.** 충돌 시 우선순위: 본 문서 > per-folder 규격.
> 범위: 공유 타입 · 전역 상수 · 영속/사이드카 스키마 · 봇↔엔진↔콘텐츠 와이어링 · 능력 레지스트리 형태 · AI 어댑터 · 에러.
> 콘텐츠/config 주입 상세 = `주입인터페이스.md`. 엔진 함수·집계 entry 상세 = `엔진계약.md`. 둘 다 본 SoT에 conform.
> 런타임 = 파이썬 3.11+. 타입 = 의사타입(dataclass/TypedDict 권장). 검증 = 런타임 가드. 명명 = snake_case.
> (G2 무결성 리뷰 반영: DashboardData·HandlerResult 정의 · 사이드카 계약 · 엔진 완전 순수 · target 주입화.)

## 0. 용어·전역 상수

| 상수 | 값 | 설명 |
|---|---|---|
| `BOX_MIN` | `1` | 최소 박스 |
| `BOX_MAX` | `3` | 최대 박스(도달 후 cold 정답 시 졸업) |
| `BOX_INTERVALS_DAYS` | `{1: 1, 2: 3, 3: 7}` | 박스→재출제 간격(일). config 주입 오버라이드 |
| `SCHEMA_VERSION` | `1` | 진도 스키마 버전(정수, 단조증가) |
| `MS_PER_DAY` | `86_400_000` | 1일 밀리초 |
| `DDAY_COMPRESS_DAYS` | `1` | D-day 간격압축 상한일(기본, config 주입 가능) |

- **시각 = epoch ms(정수, UTC).** `now`는 호출자 주입(테스트 결정성).
- **due 판정**: `now >= due_at` → due(경계 포함).
- 상수는 코어 **기본값**. `LeitnerConfig`(주입)가 있으면 그 값(2번째 과목 무재협상).

**.env 키** — 필수 4 + 선택:
- 필수: `DISCORD_BOT_TOKEN` · `DISCORD_GUILD_ID` · `DISCORD_CHANNEL_ID` · `DISCORD_ALLOWED_USER_ID`. (브리지와 별개 앱/토큰/서버.)
- 선택: `USER_LANG`(사용자 출력 언어, 기본 `ko`) · `MOUNT`(기본 마운트 폴더). 미지정 시 기본값.

## 1. 공유 데이터 타입 (의사타입)

```python
Verdict   = Literal["correct", "incorrect", "skip"]   # score()는 skip 미생성
ScoreMode = Literal["exact", "keyword", "cloze", "self"]
CardType  = Literal["func", "proc", "recall_seq", "cloze", "judge"]
Effort    = Literal["low", "medium", "high"]

@dataclass
class AnswerSpec:
    accepted: list[str] | None          # exact 후보(OR)
    required_keywords: list[list[str]] | None  # keyword. 내부=한 그룹 동의어(any-of), 모든 그룹 필수
    blanks: list[list[str]] | None      # cloze. blanks[i]=i번 빈칸 후보(0-base)
    sequence: list[str] | None          # recall_seq 순서 단계
    normalize: list[str]                # 정규화 규칙 id 순서배열. self면 생략 가능

@dataclass
class CardDef:
    card_id: str                        # ^[a-z][a-z0-9-]{2,63}$ (콜론 금지)
    schema_version: int
    subject: str; unit: str
    type: CardType
    grade_mode: ScoreMode               # 콘텐츠 기본. 실효 모드는 봇이 scoring_overrides 반영(봇계약 §4)
    front: dict; back: dict
    answer_spec: AnswerSpec | None      # self면 None
    tags: dict                          # {weight:int[1,10]=5, area?:str, subarea?:str}
    links: dict                         # {concept_ref?:str}
    enabled: bool

@dataclass
class DeckData:
    namespace: str
    cards: list[CardDef]

@dataclass
class CardProgress:                     # 진도(영속 §2). 학습 알고리즘 상태만(보조 상태는 사이드카 §2.1)
    card_id: str
    box: int                            # BOX_MIN..BOX_MAX
    due_at: int                         # epoch ms
    graduated: bool
    cold_attempts: int                  # skip 포함
    cold_correct: int
    last_attempt_at: int | None
    last_verdict: Verdict | None

@dataclass
class ProgressStore:
    schema_version: int
    deck_namespace: str
    cards: dict[str, CardProgress]

@dataclass
class ScoreInput:
    mode: ScoreMode
    user_answer: str | list[str] | Literal["correct", "incorrect"]
    answer_spec: AnswerSpec
    synonyms: dict[str, str] | None     # 역인덱스(동의어->대표어). 봇 부트가 config에서 컴파일해 공급

@dataclass
class ScoreResult:
    verdict: Literal["correct", "incorrect"]
    matched: list[str]; missed: list[str]
    normalized_user: str | list[str]
    feedback: dict                      # {"highlight_missed": list[str]}

@dataclass
class HandlerResult:                    # 학습 능력 핸들러 반환(봇계약 §3·§4)
    card_id: str
    verdict: Verdict | None             # None=제시만/미응답. skip은 "skip"
    requeue: bool = False               # 세션내 재큐(오답 교정)
    done: bool = True                   # 이 카드 상호작용 종료 여부

@dataclass
class LeitnerConfig:
    intervals_days: dict[int, int]      # {1:1,2:3,3:7}
    dday_compress_days: int             # 기본 1

@dataclass
class QueueOptions:
    deck_namespace: str
    dday_mode: bool = False
    new_card_limit: int | None = None
    review_limit: int | None = None
    weight_overrides: dict[str, int] | None = None   # card_id->weight(사이드카 §2.1 주입). 없으면 CardDef.tags.weight

@dataclass
class DashboardData:                    # get_dashboard_data 반환. entry 타입은 엔진계약 §5 정의
    by_area: list                       # ByAreaEntry[]   (엔진계약 §5)
    weakness: list                      # WeaknessEntry[] (엔진계약 §5)
    pass_path: list                     # PassPathEntry[] (엔진계약 §5)
    completion: list                    # CompletionEntry[](엔진계약 §5)
```
- 식별 필드는 `area`+`subarea`(area_id 금지). 4키·식별필드가 SoT 권위. entry 필드 = 엔진계약 §5.
- **SubjectConfig**(정규화 프로파일·동의어·채점매핑·Leitner·**pass_targets**)는 `주입인터페이스.md §5`. 엔진은 컴파일·주입된 값만 소비.

## 2. 영속 스키마 (진도)

- 진도 = **마운트 폴더** 하위 `<mount>/_상태/progress-<deck_namespace>.json`(deck별) → `ProgressStore`.
- 메타 = `<mount>/_상태/meta.json` → `{schema_version, created_at, last_opened_at}`(ms).
- card_id 1차 키. 키/namespace에 `:` 금지. card_id 정규식 = §1.
- **파일 I/O 소유 = 봇(harness `automation/store.py`).** 원자적 쓰기(`.tmp` 후 replace) + `.bak` 손상 폴백. 엔진코어는 파일 I/O 안 함(§3).
- **마이그레이션 shim**: `schema_version` 부재=v0(레거시)→migrate 체인, `> SCHEMA_VERSION`→`SchemaVersionError`, parse 실패=빈 스토어 폴백 + `.bak` 보존(throw 안 함). migrate(순수)는 엔진코어 소관(엔진계약 §4).
- 세션 상태(seen_card_ids 등)는 영속 안 함(휘발, 봇 재시작=세션 재시작). `persistviews` 버튼은 custom_id로 복원(봇계약 §7).

### 2.1 사이드카 저장 계약 (엔진 스키마 외 보조 상태)

CardProgress(학습 알고리즘 상태) **밖의** 보조 상태가 필요한 능력(자신감 평정·읽기위치·정교화 서술·due 중복방지·weight 재산출 등)은 **사이드카 파일**을 쓴다. **CardProgress 스키마를 오염시키지 않는다**(불변 §7.4).
- 경로: `<mount>/_상태/sidecar-<capability_id>-<deck_namespace>.json`. 형태: `{card_id: <capability가 정의한 값>}` 또는 `{"_meta": ...}`.
- **소유 = 봇(harness store).** 엔진코어는 사이드카를 **모른다**(읽기·쓰기 안 함). 마이그레이션은 capability별 자체(기본값 폴백).
- 예: `confidence`(card_id→"easy"|"med"|"hard") · `read_pos`(deck→idx) · `elaboration`(card_id→list[str]) · `alert_sent`(card_id→last_ms) · `adaptive_weight`(card_id→int). 후자는 buildQueue에 `QueueOptions.weight_overrides`로 주입(봇이 사이드카에서 로드).

## 3. 봇 ↔ 엔진 ↔ 콘텐츠 와이어링

**의존 방향(단방향):**
```
봇/ (discord.py 셸)  ──▶  엔진코어/ (완전 순수 파이썬)
       │            ──▶  harness/ (복사 카탈로그: Discord I/O·파일 store)
       └──마운트(읽기)──▶  콘텐츠(소비 프로젝트 주입)
```
- **엔진코어/는 완전 순수** — 입력→판정·집계, **부수효과·파일 I/O 0**. discord·harness import 금지. (진도 파일 읽기·쓰기는 봇이 harness로 수행하고, 엔진엔 in-memory 데이터만 넘긴다.)
- **봇/만 discord·harness·콘텐츠를 안다.** 엔진코어 import해 채점·Leitner·큐·대시보드·migrate 호출, harness로 Discord 표현·파일 store.
- **콘텐츠는 데이터**(코드 아님): manifest·deck·config 파일(주입인터페이스).

**부트 순서(봇):**
1. `.env` 로드(필수 4키 검증, 누락→구동 거부).
2. 마운트 = `argv[1] > .env MOUNT > cwd`.
3. manifest→deck·config 로드, 주입 검증(`ContentInjectionError`→부트 차단), 컴파일(정규화 프로파일·synonyms 역인덱스·card_id→실효 grade_mode 맵·pass_targets).
4. harness store로 진도 JSON 읽기 → `engine.migrate`로 버전 정규화 → ProgressStore.
5. 능력 레지스트리(§4) 구성 → discord.py Client 기동 → `on_ready`: persistviews 재등록 + 권한 체크 + presence.
- 네트워크 = Discord 게이트웨이만. AI 모드만 `_invoke`(§5).

## 4. 능력 레지스트리 (학습 타입 ↔ 프리미티브)

```python
@dataclass
class LearningCapability:
    capability_id: str                  # 안정 id
    layer: Literal[1, 2, 3, 4]
    tier: Literal["core", "extension"]
    primitives: list[str]               # harness 파일
    engine_fns: list[str]               # 의존 엔진 함수
    needs_ai: bool                      # True면 _invoke 의존(layer 3). 필수 명시(미결정 금지)
    handler: str                        # 봇 핸들러 식별자(봇계약 dispatch)
```
- 활성 집합 = config `capabilities.enabled`(주입인터페이스 §7). layer/tier/primitives/needs_ai = `학습타입규격.md` 권위(각 능력 needs_ai 확정값 보유).
- layer 4(인프라)는 핸들러가 아니라 부트/미들웨어 상시 와이어(레지스트리 enabled와 무관, 항상 활성).

## 5. AI 어댑터 인터페이스 (`_invoke`)

AI 능력(layer 3)은 **이 어댑터만** 의존(discord-bridge 미의존, 패턴 복제). **본 §5 = 인터페이스 권위, 구동 상세(CLI 플래그·OS) = `AI모드골격.md §1`.**

```python
@dataclass
class AIResult:
    text: str; ok: bool; error: str | None

async def invoke(prompt, *, system=None, model=None, effort: Effort = "low",
                 max_tokens=None, session_id=None, on_stream=None) -> AIResult: ...
```
- `session_id=None` → 단발(신규 세션). 지정 → 그 세션 재개(멀티턴). 구동 명령(`claude -p stream-json`·`--session-id`/`--resume`·Windows) = AI모드골격 §1.
- 토큰 통제(계약): 짧은 system · effort low 기본 · max_tokens 상한 · 조건 미충족 시 미호출. 상세 = AI모드골격 §2.

## 6. 에러 규약

| 에러 | 발생 | 처리 |
|---|---|---|
| `ScoreInputError` | mode↔user_answer 타입·cloze 빈칸수·self 비허용값·알 수 없는 grade_mode·키 콜론 | throw |
| `SchemaVersionError` | 저장 버전 > 코드 버전 | throw, 로드 중단 |
| `StorageError` | 파일 쓰기 실패·쿼터 | throw, export 권고 |
| `ManifestMissingError` | 마운트 manifest 부재 | throw, 부트 실패 |
| `DeckNotFoundError` | 요청 namespace 미등록 | throw |
| `ContentInjectionError` | 주입 콘텐츠/config 스키마 위반 | throw, 부트 차단 |
| `AIInvokeError` | `_invoke` 실패·타임아웃 | catch, graceful 폴백 |

- 복구 가능 손상(parse 실패·migrate 예외)은 폴백+`.bak`(throw 아님). 비밀값 미노출.

## 7. 충돌 우선순위 · 불변 (체크리스트)

**우선순위:** 본 SoT > per-folder > 콘텐츠/config. per-folder는 SoT에 conform.

**불변(모순 금지):**
1. **과목 무관** — 엔진·harness·봇에 과목 어휘·콘텐츠·임계값 하드코딩 금지(pass_targets 등 전부 주입).
2. **복사 모델** — discord-bridge import·실행·호출 0. harness 복사 + `_invoke` 패턴 복제만.
3. **봇 격리** — 새 Discord 앱·토큰·서버. 토큰당 1봇.
4. **엔진 vs harness 경계** — 학습 알고리즘(채점·정규화·Leitner·인터리빙·큐·대시보드·migrate)=엔진코어 **완전 순수**. Discord I/O·진도/사이드카 파일저장=harness(봇이 호출). 엔진코어는 파일 I/O·discord·harness import 0.
5. **이진 채점** — 정/오 이진(부분점수·합격확률·자동채점 산출 없음).
6. **박스 이동 = cold 첫시도 정답만.** warm/skip 무관.
7. **card_id 안정성** — 작성자 안정 id, 콘텐츠 수정 불변, content-hash 아님.
8. **결정적=토큰0** — layer 1·2·4 외부 호출 0. layer 3(AI)만 `_invoke`. (결정적 규칙 부분은 layer 1/2, AI 제안 부분만 layer 3.)
9. **스키마 버전 + 마이그레이션 shim** v1부터. CardProgress 스키마 오염 금지(보조 상태=사이드카 §2.1).
10. **synonyms 치환 위치** = 정규화 lower 직후·비교 직전.
