# 학습 프레임워크 — 자율작업 종료 브리핑

> 전권 위임 자율작업 결과. 네가 판단·실행해야 할 것 정리. 원시 파킹 로그 = `_사용자판단대기.md`.
> 작업방식: 플러그인별 park-and-continue(사용자 판단부 회피·기록, 같은 작업 계속) + 복수에이전트 웹서치 검증.

## 1. 완료된 것 (cycle-2: 프레임워크 + 5 플러그인)
- **플러그인 라이브러리 아키텍처 완성**: 공통셸(레지스트리·라우터·진도·대시보드·키설정) + 5 플러그인 전부 등록·통합 확인(node 하니스로 5종 PLUGIN_REGISTRY 등록 검증).
- 각 플러그인 = **2면 계약**(런타임 소비 규격 + 생성규칙). 도메인 미리 안 박음 — 온디맨드 생성용 규칙. + 적응형 `깊이스무고개.md`(생성 전 진단형 인테이크).
- git 커밋·태그: `cycle2-foundation` → `coding` → `excel` → `english` → `aws`. (롤백 가능)

| 플러그인 | 런타임 | 채점 | node 검증 |
|---|---|---|---|
| card-quiz | 카드 타이핑·Leitner·오답회귀 | 정적 | onCard 발화 ✓ |
| coding | CodeMirror + Pyodide(파이썬) | 테스트케이스(정적) | Pyodide 채점 실측 ✓(정답3/3·오답1/3) |
| excel | Univer 스프레드시트 | 셀 값비교(정적) | 채점로직 21 pass ✓ (Univer canvas=브라우저 필요) |
| english | 멀티모달(vocab/grammar/reading/listening) | 텍스트비교(정적) | 17 pass ✓ (Web Speech=브라우저 필요) |
| aws | 가이드 핸즈온(과제+검증) | boto3 백엔드(외부) | 프론트 graceful ✓ (백엔드=네 인프라) |

## 2. 지금 바로 되는 것 / 준비 필요한 것
- **바로**(더블클릭 `엔진/app/index.html`, 인터넷만): card-quiz, coding(파이썬), english(vocab/grammar/reading/listening), excel UI.
- **준비 필요**: english writing/speaking(키), aws(Docker/계정), excel 수식동치 채점(PySheetGrader).

## 3. ★ 네 판단·작업이 필요한 것 (카테고리별)

### A. 브라우저 실측 (열어서 눈으로 — 내가 헤드리스라 못 함)
- **전체 UI/UX·5개 플러그인 탭 동작**을 `index.html` 더블클릭해 확인.
- **Univer(excel)**: canvas 렌더라 node 불가 — 셀편집·수식계산·채점이 실제 뜨는지. UMD 전역명/버전(현재 unpkg latest, 핀 안 됨) 안 뜨면 CDN 버전 조정 필요.
- **Web Speech(english)**: listening TTS 재생·speaking STT 녹음(Chrome/Edge만).

### B. BYO API-key 입력 (네 과금)
- **english writing**(LLM 채점): OpenAI-compatible 키. → 어느 엔드포인트 쓸지 결정(OpenAI/Anthropic/로컬).
- **english speaking**(발음): Azure Speech 키 + region. (실 SDK 통합은 v2 — 현재 pending stub)

### C. 로컬 인프라 실행 (네 PC)
- **aws**: ① `docker run --rm -p 4566:4566 localstack/localstack`(또는 네 AWS 계정+creds, 비용) ② `pip install boto3` ③ `python 엔진/plugins/aws/backend/grade.py`(README 참조). 연결 전엔 과제 열람만.
- **excel 수식동치 채점**: PySheetGrader(파이썬 백엔드) — 현재는 값비교만.
- **coding 다언어**(C/Java 등): Judge0 Docker. 현재 파이썬(Pyodide)만.

### D. 설계 결정 (네 의견 필요)
- **깊이스무고개 진단문항 형태**(객관식/단답/미니문제) — 과목·플러그인별 어떻게? (`깊이스무고개.md` [OPEN])
- **재학습 시 LearningProfile 재사용/갱신** 흐름 — 미정.
- excel 값비교 string/number 크로스타입('15' vs 15) 허용 여부.

### E. 정리(cleanup)
- aws `런타임규격.md` 문서에 plugin_id "aws-lab" 오기(실제 코드는 "aws"로 일관) — 문서만 수정.

## 4. v2 백로그 (지금 불필요, 나중)
