## @comp1-ss-func-0001
- type: func
- grade_mode: exact
- weight: 9
- concept_ref: comp.ss.vlookup
- answers: VLOOKUP

---FRONT---
세로 방향으로 기준값을 검색해 지정한 열의 값을 반환하는 함수는?

---BACK---
**구문**: `=VLOOKUP(기준값, 범위, 열번호, 옵션)`

**인수**
| 이름 | 필수 | 설명 |
|------|:---:|------|
| 기준값 | Y | 찾을 값 |
| 범위 | Y | 검색 대상 표 범위 (첫 열이 기준 열) |
| 열번호 | Y | 반환할 열 번호 (1-base) |
| 옵션 | N | TRUE(생략)=근사일치 / FALSE=정확일치 |

**반환**: 범위의 열번호 열에서 해당 행의 값(텍스트·숫자·논리값 등)

**실수포인트**
- 정확일치 검색 시 반드시 옵션=FALSE(또는 0) 지정
- 범위 첫 열이 기준값 열이어야 함 — 왼쪽 열 반환 불가
- 여러 열 반환 시 MATCH와 조합, 또는 INDEX+MATCH 사용
- 옵션=TRUE(근사)이면 기준 열이 오름차순 정렬돼 있어야 정확한 결과

**유사함수 대조**: HLOOKUP — 가로 방향 검색(행 번호 반환) / INDEX+MATCH — 왼쪽 열 반환 가능

## @comp1-ss-func-0002
- type: func
- grade_mode: exact
- weight: 9
- concept_ref: comp.ss.sumif
- answers: SUMIF

---FRONT---
단일 조건에 맞는 셀의 합계를 구하는 함수는?

---BACK---
**구문**: `=SUMIF(범위, 조건, [합계범위])`

**인수**
| 이름 | 필수 | 설명 |
|------|:---:|------|
| 범위 | Y | 조건을 검사할 범위 |
| 조건 | Y | 합계 대상을 결정하는 조건 (숫자·텍스트·수식) |
| 합계범위 | N | 실제 합계할 셀 범위. 생략 시 범위 자체를 합산 |

**반환**: 조건을 만족하는 셀에 대응하는 합계범위 값의 합(숫자)

**실수포인트**
- 다중 조건이면 SUMIFS 사용 (범위/조건 쌍을 여러 개 지정)
- 조건에 와일드카드 `*` `?` 사용 가능
- 합계범위 생략 시 범위 자체가 합산 대상 — 범위와 합계범위가 다른 경우 반드시 명시

**유사함수 대조**: SUMIFS — 다중 조건 합계 / COUNTIF — 개수 / AVERAGEIF — 평균

## @comp1-ss-func-0003
- type: func
- grade_mode: exact
- weight: 8
- concept_ref: comp.ss.countif
- answers: COUNTIF

---FRONT---
단일 조건에 맞는 셀의 개수를 세는 함수는?

---BACK---
**구문**: `=COUNTIF(범위, 조건)`

**인수**
| 이름 | 필수 | 설명 |
|------|:---:|------|
| 범위 | Y | 조건을 검사할 셀 범위 |
| 조건 | Y | 개수를 셀 조건 (숫자·텍스트·셀참조·수식) |

**반환**: 조건을 만족하는 셀의 개수(정수)

**실수포인트**
- 다중 조건이면 COUNTIFS 사용
- 조건에 비교 연산자 포함 시 큰따옴표로 묶음: `">100"`, `"<>"&A1`
- 텍스트 조건 와일드카드 `*`(임의 문자열) / `?`(임의 한 문자) 사용 가능
- COUNT는 숫자만, COUNTA는 비어있지 않은 셀, COUNTBLANK는 빈 셀

**유사함수 대조**: COUNTIFS — 다중 조건 / COUNT — 숫자 셀만 / COUNTA — 비어있지 않은 셀

## @comp1-ss-cloze-0001
- type: cloze
- grade_mode: cloze
- weight: 9
- concept_ref: comp.ss.vlookup

---FRONT---
다음 VLOOKUP 수식의 빈칸을 채우시오.
=VLOOKUP(찾을값, 범위, {{열번호}}, {{FALSE}})

---BACK---
VLOOKUP 인수 구성: `=VLOOKUP(기준값, 범위, 열번호, 옵션)`

- 세 번째 인수 **열번호**: 범위에서 반환할 열의 번호(1-base 정수)
- 네 번째 인수 **옵션**: FALSE(또는 0)=정확일치, TRUE(생략)=근사일치
- 정확일치 검색이 필요한 상황에서 옵션 생략 시 근사일치가 적용돼 오답 가능

## @comp1-ss-judge-0001
- type: judge
- grade_mode: exact
- weight: 8
- answers: FALSE로 변경|FALSE|옵션을 FALSE로 변경

---FRONT---
VLOOKUP 수식에서 #N/A 오류가 발생했다. 검색 대상은 분명히 범위 안에 있는데 찾지 못한다. 가장 먼저 확인해야 할 것은?

---BACK---
**정답 근거**: 네 번째 인수(옵션)가 TRUE(근사일치) 상태이면 기준 열이 오름차순 정렬돼 있지 않을 때 정확한 값도 못 찾아 #N/A가 발생한다. 옵션을 FALSE(정확일치)로 변경하면 정렬 무관하게 동작한다.

**오류별 원인 요약**
- `#N/A`: 사용할 수 없는 값 — 찾는 값이 범위에 없거나 옵션 불일치
- `#REF!`: 유효하지 않은 셀 참조 (삭제된 행/열 참조)
- `#VALUE!`: 잘못된 인수 타입
- `#NAME?`: 인식 불가 텍스트 (함수명 오타 등)

- type: proc
- grade_mode: keyword

---FRONT---
엑셀에서 피벗 테이블을 새 워크시트에 만드는 절차는?

---BACK---
**단계**
1. 원본 데이터 범위의 임의 셀 선택
2. [삽입] 탭 클릭
3. [표] 그룹 → [피벗 테이블] 클릭
4. '피벗 테이블 만들기' 대화상자 → 범위 확인
5. 위치: '새 워크시트' 선택 (기본값)
6. [확인] 클릭

**메뉴경로**: [삽입] → [표] → [피벗 테이블]

**주의**
- 원본 변경 후 반드시 [새로 고침] 필요 (자동 갱신 안 됨)
- 값 영역의 셀은 직접 수정 불가
- 피벗 차트 생성 시 피벗 테이블 자동 생성됨
- 공유 통합 문서에서는 피벗 테이블 변경 불가

---FRONT---
조건부 서식의 기존 규칙을 수정·삭제·우선순위 변경하려면 어디서 설정하는가?

---BACK---
**단계**
1. 조건부 서식이 적용된 셀 선택 (또는 전체 범위)
2. [홈] 탭 클릭
3. [스타일] 그룹 → [조건부 서식] 클릭
4. [규칙 관리] 클릭
5. '조건부 서식 규칙 관리자' 대화상자에서 규칙 수정·삭제·순서 변경

**메뉴경로**: [홈] → [스타일] → [조건부 서식] → [규칙 관리]

**주의**
- 우선순위가 높은 규칙(위쪽)이 충돌 시 적용됨
- 수식으로 조건 지정 시 반드시 `=` 입력
- 다른 워크시트·통합 문서를 조건 참조로 지정 불가
- 조건 불만족 시 서식 자동 해제

---FRONT---
엑셀 고급 필터를 실행하는 단계 순서를 나열하시오.

---BACK---
**단계 순서**
1. 데이터 범위 내 셀 선택
2. [데이터] 탭 클릭
3. [정렬 및 필터] 그룹 → [고급] (고급 필터) 클릭
4. 조건 범위 지정 (첫 행 = 원본 필드명과 동일; 수식 조건 시 필드명을 다르게)
5. 결과 위치 지정 (현재 위치 필터 또는 다른 장소에 복사)
6. [확인] 클릭

**주의**
- 조건 범위 첫 행: 원본 필드명과 동일 (수식 조건만 예외)
- 같은 행 조건 = AND, 다른 행 조건 = OR
- 고급 필터 결과는 자동 필터와 달리 중복 제거 옵션 있음
