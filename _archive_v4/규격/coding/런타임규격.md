## 1. PluginManifest 선언

```js
// 파일: plugins/coding/manifest.js
window.MANIFEST_CODING = {
  plugin_id:              "coding",
  label:                  "코딩 실습",
  version:                "1.0.0",
  infra:                  "static",          // Pyodide + CodeMirror CDN, entry_url 불요
  byok:                   null,              // 키 불요
  capabilities:           ["code-exec"],
  scoring_mode:           "auto",
  activity_type:          "code-problem",
  progress_schema_version: 1
};
```

**infra = static 근거**: Pyodide(WebAssembly Python 런타임)·CodeMirror(에디터)를 CDN에서 로드, 채점 전부 브라우저 내 실행. 서버·키·외부 API 0.

## 3. Pyodide 채점 인터페이스

### 3-1. 채점 절차

```
채점(activity, userCode):
  compare   ← activity.grading.compare   // "stdout-trim" | "stdout-float-tol"
  tolerance ← activity.grading.tolerance // stdout-float-tol 전용, 기본 1e-6
  passed ← 0
  first_fail ← null
  cases ← []                             // 전체 케이스 결과 배열
  for (idx, tc) in enumerate(activity.grading.test_cases):
    actual_stdout ← pyodide_run(userCode, stdin=tc.input)   // §3-2
    ok ← compare(actual_stdout, tc.expected, compare, tolerance)  // §3-3
    cases.append({ idx, input: tc.input, expected: tc.expected,
                   actual: actual_stdout, pass: ok })
    if ok: passed++
    else if first_fail is null:
      first_fail ← { idx, input: tc.input, expected: tc.expected, actual: actual_stdout }
  total ← len(test_cases)
  verdict ← (passed === total) ? "correct" : "incorrect"
  return ScoreResult(verdict, passed/total, first_fail, cases)
```

### 3-2. Pyodide 실행 규격

| 항목 | 규격 |
|---|---|
| 런타임 | `pyodide.runPythonAsync(code)` (Pyodide CDN, ≥0.26) |
| stdin | `test_case.input` 문자열을 `sys.stdin`에 주입. 구현: `pyodide.globals.set('_stdin_str', tc.input)` + 코드 앞에 `import sys; sys.stdin = __import__('io').StringIO(_stdin_str)` 주입 |
| stdout 캡처 | `sys.stdout` 을 `io.StringIO`로 교체하여 캡처. 실행 후 `.getvalue()` |
| 타임아웃 | 단일 케이스 5초. `Promise.race([run, timeout(5000)])` → 초과 시 verdict='incorrect', feedback.error='timeout' |
| 오류(예외) | Python 예외 발생 시 verdict='incorrect', feedback.error=예외 메시지 첫 줄 |
| 격리 | 케이스 간 전역 오염 방지: 각 케이스 실행 전 `pyodide.runPython("import importlib, sys; [sys.modules.pop(k) for k in list(sys.modules.keys()) if k not in _safe_modules]")` — 사용자 정의 모듈만 제거, 표준라이브러리 유지 |
| 허용 패키지 | 표준라이브러리 전체. `activity.grading.allowed_packages[]`에 명시된 패키지를 mount() 시 `micropip.install()`로 사전 설치. 빈 배열 또는 미설정 시 설치 없음. |

### 3-3. 비교 규칙

#### stdout-trim (기본)

```
비교: actual.trim() === expected.trim()
```

- 줄 수·개행 내부는 그대로 비교. 앞뒤 공백·개행만 무시.
- 대소문자 구분 있음.
- 부동소수를 exact match로 비교. 필요 시 expected를 `"3.14"` 처럼 고정 소수점 문자열로 작성.

#### stdout-float-tol

```
비교:
  actual_lines  = actual.trim().split('\n')
  expected_lines = expected.trim().split('\n')
  줄 수 불일치 → false
  각 줄: 토큰 분리(whitespace), 토큰 수 불일치 → false
  각 토큰:
    둘 다 숫자 → |parseFloat(a) - parseFloat(e)| ≤ tolerance
    아니면      → 문자열 exact match
```

- `activity.grading.tolerance` 미설정 시 기본값 `1e-6`.
- 여러 줄, 여러 토큰(숫자+문자열 혼합) 출력 모두 처리.

### 3-4. ScoreResult 반환 형태

플러그인계약 §4 ScoreResult conform:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect";          // passed===total ? correct : incorrect
  score_raw: number;                            // passed / total, 0..1
  grader_id: "pyodide";
  feedback: {
    passed:     number;
    total:      number;
    first_fail: {                               // null if all passed
      idx:      number;                         // 0-based 케이스 인덱스
      input:    string;
      expected: string;
      actual:   string;
    } | null;
    cases: Array<{                              // 전체 케이스 결과 배열
      idx:      number;                         // 0-based
      input:    string;
      expected: string;
      actual:   string;
      pass:     boolean;
      error?:   string;                         // "timeout" | 예외메시지 | "skipped"
    }>;
    error?: string;                             // timeout | 예외메시지(첫줄)
  };
}
```

## 4. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. coding 전용 사항:

### 4-1. mount(container, ctx)

```
1. Pyodide 로드 확인(아직 안 됐으면 loadPyodide() → 완료 대기 + 로딩 표시)
2. CodeMirror 에디터를 container에 주입
   - 초기값: activity.front.starter_code
   - 언어 모드: python (CodeMirror python 모드)
3. "실행" + "제출·채점" 버튼 렌더링 (로드 완료 전: "로딩 중..." 레이블 + disabled)
4. onProgressRestored() 호출됐으면 저장된 코드 복원(plugin_extra.last_code)
5. 테스트 케이스 패널: 입력(input)만 공개, 기대 출력(expected)은 채점 후 결과표에서 표시
   → 채점 전 expected 노출은 하드코딩 우회를 유발하므로 금지
```

### 4-2. score(userAnswer)

```ts
// userAnswer: { code: string }
// 반환: Promise<ScoreResult>
score(userAnswer): Promise<ScoreResult>
```

- `userAnswer.code` = 에디터 현재 코드.
- §3 채점 절차 실행 → ScoreResult 반환.
- 완료 후 셸에 `emit('activity-completed', result)` 방출(셸이 진도 저장).

### 4-3. getProgressSnapshot()

```ts
// PluginProgressSnapshot conform (플러그인계약 §4)
{
  plugin_id: "coding",
  schema_version: 1,
  activities: {
    [activity_id]: {
      cold_attempts: number,
      cold_correct:  number,
      last_verdict:  "correct" | "incorrect" | null,
      plugin_extra: {
        last_code:   string,    // 마지막 제출 코드
        passed:      number,
        total:       number,
        solve_ms:    number     // [mid] mount(또는 문제 전환) 이후 첫 제출까지 경과 ms
      }
    }
  }
}
```

### 4-4. getDashboardContrib() (선택 구현)

미구현 시 공통셸 기본 집계 사용(cold_correct/cold_attempts 기반 정확도). 구현 시 플러그인계약 §4 DashboardContrib 형태 conform.

## 7. mid 기능 규격 (cycle-4)

### 7-1. matplotlib 플롯 렌더링

- **적용 범위**: `실행` 버튼(Run) 전용. 채점 케이스(_grade)에는 미적용.
- **트리거 조건**: `activity.grading.allowed_packages[]`에 `"matplotlib"` 포함된 경우에만 captureplot=true.
- **백엔드**: `matplotlib.use("agg")` 강제 (브라우저 내 WebAgg 불필요). Pyodide 0.26 호환.
- **캡처 방식**: exec 후 `plt.get_fignums()` 체크 → `plt.savefig(BytesIO, format="png", dpi=96)` → base64 인코딩 → `<img data:image/png;base64,...>` 태그로 `.coding-plot-output` 영역에 표시.
- **stdout**: 플롯 외 stdout도 `.coding-output` `<pre>`에 병행 표시.
- **설치 선행 조건**: `allowed_packages: ["matplotlib"]`이면 mount()의 micropip 화이트리스트 경로에서 사전 설치됨.

### 7-2. 문제 태그별 네비 필터

- activities 2개 이상일 때 nav 상단에 `area` / `subarea` 드롭다운 표시.
- 드롭다운 선택 시 해당 태그와 일치하지 않는 nav 버튼을 `display:none` 처리.
- `area` 변경 시 `subarea` 드롭다운을 해당 area의 subarea 목록으로 재구성.
- 전체 선택(빈 값)이면 모든 버튼 표시.
- 필터 적용 중 버튼 수 표시 (`N개 표시`).

### 7-3. 풀이 시간 수집 (solve_ms)

- `mount()` 또는 문제 전환(`_applyActivity`) 호출 시 `_state.mountTime = Date.now()` 기록.
- `제출·채점` 버튼 클릭 후 채점 완료 시 `Date.now() - _state.mountTime` 계산.
- **첫 제출 시에만 기록**: `plugin_extra.solve_ms`가 이미 0이 아닌 경우 덮어쓰지 않음.
- UI: 의미있는 측정 데이터가 충분히 쌓인 후 별도 위젯으로 표시 예정. 현재 UI 비노출(빼기 미학).
- 진도: `plugin_extra.solve_ms`에 저장 (localStorage 영속).

## 8. 파킹 (v1 밖)

| 항목 | 이유 |
