## 계약 의존

> SoT 변경 동반 항목 — 사용자 승인 선행 필수. 승인 전 해당 기능은 **계약대기** 표시.

| # | 기능 | SoT 변경 내용 | 상태 |
|---|---|---|---|
| C1 | **전체 케이스 결과표 + 케이스 인덱스** | `플러그인계약.md §4` ScoreResult.feedback에 `cases: [{idx, input, expected, actual, pass}]` 배열 추가. 현재 first_fail 단건만 정의 → 다건 배열 확장. coding·excel 공통 영향 | **완료** — 런타임규격 §3-4 cases[] 추가, plugin.js _grade()·_showResult() 구현 완료 |
| — | **stdout-float-tol 비교 모드** | `런타임규격.md §2-1` grading.compare `const:"stdout-trim"` → enum 확장(`"stdout-float-tol"` 추가). 기계검증 V4 재정의 동반 | **완료** — 런타임규격 compare enum + tolerance 필드 추가, plugin.js _compare() 구현 완료 |

> **killer_point 주의**: stdout-float-tol(런타임규격 const 변경)·전체 케이스 결과표(플러그인계약 ScoreResult 확장) 둘 다 SoT 변경 비용을 내포함. 이데이션에서 "규격 enum만 추가하면 완성" / "추가 API 불필요"로 기술했으나 **코어계약 변경 게이트 통과 필수**. C1 승인 전 구현 착수 금지.

## 구현 순서 (사용자 참고)

1. back 해설 패널 — 코드 없이 렌더만, 즉시 학습루프 완성
2. 실행전용 버튼 — 기존 채점 인프라 재사용, S 규모
3. micropip 화이트리스트 — matplotlib·과학 문제 선행 조건 확보
4. **C1 계약 승인** → 전체 케이스 결과표 + 케이스 인덱스 구현
5. stdout-float-tol SoT 승인 → 비교 모드 확장
6. mid 기능은 위 완료 후 단계 진입 (한 번에 다구현 금지)
