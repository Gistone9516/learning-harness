## 3. 콘텐츠 소스 — fk_슬라이더실습.md 작성 규칙

### 3-1. 파일 구조 (YAML front-matter + 본문)

```yaml
---
plugin_id:   robot-arm
activity_id: <자동생성 또는 명시 — §3-2 참조>
weight:      <1..10, 기본 5>
tags:
  area:    로봇공학
  subarea: 기구학
  unit:    <단원명>
enabled: true
---

## 과제 설명 (front.prompt)
<학습자에게 보여줄 안내 문자열. 1~3문장.>

## 링크 구성 (front.links)
```yaml
links:
  - length: <number>
  - length: <number>
  # 2~4개, MVP는 2~3개 권장
```

## 목표점 (front.target)
```yaml
target:
  x: <number>
  y: <number>
```

## 정답 허용 거리 (front.tolerance)
tolerance: <number>   # 미지정 시 config.default_tolerance_ratio × Σlinks.length

## 관절 한계 (front.joint_limits)  — links와 동일 길이, 라디안
```yaml
joint_limits:
  - [-1.5707963, 1.5707963]   # ≈ [-π/2, π/2]
  - [-1.5707963, 1.5707963]
```

## 해설 (back)
**왜(why):** <핵심 학습 포인트 1~2문장>
**설명(explanation):** <상세 개념 보충>
**예시 정답 각도 (선택):** [<rad>, <rad>]   # example_solution_angles
```

### 3-2. activity_id 생성 규칙

- 패턴: `robot-fk-<unit_slug>-<3자리 순번>` (예: `robot-fk-2link-001`)
- 정규식: `^[a-z][a-z0-9-]{2,63}$` (플러그인계약 §2, _인터페이스계약 §1 동일 패턴)
- 단위: `tags.unit`을 소문자 알파벳+하이픈으로 정규화. 예: "정기구학(FK)" → "fk"
- 생성 후 **deck 범위 내 중복 금지** (_인터페이스계약 §1 — 전역 유일성 강제 안 함).

### 3-3. tolerance 자동계산 (미지정 시)

```python
# generate.py 내부 로직
link_total = sum(link['length'] for link in front['links'])
tolerance  = link_total * config['robot_arm']['default_tolerance_ratio']  # 기본 5%
```

### 3-4. joint_limits 기본값 (미지정 시)

```python
# links 개수만큼 기본 [-π/2, π/2] 적용
import math
joint_limits = [[-math.pi/2, math.pi/2]] * len(front['links'])
```

### 3-5. 작성 제약 (검증 선행)

| 항목 | 규칙 |
|---|---|
| links 개수 | 2 ≤ n ≤ 4. MVP 권장: 2~3 |
| links[i].length | > 0. 단위 일관성 필수(모든 links, target, tolerance 동일 단위) |
| tolerance | > 0. 권장 범위: 전체 링크 길이의 3~10% |
| joint_limits | links와 동일 길이. min < max 필수 (라디안) |
| target 도달 가능성 | Σ links[i].length ≥ sqrt(target.x² + target.y²) 필수 (생성 시 자동 검증) |
| weight | [1, 10]. 기본 5 |
| enabled | 기본 true |

## 4. generate.py 파이프라인 (robot-arm 전용 경로)

```
입력:  과목/robot/콘텐츠/문제/fk_슬라이더실습.md (+ 추가 md)
       과목/robot/config/규칙.json
출력:  과목/robot/생성물/activities-robot-arm.js
       과목/robot/생성물/manifest.js (subject = 'robot')
```

### 4-1. 파싱 절차

```python
import yaml, re, math, json

def parse_robot_activity(md_path, config):
    text = open(md_path, encoding='utf-8').read()

    # front-matter 추출
    fm_match = re.match(r'^---\n(.*?)\n---\n', text, re.DOTALL)
    fm = yaml.safe_load(fm_match.group(1))

    # 각 섹션 파싱 (## 헤딩 기준)
    sections = _parse_sections(text)

    # front 조립
    links        = sections['links']             # [{length:...}, ...]
    target       = sections['target']            # {x:..., y:...}
    tolerance    = sections.get('tolerance') or _auto_tolerance(links, config)
    joint_limits = sections.get('joint_limits') or _default_joint_limits(links)
    prompt       = sections.get('과제 설명', '').strip()

    # 검증 (§4-2)
    _validate(links, target, tolerance, joint_limits)

    # activity_id 생성 (front-matter 명시 우선)
    activity_id = fm.get('activity_id') or _gen_activity_id(fm)

    # ActivitySpec 조립
    return {
        'activity_id':  activity_id,
        'plugin_id':    'robot-arm',
        'type':         'robot-fk',
        'weight':       fm.get('weight', config['robot_arm']['default_weight']),
        'tags':         fm['tags'],
        'enabled':      fm.get('enabled', True),
        'front': {
            'prompt':       prompt,
            'links':        links,
            'target':       target,
            'tolerance':    tolerance,
            'joint_limits': joint_limits
        },
        'back': _parse_back(sections)   # 없으면 None → 출력에서 생략
    }

def _validate(links, target, tolerance, joint_limits):
    assert 2 <= len(links) <= 4, f"links 개수 범위 위반: {len(links)}"
    for i, lk in enumerate(links):
        assert lk['length'] > 0, f"links[{i}].length <= 0"
    assert tolerance > 0, "tolerance <= 0"
    assert len(joint_limits) == len(links), "joint_limits 길이 != links 길이"
    for i, jl in enumerate(joint_limits):
        assert jl[0] < jl[1], f"joint_limits[{i}]: min >= max"
    # 도달 가능성
    max_reach = sum(lk['length'] for lk in links)
    dist = math.sqrt(target['x']**2 + target['y']**2)
    assert max_reach >= dist, (
        f"목표점({target}) 도달 불가: Σlinks={max_reach:.2f} < dist={dist:.2f}"
    )
```

## 5. MVP 콘텐츠 분량 (원칙1 — 빼기 미학)

| 구분 | 목표 수량 | 비고 |
|---|---|---|
| robot-arm ActivitySpec | 3~5개 | 각 링크 수, 목표 위치 다양화 |
| card-quiz 개념카드 | 6~8장 | 개념서 3~5섹션 기반 |

### 5-1. ActivitySpec 예시 3개 (MVP 구성)

|---|---|---|---|
