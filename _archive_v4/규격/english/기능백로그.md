## core

| 기능 | 동작 | infra | 공수 | 검토 반영·근거 |
|---|---|---|---|---|
| **6모달 정적 채점 완전 동작** (vocab/grammar/reading/listening) | exact·keyword·cloze·dictation-diff 4채점기 + 피드백 렌더 완전 동작. writing/speaking은 키 없으면 pending graceful. mount 항상 성공 | static(정적 4모달) + byok(writing/speaking) | M | 원칙1·2: 학습루프 닫힘의 전제. speaking은 **본 티켓에서 분리** — stub=graceful 정상 동작으로 6모달 차단 아님 |
| **llm_endpoint·llm_model byok 필드 추가** (manifest 선행 계약) | `manifest.js` byok.keys에 `{id:"llm_endpoint",...}`, `{id:"llm_model",...}` 추가. 셸이 이 선언을 읽어 settings UI 자동 생성 — 이 필드 없으면 사용자가 엔드포인트 변경 경로 없음 | static | S | 검토 `missing_high_value`: writing LLM 유연화의 **선행 독립 조건**. 셸 settings UI 자동생성 불가 막음. byok 계약 §6 conform |
| **writing LLM 엔드포인트 유연화** | plugin.js 297줄 `api.openai.com + gpt-4o-mini` 하드코딩 제거. `ctx.getKey("llm_endpoint")` + `ctx.getKey("llm_model")` 참조. OpenAI·Anthropic·로컬 ollama/vLLM 수용. 점수 0-10 → score_raw, 한국어 피드백 렌더링 완성 | byok | M | 원칙2: 규격 정의·키 선언 있으나 하드코딩 방치. OpenAI 계정 없는 사용자 writing 완전 차단 해소. **llm_endpoint byok 필드 추가 선행 필수** |
| **dictation_threshold 문항별 재정의** | plugin.js 263줄 `threshold=0.9` 하드코딩 → `activity.grading.dictation_threshold ?? 0.9` 로직 1줄 수정. | **SM-2 스케줄러 + due_at mount 정렬** (결합 구현) | vocab/grammar 활동에 SM-2 알고리즘 적용 — 오답 간격 단축·정답 간격 확장. `plugin_extra.due_at`, `interval`, `efactor` 저장. **mount() 진입점에 due_at 기준 activities 정렬 로직 포함** — 분리 시 절반 구현, due_at이 학습 흐름 미반영 | static | M | 방향성 §4: "due_at mount 정렬 결합 포함해야 core(분리 시 절반구현)". 콘텐츠 데이터 선행 조건(목업 ActivitySpec 수개 먼저). SM-2 JS 순수 구현(CDN 불요) — 차용우선 원칙5 conform |

## mid

| 기능 | 동작 | infra | 공수 | 검토 반영·근거 |
|---|---|---|---|---|
| **listening TTS 재생 속도 제어** | listening 모달에 속도 버튼(0.75× / 1.0× / 1.25×) 추가. `SpeechSynthesisUtterance.rate` 파라미터 적용. 마지막 선택값 `plugin_extra`에 저장 | static | S | 이데이션 mid 유지. 받아쓰기 난이도 핵심 변수. SpeechSynthesis 내장 파라미터 — 추가 API 없음. 빼기 미학: 버튼 3개로 한정 |
| **오답 노트 위젯** (getDashboardContrib 강화) | incorrect 누적 시 `plugin_extra.wrong_notes[]`(최대 20개)에 `back.explanation` 저장. getDashboardContrib `extra_widgets`에 "자주 틀리는 표현 TOP5" 위젯 추가 | static | M | 이데이션 mid 유지. `plugin_extra:object` 확장 = 플러그인계약 §4 허용 범위. self-explanation 효과. C4 계약대기 **아님** — back.explanation은 현재 스키마에 존재 |
| **writing 제출 히스토리 저장** | 채점 후 `plugin_extra.history[]`(최대 5회 순환)에 `{answer, feedback, score_raw, ts}` 저장. UI에 이전 제출·피드백 타임라인 표시 | byok | M | 이데이션 mid 유지. writing 반복 수정이 핵심 학습 루프. **C5 경고**: 수백자 × 5회 × N문항 누적 시 localStorage 5MB 접근 — 셸 quota 가드(C5) 완료 후 안전 확인 필요. history 크기 상한(5회 순환) 명시로 위험 제한 |
