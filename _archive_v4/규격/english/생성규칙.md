# english 플러그인 생성규칙

> buildflow cycle-2 ②명세. 대상: LearningProfile(깊이스무고개.md STEP 2 산출) → `lang-task` ActivitySpec 다수 생성하는 Claude.
> 런타임규격.md conform — ActivitySpec 형태·채점 모드는 런타임규격 §2 SoT.

---

## 0. 생성 전 필수 절차

### 0-1. 웹그라운딩 의무 (환각·오역 방지)

**영어 예문·어휘·문법 설명은 임베딩 단정 금지.**
생성 시작 전 반드시 웹서치로 확인:

| 항목 | 서치 대상 |
|---|---|
| 단어 정의·용례 | Merriam-Webster, Cambridge Dictionary, Longman |
| 문법 규칙 | 공신력 있는 ESL 문법 레퍼런스(British Council, Purdue OWL 등) |
| 숙어·구동사 | 사전 or Collins COBUILD |
| 독해 지문 출처 | 실제 텍스트 인용 시 출처 명시; 창작 지문은 오류 없음 자체 검토 |
| 발음 기준(IPA) | Cambridge 또는 Merriam-Webster 발음 표기 |

**출처 없는 예문·번역·문법 설명은 생성 불가.** 불확실하면 웹서치 후 확인된 내용만 사용.

### 0-2. LearningProfile 입력 검증

```
입력: LearningProfile (깊이스무고개.md §STEP2)
필수 필드:
  subject = "english" (또는 영어 관련 과목명)
  estimated_level: string       // 진단 기반 레벨
  goal: enum                    // 시험/실무/교양/특정스킬
  scope: string[]               // 대상 토픽(문법영역, 어휘주제 등)
  depth_ratio: {theory, practice}
  volume: string                // 문항 수 또는 분량 기준
  target_plugins: ["english"]
```

---

## 1. 모달 선택 규칙

`LearningProfile.scope` + `depth_ratio` + `goal`로 모달 배분 결정:

| 조건 | 권장 모달 |
|---|---|
| goal = 시험(토익·수능·공인영어) | vocab(40%) + grammar(30%) + reading(20%) + listening(10%) |
| goal = 실무·프로젝트(비즈니스 영어) | vocab(25%) + writing(35%) + speaking(20%) + reading(20%) |
| goal = 교양·이해 | reading(40%) + vocab(30%) + listening(20%) + grammar(10%) |
| goal = 특정스킬(어휘만, 문법만 등) | scope 명시 모달 100% |
| depth_ratio.practice > 0.6 | writing/speaking 비중 ↑ |
| depth_ratio.theory > 0.6 | grammar/reading/vocab 비중 ↑ |

**BYO-key 모달(writing/speaking)은 사용자가 키를 설정한 경우에만 주력 배분.** 키 미설정 상태에서도 생성은 하되(graceful mount), 배분 비중은 키 보유 여부 확인 후 조정.

## 2. 모달별 ActivitySpec 작성 규칙

### 공통 규칙 (전 모달)

1. **activity_id 패턴**: `eng-{modality_prefix}-{unit_slug}-{seq:03d}` 예: `eng-vcb-phrasal-001`
   - modality prefix: vcb/grm/rdg/lst/wrt/spk
   - unit_slug: 영소문자+하이픈, 최대 20자
2. **weight**: 쉬운 문제 1-3, 중간 4-6, 어려운 7-10. 레벨별 분포:
   - beginner: 1-3 위주
   - intermediate: 3-6 위주
   - advanced: 6-9 위주
3. **tags.area**: 모달명 소문자 (vocabulary / grammar / reading / listening / writing / speaking)
4. **tags.subarea**: 토픽 분류 (예: phrasal-verb, present-perfect, main-idea, dictation)
5. **tags.unit**: scope에서 가져온 단위 슬러그
6. **enabled**: 기본 `true`
7. **back.explanation**: 한국어 해설 허용(학습자가 한국인인 경우). 단 예문·정답은 영어 원문 유지.

---

### 2-1. vocab 모달

**grading.mode**: `exact` (단답형) 또는 `exact`+`options`(객관식)

```
front.modality = "vocab"
front.prompt   = 영어 뜻풀이 or 문맥 문장 + "What word/phrase is this?"
                 OR 한국어 뜻 + "Choose the correct English word."
front.options  = ["단어1","단어2","단어3","단어4"]  // 객관식 시 4지선다 권장
back.answer    = 정답 단어
back.explanation = 어원·용례·유의어·주의사항 (웹그라운딩 출처 반영)
grading.mode   = "exact"
grading.accept = [정답, 정답_복수형, 정답_축약형]  // 허용 표기 변형 전부 열거
```

**작성 제약**:
- 정의문은 Cambridge/Merriam-Webster 실제 정의 참조. 임의 재작성 금지.
- 객관식 오답(distractors)은 의미가 유사하거나 혼동 빈도 높은 실제 단어로 구성. 임의 날조 금지.
- accept[]에 주요 변형(복수형, 동사 원형, -ing 형 등) 포함.

**예시**:
```json
{
  "activity_id": "eng-vcb-collocations-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 3,
  "tags": { "area": "vocabulary", "subarea": "collocation", "unit": "collocations" },
  "enabled": true,
  "front": {
    "modality": "vocab",
    "prompt": "Choose the word that correctly completes the collocation: 'make a ___'",
    "options": ["decision","opinion","believe","argue"]
  },
  "back": {
    "answer": "decision",
    "explanation": "'make a decision'은 영어의 고빈도 연어(collocation). 'make an opinion'은 비문법적 — 올바른 표현은 'form/hold an opinion'."
  },
  "grading": {
    "mode": "exact",
    "accept": ["decision"]
  }
}
```

### 2-2. grammar 모달

**grading.mode**: `exact` (선택지/단답) 또는 `cloze` (빈칸 채우기)

```
front.modality = "grammar"
front.prompt   = 문법 개념 + 빈칸 문장 or 틀린 문장 교정 지시
                 OR 선택지 중 문법적으로 맞는 것 고르기
front.options  = 객관식 시 사용
back.answer    = 정답 형태
back.explanation = 문법 규칙 설명 (규칙명 + 예시) — Purdue OWL/British Council 등 출처 반영
grading.mode   = "exact"  // 또는 "cloze"
grading.accept = [정답, 허용 변형]
```

**cloze 규칙**:
- `front.prompt`에 빈칸을 `___`로 표시.
- `grading.accept`에 허용 정답 전부 열거 (예: `["had gone","had been gone"]`).
- 빈칸 1개 원칙(복수 빈칸 시 activity 분리).

**작성 제약**:
- 문법 규칙은 공신력 있는 레퍼런스 확인 후 작성. 예외·변형도 명시.
- 예문은 자연스러운 실제 용례 사용. 교과서 수준으로 단순화해도 되나 비문(ungrammatical) 예문 예시로 사용 시 명확히 표시.

**예시**:
```json
{
  "activity_id": "eng-grm-pres-perf-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 4,
  "tags": { "area": "grammar", "subarea": "present-perfect", "unit": "tense-aspect" },
  "enabled": true,
  "front": {
    "modality": "grammar",
    "prompt": "Complete the sentence: 'She ___ in London since 2019.'",
    "options": ["lives","lived","has lived","is living"]
  },
  "back": {
    "answer": "has lived",
    "explanation": "현재완료(have/has + p.p.)는 과거 시점부터 현재까지 지속되는 상태에 사용. 'since 2019'가 기준점을 나타내므로 현재완료가 적합."
  },
  "grading": {
    "mode": "exact",
    "accept": ["has lived"]
  }
}
```

### 2-3. reading 모달

**grading.mode**: `keyword` (주관식 답) 또는 `exact` (객관식)

```
front.modality  = "reading"
front.passage   = 독해 지문 (100-400단어. 실제 출처 명시 또는 창작 명시)
front.prompt    = 지문에 관한 질문
front.options   = 객관식 시 사용
back.answer     = 모범 답안 (주관식) or 정답 선택지
back.explanation = 지문 내 근거 문장 인용 + 해설
grading.mode    = "keyword"  // 주관식
grading.keywords = [[핵심단어1a, 핵심단어1b], [핵심단어2]]  // AND 그룹
              OR
grading.mode    = "exact"   // 객관식
grading.accept  = [정답 선택지 텍스트]
```

**passage 작성 제약**:
- 실제 글 인용 시: 출처(작가/URL/출판년도) `back.explanation` 또는 별도 주석에 명시.
- 창작 지문: 사실 관계 오류 0. 불확실한 사실 포함 금지.
- 지문 길이: beginner 100-150자, intermediate 200-300자, advanced 300-400자.
- 질문 유형: 주제(main idea) / 세부사항(detail) / 추론(inference) / 어휘(vocabulary in context) 고루 분배.

**keyword 그룹 설계 원칙**:
- 각 그룹 = 필수 개념 1개. OR 요소 = 동의어/다른 표현.
- 예: `[["economy","economic"],["decline","fall","slowdown"]]` → 경제 관련 + 하락 관련 단어 둘 다 포함해야 correct.

**예시**:
```json
{
  "activity_id": "eng-rdg-main-idea-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 5,
  "tags": { "area": "reading", "subarea": "main-idea", "unit": "environment" },
  "enabled": true,
  "front": {
    "modality": "reading",
    "passage": "Deforestation is one of the leading causes of climate change. When forests are cleared, the carbon stored in trees is released into the atmosphere as CO2. This contributes to the greenhouse effect and rising global temperatures. Protecting and restoring forests is therefore essential to any serious climate strategy.",
    "prompt": "What is the main idea of this passage?"
  },
  "back": {
    "answer": "Deforestation contributes to climate change, and forest protection is essential.",
    "explanation": "핵심 주장: 산림 벌채 → CO2 방출 → 기후 변화. 해결책: 산림 보호·복원. 근거 문장: 'Protecting and restoring forests is therefore essential to any serious climate strategy.'"
  },
  "grading": {
    "mode": "keyword",
    "keywords": [
      ["deforestation","forest","forests"],
      ["climate","carbon","CO2","greenhouse"],
      ["protect","restore","protection","conservation"]
    ]
  }
}
```

### 2-4. listening 모달

**grading.mode**: `dictation-diff` (기본) 또는 `exact` (단답/객관식)

```
front.modality  = "listening"
front.prompt    = 받아쓰기 지시 or 질문 ("Listen and type what you hear." 또는 "Listen and answer:")
front.audio_text = TTS로 재생할 영어 원문 (단어 10-50개 권장. TTS 발화에 적합한 자연스러운 문장)
front.options   = 객관식 청취 시 사용 (선택지 별도 제공)
grading.mode    = "dictation-diff"
grading.expected = audio_text와 동일 (생략 가능 — 런타임이 audio_text 사용)
               OR 단답/객관식이면 "exact", grading.accept 사용
```

**audio_text 작성 제약**:
- TTS 발화에 적합: 약어/기호 최소화. 숫자는 영어 표기(`thirty-five` not `35`).
- 길이: beginner 1문장(10-15단어), intermediate 2-3문장(20-35단어), advanced 3-4문장(35-50단어).
- 자연스러운 구어체. 문어체(formal written English)는 TTS 발화 시 어색할 수 있으므로 피함.
- audio_text에 포함된 사실 정보(날짜, 지명, 수치)는 웹그라운딩으로 검증.

**예시**:
```json
{
  "activity_id": "eng-lst-dictation-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 3,
  "tags": { "area": "listening", "subarea": "dictation", "unit": "daily-conversation" },
  "enabled": true,
  "front": {
    "modality": "listening",
    "prompt": "Listen carefully and type exactly what you hear.",
    "audio_text": "The meeting has been rescheduled to three o'clock on Friday afternoon."
  },
  "back": {
    "answer": "The meeting has been rescheduled to three o'clock on Friday afternoon.",
    "explanation": "reschedule = 일정 변경. 'three o'clock' — 시간 표현 숫자 대신 영어 단어로 발화됨에 주의."
  },
  "grading": {
    "mode": "dictation-diff",
    "expected": "The meeting has been rescheduled to three o'clock on Friday afternoon."
  }
}
```

### 2-5. writing 모달 (BYO-key, grader_id="llm")

**grading.mode**: `llm-rubric`

```
front.modality  = "writing"
front.prompt    = 글쓰기 과제 지시문 (목적·형식·분량 명시)
grading.mode    = "llm-rubric"
grading.rubric  = LLM에 전달할 채점 기준 (구체·수치화. 아래 템플릿 사용)
back.explanation = 모범 답안 또는 핵심 포인트 (학습자 참고용)
```

**rubric 템플릿**:
```
Score 0-10. Criteria:
- Task completion (0-3): Does the response address the prompt fully?
- Grammar & vocabulary (0-3): Are sentences grammatically correct? Is vocabulary appropriate?
- Coherence & cohesion (0-2): Is the writing logically organized with clear connectors?
- Style & register (0-2): Is the tone appropriate for the task (formal/informal)?
Return JSON: {"score": <0-10>, "feedback": "<2-3 sentence comment in Korean>"}
```

**작성 제약**:
- 루브릭은 수치 기준(0-N점) 포함. "좋으면 점수 높게" 식의 애매한 기준 금지.
- 지시문(prompt)에 글의 목적·대상·형식·분량 명시(예: "Write a 3-sentence email to a colleague...").

**예시**:
```json
{
  "activity_id": "eng-wrt-email-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 6,
  "tags": { "area": "writing", "subarea": "email", "unit": "business-writing" },
  "enabled": true,
  "front": {
    "modality": "writing",
    "prompt": "Write a short professional email (3-4 sentences) to a colleague asking to reschedule a meeting from Tuesday to Thursday at 2 PM."
  },
  "back": {
    "explanation": "핵심 요소: 사유 언급(brief reason), 새 시간 제안(specific new time), 확인 요청(confirmation request). 예: 'Hi [Name], I hope you are well. Would it be possible to move our Tuesday meeting to Thursday at 2 PM? Please let me know if that works for you.'"
  },
  "grading": {
    "mode": "llm-rubric",
    "rubric": "Score 0-10. Criteria: Task completion (0-3): mentions rescheduling + new time Thursday 2PM. Grammar & vocabulary (0-3): professional tone, no major errors. Coherence (0-2): polite opening/request/closing. Style (0-2): appropriate business email register. Return JSON: {\"score\": <0-10>, \"feedback\": \"<2-3 sentence comment in Korean>\"}"
  }
}
```

### 2-6. speaking 모달 (BYO-key, grader_id="external")

**grading.mode**: `pronunciation`

```
front.modality  = "speaking"
front.prompt    = 발음 연습 지시문 + 읽어야 할 텍스트
grading.mode    = "pronunciation"
grading.expected = 발음 채점 기준 문장 (Azure Speech referenceText)
grading.rubric  = 발음 포인트 설명 (선택. 학습자 안내용)
back.explanation = 발음 주의 포인트, IPA 표기 (Cambridge/Merriam-Webster 출처)
```

**expected 작성 제약**:
- Azure Speech Pronunciation Assessment의 `referenceText`로 직접 사용됨. 철자 100% 정확.
- IPA 표기는 Cambridge 사전 British English 또는 Merriam-Webster American English 기준 명시.
- 발음 포인트가 있는 단어(동음이의어·묵음·연음)는 back.explanation에 설명.

**예시**:
```json
{
  "activity_id": "eng-spk-minimal-pair-001",
  "plugin_id": "english",
  "type": "lang-task",
  "weight": 5,
  "tags": { "area": "speaking", "subarea": "minimal-pair", "unit": "vowel-sounds" },
  "enabled": true,
  "front": {
    "modality": "speaking",
    "prompt": "Read the following sentence aloud clearly: 'I will live in this village and live a simple life.'"
  },
  "back": {
    "explanation": "live (동사, /lɪv/) vs live (형용사, /laɪv/) — 동일 철자 다른 발음. 첫 번째 'live' = /lɪv/ (거주하다), 두 번째 'live' = 여기서는 /lɪv/ (단순한 삶). 'village' /ˈvɪlɪdʒ/ — 묵음 없음, 두 번째 음절 약모음(/ɪ/). [출처: Cambridge Dictionary]"
  },
  "grading": {
    "mode": "pronunciation",
    "expected": "I will live in this village and live a simple life.",
    "rubric": "Focus on vowel sounds: /lɪv/ in both instances of 'live'. Clear articulation of /v/ in 'village'."
  }
}
```

## 3. 배치 생성 템플릿

LearningProfile 1개 → `target_plugins: ["english"]`인 경우 아래 순서로 생성:

```
STEP A. scope 토픽 → 모달 배분 결정 (§1 모달 선택 규칙 적용)
STEP B. 각 토픽·모달 조합에 대해 웹그라운딩 (§0-1 의무)
STEP C. ActivitySpec 생성 (§2 모달별 규칙 준수)
         - 문항 수: volume 파라미터 기준
           예: "30문항" → 모달 배분 비율 × 30 = 각 모달 문항 수
         - 문항당 difficulty 분포: estimated_level 기준 (§2 공통 weight 규칙)
STEP D. JSON Schema validate (런타임규격.md §2-1)
STEP E. 결과: ActivitySpec[] + grounding_sources[]
```

**결과물 형태**:
```json
{
  "plugin_id": "english",
  "generated_at": "<ISO timestamp>",
  "profile_summary": {
    "level": "<estimated_level>",
    "scope": ["<topic1>", "..."],
    "volume": "<N>문항",
    "modal_distribution": {"vocab": N, "grammar": N, "reading": N, "listening": N, "writing": N, "speaking": N}
  },
  "grounding_sources": [
    {"topic": "<단어/규칙/지문>", "source": "<URL or 사전명>", "verified": true}
  ],
  "activities": [ /* ActivitySpec[] */ ]
}
```
