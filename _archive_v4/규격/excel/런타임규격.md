## 1. PluginManifest 선언

```js
// 파일: plugins/excel/manifest.js
window.MANIFEST_EXCEL = {
  plugin_id:               "excel",
  label:                   "엑셀 실습",
  version:                 "1.0.0",
  infra:                   "static",      // Univer CDN, 서버·키 불요
  entry_url:               null,
  byok:                    null,
  capabilities:            ["sheet"],
  scoring_mode:            "auto",
  activity_type:           "sheet-task",
  progress_schema_version: 1
};
```

**infra = static 근거**: Univer 스프레드시트 엔진(@univerjs/presets + @univerjs/preset-sheets-core)을 CDN(unpkg)에서 UMD로 로드하고, 채점 전부 브라우저 내 Facade API 실행. 서버·키·외부 API 0.

### 2-2. target_cells ↔ expected 정합 불변식

- `front.target_cells.length === grading.expected.length` (기계 검증 V5)
- `grading.expected[i].cell === front.target_cells[i]` 순서 일치 권장 (채점 루프 단순화)
- `front.target_cells`에 없는 셀을 `grading.expected[].cell`에 넣는 것 금지

### 2-3. 최소 예시

### 3-2. Univer 인스턴스 생성 및 초기 데이터 주입

```js
// UMD 전역: UniverPresets, UniverCore, UniverPresetSheetsCore
const { createUniver }             = UniverPresets;
const { LocaleType, mergeLocales } = UniverCore;
const { UniverSheetsCorePreset }   = UniverPresetSheetsCore;

/**
 * Univer 스프레드시트를 container에 마운트하고 초기 데이터를 주입.
 * @param {HTMLElement} container
 * @param {Array<Array<string|number>>} initialGrid  — activity.front.initial_grid
 * @returns {{ univerAPI: object, fWorkbook: object }}
 */
function mountUniver(container, initialGrid) {
  const { univerAPI } = createUniver({
    locale: LocaleType.EN_US,
    locales: { [LocaleType.EN_US]: mergeLocales(UniverPresetSheetsCoreEnUS) },
    presets: [UniverSheetsCorePreset({ container })],
  });

  // initial_grid(2D 배열) → IWorkbookData.sheets[0].cellData 변환
  const cellData = {};
  initialGrid.forEach((row, rowIdx) => {
    cellData[rowIdx] = {};
    row.forEach((val, colIdx) => {
      if (val !== "" && val !== null && val !== undefined) {
        cellData[rowIdx][colIdx] = typeof val === "number"
          ? { v: val, t: 2 }     // t:2 = number
          : { v: val, t: 1 };    // t:1 = string
      }
    });
  });

  const fWorkbook = univerAPI.createWorkbook({
    id:         "sheet-task-wb",
    name:       "실습 시트",
    sheetOrder: ["sheet1"],
    sheets: {
      sheet1: {
        id:          "sheet1",
        name:        "Sheet1",
        rowCount:    Math.max(initialGrid.length + 10, 20),
        columnCount: Math.max((initialGrid[0] || []).length + 5, 10),
        cellData
      }
    }
  });

  return { univerAPI, fWorkbook };
}
```

## 4. Univer Facade 채점 인터페이스

### 4-1. 채점 절차 (compare = "value")

```
채점(activity, univerAPI):
  fWorkbook  ← univerAPI.getActiveWorkbook()
  fWorksheet ← fWorkbook.getActiveSheet()
  passed ← 0
  first_fail ← null
  for each expected_entry in activity.grading.expected:
    fRange     ← fWorksheet.getRange(expected_entry.cell)   // A1 표기
    actual_val ← fRange.getValue()                          // 계산된 값(수식 포함)
    ok ← compareValue(actual_val, expected_entry.value)     // §4-2
    if ok: passed++
    else if first_fail is null:
      first_fail ← { cell: expected_entry.cell,
                      expected: expected_entry.value,
                      actual: actual_val }
  total ← len(activity.grading.expected)
  verdict ← (passed === total) ? "correct" : "incorrect"
  return ScoreResult(verdict, passed/total, first_fail)
```

### 4-2. value 비교 규칙

```
compareValue(actual, expected, tolerance=0):
  eps = tolerance (grading.tolerance 또는 0)
  // 1. 둘 다 number → |actual - expected| ≤ eps (eps=0이면 exact)
  if typeof expected === 'number' && typeof actual === 'number':
    return |actual - expected| <= eps
  // 2. expected number, actual string → 파싱 후 epsilon 비교
  if typeof expected === 'number' && typeof actual === 'string':
    parsed = Number(actual.trim())
    if !isNaN(parsed): return |parsed - expected| <= eps
  // 3. 그 외 → String 변환 후 trim 비교
  return String(actual).trim() === String(expected).trim()
```

- 대소문자 구분 있음.
- 부동소수 허용오차: `grading.tolerance` 필드로 절대 오차 지정. 예: `"tolerance": 0.001`.
  또는 `grading.compare = "value-tol"` + tolerance 병기 방식 모두 지원.
- 생성규칙 E6 대로 ROUND(expr, n) 적용 시 tolerance 불요(권장).

### 4-3. 수식 계산 완료 대기 (formulaEngine)

수식이 있는 셀은 Univer의 비동기 계산 완료 후 getValue()가 올바른 값을 반환함.

```js
/**
 * 수식 계산 완료를 기다린 뒤 채점 실행.
 * formulaEngine.calculationEnd 콜백 내에서 getValue() 호출.
 * @param {object} activity
 * @param {object} univerAPI
 * @returns {Promise<ScoreResult>}
 */
async function scoreAfterCalc(activity, univerAPI) {
  return new Promise((resolve) => {
    const formulaEngine = univerAPI.getFormula?.() ?? null;
    if (!formulaEngine) {
      // 수식 없는 과제: 즉시 채점
      resolve(runScore(activity, univerAPI));
      return;
    }
    formulaEngine.calculationEnd((state) => {
      // state === 3 → 계산 완료(Univer 내부 상태코드)
      if (state === 3) {
        resolve(runScore(activity, univerAPI));
      }
    });
    // 5초 타임아웃: state===3 미발화 시 verdict='timeout' ScoreResult 반환
    // feedback.timeout = true → _showResult가 "수식 계산 실패, 수식을 확인하세요" 안내 렌더
    setTimeout(() => {
      if (!settled) {
        settled = true;
        resolve({ verdict: 'timeout', score_raw: 0, grader_id: 'engine',
                  feedback: { passed: 0, total: expected.length, first_fail: null,
                              cell_results: [], timeout: true } });
      }
    }, 5000);
  });
}
```

> **주의**: `getValue()`는 수식 셀에서 `cellData.v`(계산 결과)를 반환. 수식 문자열이 필요하면 `getFormula()` 사용.

### 4-4. ScoreResult 반환 형태

플러그인계약 §4 ScoreResult conform:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect" | "timeout";  // timeout = calculationEnd 5초 무응답
  score_raw: number;                    // passed / total, 0..1
  grader_id: "engine";                  // 정적 값 비교 채점기
  feedback: {
    passed:     number;
    total:      number;
    first_fail: {                        // null if all passed
      cell:     string;                  // 예: "B7"
      expected: string | number;
      actual:   string | number | null;
    } | null;
    cell_results: Array<{               // 다중셀 결과표 — 전체 셀 pass/fail 배열
      cell:     string;
      expected: string | number;
      actual:   string | number | null;
      ok:       boolean;
    }>;
    timeout?: boolean;                  // calculationEnd 5초 타임아웃 시 true
  };
}
```

## 5. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. excel 전용 사항:

### 5-1. mount(container, ctx)

```
1. container에 Univer 스프레드시트 마운트 (§3-2 mountUniver 호출)
2. activity.front.initial_grid를 IWorkbookData cellData로 변환하여 주입
3. activity.front.prompt를 시트 위 지문 영역(또는 별도 div)에 표시
4. "채점" 버튼 렌더링
5. onProgressRestored() 호출됐으면 저장된 userSnapshot 복원
```

### 5-2. score(userAnswer)

```ts
// userAnswer: void (Univer 내부 상태에서 직접 읽음) 또는 { snapshot?: IWorkbookData }
// 반환: Promise<ScoreResult>
score(userAnswer?): Promise<ScoreResult>
```

- `univerAPI.getActiveWorkbook().getActiveSheet().getRange(cell).getValue()`로 각 target_cell 값 읽기.
- §4-3 수식 계산 완료 대기 후 채점.
- 완료 후 셸에 `ctx.emit('activity-completed', result)` 방출.

### 5-3. getProgressSnapshot()

```ts
// PluginProgressSnapshot conform (플러그인계약 §4)
{
  plugin_id: "excel",
  schema_version: 1,
  activities: {
    [activity_id]: {
      cold_attempts: number,
      cold_correct:  number,
      last_verdict:  "correct" | "incorrect" | null,
      plugin_extra: {
        last_score:    { passed: number, total: number },
        cell_snapshot: IWorkbookData | null  // getSnapshot() 지원 시 저장 (mid 구현 완료)
      }
    }
  }
}
```

### 5-4. unmount()

```
1. Univer 인스턴스 dispose (univerAPI.dispose() 또는 univer.dispose())
2. container 내부 DOM 비우기
3. 내부 상태 초기화
```
