# aws 플러그인 런타임규격 (소비 계약)

> buildflow cycle-2 ②명세. **플러그인계약.md conform** — 충돌 시 플러그인계약 > 본 문서.
> 대상: aws 플러그인(`plugin_id = "aws"`) 구현자 + 공통셸.
> 범위: ActivitySpec 형태·백엔드 검증 프로토콜(POST /grade)·graceful 처리·PluginInstance 구현 요건.

## 1. PluginManifest 선언

```js
// 파일: plugins/aws/manifest.js
window.MANIFEST_AWS = {
  plugin_id:              "aws",
  label:                  "AWS 실습",
  version:                "1.0.0",
  infra:                  "backend",          // boto3 채점 서버 필요 (§8 분기)
  entry_url:              "http://localhost:5001",  // grade.py 실행 위치 (사용자 설정 가능)
  byok:                   null,               // 키 불요 (LocalStack은 dummy key 사용)
  capabilities:           ["cloud-lab"],
  scoring_mode:           "auto",             // 동기 POST /grade (C2 fix — §6-2 conform)
  activity_type:          "cloud-task",
  progress_schema_version: 1
};
```

**infra = backend 근거**: 채점은 boto3가 LocalStack(Docker) 또는 사용자 AWS 계정에 실제 API 호출로 리소스 상태를 확인. 브라우저-내 실행 불가 → 반드시 로컬 백엔드 프로세스(grade.py) 필요.

**AWS 콘솔 임베드 불가**: AWS Management Console은 `X-Frame-Options: SAMEORIGIN` / CSP `frame-ancestors` 헤더로 외부 iframe 임베드를 차단함(클릭재킹 방어). 플러그인은 콘솔을 iframe으로 내장하지 않음. 대신 **과제 제시 + CLI/절차 힌트 + 검증** 방식으로 동작.

## 3. 백엔드 검증 프로토콜 (POST /grade)

### 3-1. 엔드포인트 규격

플러그인 "검증" 버튼 클릭 → 프론트엔드가 `entry_url + "/grade"` 에 POST.

**요청 (Request)**:
```json
{
  "activity_id": "aws-s3-create-bucket-001",
  "checks": [
    {
      "type": "resource-exists",
      "service": "s3",
      "identifier": "my-practice-bucket-2026"
    }
  ]
}
```

**응답 (Response)**:
```json
{
  "passed": 1,
  "total": 1,
  "details": [
    {
      "check_index": 0,
      "type": "resource-exists",
      "service": "s3",
      "identifier": "my-practice-bucket-2026",
      "ok": true,
      "message": "버킷이 존재함"
    }
  ]
}
```

응답 필드:

| 필드 | 타입 | 설명 |
|---|---|---|
| `passed` | int | 통과한 check 수 |
| `total` | int | 전체 check 수 |
| `details` | array | 각 check 결과 |
| `details[].check_index` | int | 요청 checks 배열 인덱스 |
| `details[].ok` | bool | 통과 여부 |
| `details[].message` | string | 사람이 읽는 결과 설명 |

### 3-2. grade.py 백엔드 채점 로직

```python
# grade.py 채점기 참조 구현 (boto3 + LocalStack or 실제 AWS)
import boto3
from botocore.exceptions import ClientError

ENDPOINT = "http://localhost:4566"   # LocalStack 기본 포트; 실제 AWS 사용 시 None

def make_client(service: str):
    kwargs = dict(region_name="ap-northeast-2",
                  aws_access_key_id="test",
                  aws_secret_access_key="test")
    if ENDPOINT:
        kwargs["endpoint_url"] = ENDPOINT
    return boto3.client(service, **kwargs)

def check_resource_exists(service: str, identifier: str) -> tuple[bool, str]:
    """서비스별 리소스 존재 확인 패턴"""
    client = make_client(service)
    try:
        if service == "s3":
            client.head_bucket(Bucket=identifier)
            return True, f"버킷 '{identifier}' 존재함"
        elif service == "ec2":
            resp = client.describe_instances(InstanceIds=[identifier])
            exists = len(resp["Reservations"]) > 0
            return exists, f"인스턴스 '{identifier}' {'존재함' if exists else '없음'}"
        elif service == "lambda":
            client.get_function(FunctionName=identifier)
            return True, f"Lambda 함수 '{identifier}' 존재함"
        elif service == "iam":
            client.get_role(RoleName=identifier)
            return True, f"IAM 역할 '{identifier}' 존재함"
        elif service == "dynamodb":
            client.describe_table(TableName=identifier)
            return True, f"DynamoDB 테이블 '{identifier}' 존재함"
        else:
            return False, f"미지원 서비스: {service}"
    except ClientError as e:
        code = e.response["Error"]["Code"]
        return False, f"리소스 없음 (오류코드: {code})"

def check_config_equals(service: str, identifier: str, expected) -> tuple[bool, str]:
    """속성값 일치 확인 — expected는 {attribute_path: value} dict"""
    # 구현체는 서비스별 describe API로 속성 조회 후 expected와 비교
    # 예: s3 BucketVersioning → get_bucket_versioning 호출
    ...
```

## 4. ScoreResult 반환 형태

플러그인계약 §4 ScoreResult conform. grade.py 응답 → PluginInstance.score()가 변환:

```ts
interface ScoreResult {
  verdict:   "correct" | "incorrect" | "pending";
  score_raw: number;          // passed / total (0..1)
  grader_id: "external";      // 플러그인계약 §4 외부채점서버
  feedback: {
    passed:  number;
    total:   number;
    details: Array<{
      check_index: number;
      ok:          boolean;
      message:     string;
    }>;
    error?: string;           // 연결 실패 시 오류 메시지
  };
}
```

## 5. Graceful 처리 (백엔드 미연결)

셸 부트 시 `entry_url + "/health"` 3초 타임아웃 → 실패 시:

1. **mountOfflineFallback** 호출 — 활동은 표시하되 "검증" 버튼 비활성화.
2. 안내 메시지 표시:
   ```
   AWS 백엔드(grade.py)에 연결할 수 없습니다.
   LocalStack Docker 또는 실제 AWS 백엔드를 시작한 후 새로고침하세요.

   [옵션 A] LocalStack — LOCALSTACK_AUTH_TOKEN 필요 (2026.03 이후 Hobby 이상):
     LOCALSTACK_AUTH_TOKEN=<your-token> docker run --rm -p 4566:4566 \
       -e LOCALSTACK_AUTH_TOKEN localstack/localstack
     → 토큰 발급: https://app.localstack.cloud/sign-in

   [옵션 B] MiniStack — 계정·토큰 불필요 (MIT):
     docker run --rm -p 4566:4566 gresearch/ministack
     → S3·SQS·Lambda 등 기본 지원

   → grade.py: python grade.py (기본 포트 5001)
   ```
3. ScoreResult: `{ verdict: "pending", score_raw: 0, grader_id: "external", feedback: { error: "backend-unreachable" } }`.
4. 학습자는 과제 설명·checkpoints·cli_hint·back.solution을 정상 열람 가능 — **학습 콘텐츠는 오프라인에서도 접근 가능**.

## 6. PluginInstance 구현 요건

플러그인계약 §3 인터페이스 전부 구현 필수. aws-lab 전용 사항:

### 6-1. mount(container, ctx)

```
1. entry_url /health 헬스체크 (3s timeout)
   - 실패 → mountOfflineFallback(§5)
   - 성공 → 정상 UI 렌더링
2. 과제 UI 주입:
   - front.prompt (마크다운 렌더링 가능)
   - front.checkpoints → 체크리스트 (학습자가 직접 체크, 채점과 무관한 진행 트래킹)
   - front.cli_hint → "힌트 보기" 토글로 숨김 처리
   - "검증" 버튼 (entry_url 연결 시만 활성)
3. back.solution/explanation → "풀이 보기" 토글
4. 진도 복원 시 onProgressRestored() 상태 반영
```

### 6-2. score(userAnswer)

```ts
// userAnswer: { activity_id: string }  // 사용자 입력값 없음 — 리소스 상태가 정답
// 반환: Promise<ScoreResult>
async score(userAnswer): Promise<ScoreResult>
```

- entry_url 미연결 → `{ verdict: "pending", ... }` 즉시 반환.
- `POST entry_url + "/grade"` with `{ activity_id, checks: activity.grading.checks }`.
- 응답 변환 → ScoreResult (§4).
- 완료 후 셸에 `emit('activity-completed', result)` 방출.

### 6-3. getProgressSnapshot()

```ts
{
  plugin_id: "aws",
  schema_version: 1,
  activities: {
    [activity_id]: {
      cold_attempts: number,
      cold_correct:  number,
      last_verdict:  "correct" | "incorrect" | "pending" | null,
      plugin_extra: {
        last_check_details: Array<{ check_index: number; ok: boolean; message: string }> | null
      }
    }
  }
}
```

진도 저장: 셸 localStorage — 서버 저장 없음(플러그인계약 §8).

## 7. infra = backend 와이어링

플러그인계약 §8 backend 분기 conform:

| 항목 | 규격 |
|---|---|
| 헬스체크 | GET `entry_url + "/health"`, 3s timeout, HTTP 200 & `{"status":"ok"}` |
| 채점 | POST `entry_url + "/grade"` (§3-1) |
| 진도 저장 | 셸 localStorage (서버 아님) |
| 활동 데이터 | `window.ACTIVITIES["aws"]` 정적 등록 (file:// fetch 회피) |
| 헬스체크 실패 | mountOfflineFallback — 콘텐츠 열람 유지, 검증 버튼만 비활성 |

## 9. 파킹 (사용자 인프라 — 플러그인 밖)

아래 항목은 본 규격의 범위 밖. 학습자/운영자가 직접 준비:

| 항목 | 이유 |
|---|---|
| LocalStack Docker 실행 (`docker run --rm -p 4566:4566 localstack/localstack`) | 사용자 인프라 |
| 실제 AWS 계정 + credentials 설정 | 사용자 인프라, 비용 발생 |
| grade.py 서버 실행 (`python grade.py`) | 사용자 로컬 프로세스 |
| AWS 콘솔 iframe 임베드 | X-Frame-Options SAMEORIGIN/CSP frame-ancestors로 차단됨 — 기술적 불가 |
| LocalStack Pro 서비스 (EKS, ElastiCache 등) | 유료, BYO; Community 에디션만 기본 지원 |
