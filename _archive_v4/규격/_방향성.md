# 플러그인 기능 방향성 (cycle-3 SoT)

> buildflow B모드 cycle-3 ① 산출물. 5개 플러그인 기능 후보 이데이션(웹그라운딩) + 적대검토 10에이전트 → opus 종합.
> **이 문서가 기능 선별·우선순위의 상위 원칙.** 개별 `기능백로그.md`는 본 문서에 conform. 충돌 시 본 문서 > 백로그.
> 상위 계약은 여전히 `플러그인계약.md`(코어 SoT). 본 문서는 그 위의 "무엇을 만들/뺄 것인가" 판단 기준.

---

## 1. 7대 선별 원칙 (모든 기능 후보가 통과해야 함)

**원칙 1 — 콘텐츠·학습루프 완성 > 기능 확장.**
채점 인프라·서비스 커버리지 확장보다, 각 플러그인이 **"문제 제시 → 입력 → 채점 → 해설 → 재시도"의 닫힌 학습루프**를 (목업 콘텐츠로라도) 도는 것이 우선. 엔진/채점기를 먼저 만들고 ActivitySpec 데이터를 비워두는 **역순 우선순위 금지**. (대표 결함: aws — 채점 핸들러 확장은 즐비한데 실제 과제 데이터 S3 versioning 1개뿐 → 학습효과 0.)

**원칙 2 — 규격엔 있는데 코드가 안 쓰는 것부터 (낭비 제거).**
`back.hint/why/solution/explanation` 렌더, `matched`·전체 케이스 피드백, `dictation_threshold` 재정의 등 — **규격 정의됨 + 데이터 존재 + plugin.js 미구현**인 항목이 최우선 저비용 core. 새 기능 추가보다 이미 설계된 것의 완성이 먼저.

**원칙 3 — 셸 책임은 셸로 (책임 경계).**
진도 저장한계 대응(IndexedDB)·타이머/시험모드·세션 목표·키설정 UI·세션 관리는 **공통셸 레벨**(`플러그인계약.md §5·§6`). 개별 플러그인 백로그에서 분리 → 별도 "셸 백로그"로 이관. 플러그인은 {문제규격·런타임UI·채점기·진도기여}에 집중.

**원칙 4 — 인출 강도 우선 (빼기 미학의 학습판).**
인출(retrieval)을 약화시키는 편의기능은 low 또는 제외: 코드 자동완성(함수명 직접 인출 방해)·카드 앞면 통계배지(cold 첫시도 bias·`엔진규격 §1.3` warm 독립성 침범)·TTS 본문 낭독(텍스트 회피). 단서최소 원칙(`카드규격 §4.1`) 유지. "화면 난잡 + 학습 가치 불확실"은 빼기.

**원칙 5 — 차용우선 절대 (자체구현 금지).**
LCS diff 자작·PySheetGrader Flask 래퍼 자작·grade.py POST /exec(범용 shell executor)는 통합우선 위반 → 제외. 차용 도구(CDN 라이브러리/Pyodide/Univer/boto3/LocalStack)로 안 되면 **제외 또는 파킹**, 자체구현으로 우회 금지.

**원칙 6 — 계약 변경은 명시적 + 사용자 승인.**
코어 SoT(`플러그인계약.md §4` ScoreResult 등) 변경을 "내부 구현 추가"로 위장 금지. cross-folder 공개계약 변경은 buildflow triage **major → 사용자 승인 대상**(`_사용자판단대기.md` 적재). 정합 버그(scoring_mode 불일치·plugin_id 오기)는 기능보다 먼저 수정하는 **하드플로어**.

**원칙 7 — 파킹 존중.**
`_사용자판단대기.md`의 v2 파킹 항목·브라우저 실측 선행 항목(사용자 액션 필요)은 **명시적 재평가 근거 없이 core/mid로 부활 금지**. 부활시키려면 파킹 사유를 반박하는 근거를 백로그에 적고 사용자 확인.

## 2. 공통 계약 결정 (cross-plugin, 사용자 승인 필요 = 파킹)

검토에서 **여러 플러그인 공통**으로 떠오른 계약 레벨 사안 — opus가 권고안만 확정하고 실제 코어 SoT 변경은 사용자 승인 대기(`_사용자판단대기.md`에 적재):

| # | 사안 | 영향 플러그인 | 권고 | 상태 |
|---|---|---|---|---|
| C1 | `ScoreResult.feedback` 전체 케이스/셀 결과 배열 확장 (현재 `first_fail` 단건만) | coding·excel | 채택 권고 — 다건 피드백이 즉각 학습피드백의 핵심. `feedback.cases: [{idx, input?, expected, actual, pass}]` 추가 | **파킹(C1)** — 코어계약 변경 |
| C2 | aws `scoring_mode` 계약 불일치 — manifest `external` 선언 vs `score()` 동기 POST /grade(=`auto` 패턴) | aws | `scoring_mode`를 실제 구현에 맞게 `auto`로 수정하거나, 셸 분기 의미 확정 | **하드플로어** — 셸 분기 시 채점 붕괴 시한폭탄 |
| C3 | aws `plugin_id` "aws-lab" 오기 (규격 JSON Schema const) vs 코드 "aws" | aws | 규격 const를 "aws"로 정합 — ActivitySpec 기계검증(G1/V3) 실패 차단 | **하드플로어** — 선수정 |
| C4 | `back`/`plugin_extra` 스키마 `additionalProperties:false` 완화 (hints[], history[] 등) | excel·english | 필요 필드만 명시 추가(완화 X) — `back.hints`, `plugin_extra` 크기 가드 | **파킹(C4)** |
| C5 | `plugin_extra` localStorage 크기 가드 (writing history·셀 스냅샷 누적 시 5MB 초과 silent fail) | english·excel·(card-quiz) | 셸 레벨 quota 가드 + 내보내기 유도 | **셸 백로그**(원칙3) |

→ C2·C3은 기능 작업 전 **선수정**(하드플로어). C1·C4는 사용자 승인 후 반영. 그 전까지 백로그에서 해당 의존 기능은 "계약대기" 표시.

## 3. 셸 백로그로 이관 (플러그인에서 분리)

원칙3에 따라 개별 플러그인 백로그에서 빼고 **공통셸 백로그**(`규격/_셸백로그.md`, 후속 라운드 작성)로 이관:
- 진도 대용량 저장(IndexedDB 폴백) — card-quiz가 독자구현하면 `§5` 진도계약 위반
- `plugin_extra` quota 가드 + 진도 내보내기/가져오기 UI (전 플러그인 공통)
- 타이머/시험모드 — excel 단독 구현 시 타 플러그인과 불일치
- 세션 목표·분량 설정 — `session-done` 이벤트는 셸이 소비하는 게 계약 의도
- BYO-key 설정 UI 자동생성 — manifest `byok` 선언 기반(이미 계약 §6, 구현 확인 필요)

## 4. 플러그인별 우선순위 재배치 요약 (검토 fold 결과)

각 `규격/<plugin>/기능백로그.md`가 상세. 여기선 검토로 **바뀐 판정**만 요약:

**card-quiz** — back.hint/why 렌더·judge 선택지버튼·recall_seq 입력UI·진도 내보내기 = core 유지. **추가 core(검토)**: self O/X wiring·cloze 0-base 인덱스 wiring·세션내 재큐 흐름(현 구현 확인 선행). **D-day 토글 mid→core**(없으면 box2·3 카드 due 전 완전잠금). TTS 낭독 mid→low(원칙4). 앞면 통계배지 = 제외(인출순도). IndexedDB = 셸이관.

**coding** — 전체 케이스 결과표 = core지만 **C1 계약대기**. **micropip 화이트리스트 mid→core**(matplotlib·numpy 과제의 선행조건). **실행전용 버튼(채점없이 실행) 추가 core**(시도-확인 루프). 자동완성 mid→low(원칙4 인출저해). 오답 재큐 mid→low(card-quiz SRS와 중복). Judge0 다언어 = WSL2 전제 조건부(무비용 아님, 사용자환경 확인). 코드 diff·에디터 테마·난이도 자동보정 = 제외/파킹.

**excel** — back.solution/explanation 해설패널·값비교 float 허용오차·다중셀 결과표 = core. **calculationEnd 타임아웃 방어 추가 core**(무한 pending 방어). **생성규칙 예시 target_cells 오기 수정 + validate 자동화 추가 core**(오채점 직결). 진도 셀 스냅샷 core→mid(API 미검증+낮은 실마찰). 수식동치(PySheetGrader Flask래퍼) = 제외/파킹(원칙5 자체구현). XLOOKUP·ko-KR = 브라우저 실측 선행(파킹). hints mid→low(C4 대기 + 해설 core 후순). 타이머 = 셸이관.

**english** — dictation_threshold 재정의·writing LLM 엔드포인트 유연화(llm_endpoint byok 필드 추가) = core. **speaking/6모달 중복 계상 해소**(Azure speaking은 단일 티켓으로). **SM-2 due_at mount 정렬 결합 = core 전제**(분리 시 절반구현). dictation 임계는 파킹 항목이나 규격이 재정의 허용 명시 → 재평가 후 core 인정. Azure speaking 실통합 = v2 파킹 유지(stub=graceful 정상). reading 단어팝업 = 제외(vocab 커버리지 미보장). 재생횟수제한·세션목표 = 셸이관/제외.

**aws** — **과제 데이터 세트(목업 10~20개) = 최우선 core**(원칙1, 채점확장의 전제). **원클릭 기동 스크립트(start.ps1) 추가 core**(진입장벽). LocalStack 2026.03 인증변경 대응(LOCALSTACK_AUTH_TOKEN 안내 + MiniStack 대안) = core(현 docker run 명령 작동불가=버그). 멀티과제 네비 = core. config-equals 확장 core→mid(과제데이터 선행). 체크포인트 자가체크 core→mid. VPC·CFN mid→low(EC2 mock 수준·기존 resource-exists로 흡수). CLI exec 시뮬레이터 = 제외(원칙5). C2·C3 = 선수정.

## 5. 다음 단계 (구현 순서, 사용자 "목업 콘텐츠" 지시 반영)

1. **하드플로어 선수정**: C2(aws scoring_mode)·C3(plugin_id 정합).
2. **목업 콘텐츠 + 학습루프 완성**(원칙1·2): 각 플러그인에 목업 ActivitySpec 수개 + "안 쓰던 back/피드백 렌더" core를 묶어 1개 닫힌 루프 시연.
3. **C1·C4 계약변경 사용자 승인** 받은 뒤 다건 피드백 반영.
4. mid 기능은 학습효과 순 단계 진입(한 번에 다구현 금지, 빼기 미학).
5. 셸 백로그(`_셸백로그.md`)는 별도 사이클.
