# 공통셸 기능 백로그 (cycle-3)

> 방향성 §3·C5에 따라 개별 플러그인 백로그에서 분리·이관된 **공통셸 레벨** 기능. SoT=`_방향성.md`, 계약=`플러그인계약.md §5·§6`.
> 원칙: 전 플러그인 공통으로 작동해야 하는 것은 셸 1개가 소유(플러그인 중복구현 = 책임경계 위반). 플러그인은 {문제규격·런타임UI·채점기·진도기여}에만 집중.

---

## core (셸 핵심)

| 기능 | 동작 | 근거·출처 플러그인 |
|---|---|---|
| **BYO-key 설정 UI 자동생성** | 각 manifest `byok` 선언 → 셸이 settings 화면 키입력 UI 자동생성. `localStorage['clf:keys:<keyId>']` 저장(로컬만). 플러그인은 `ctx.getKey()` 조회 | 계약 §6 명시 — **구현 확인 선행**(english writing/speaking·coding/excel/aws LLM 전부 의존). english `llm_endpoint`·`llm_model` 키 추가 필요 |
| **진도 quota 가드 + 내보내기/가져오기** | `savePluginProgress` 시 5MB 초과 catch → 사용자에 내보내기 유도(현재 silent fail 위험). JSON Blob+`<a download>` 내보내기 / `<input type=file>`+FileReader 가져오기. file://에서도 동작 | C5. english writing history·excel 셀스냅샷·card-quiz 누적이 5MB 위협 [MDN localStorage 5MB]. plugin.js try-catch 삼킴 |

## mid (셸 중간)

| 기능 | 동작 | 근거 |
|---|---|---|
| **세션 완료 요약 (session-done 소비)** | 플러그인 `session-done` 이벤트를 셸이 받아 통합 세션요약(푼 수·정답률·오답목록+개념서/해설 링크) 렌더. 플러그인별 빈상태 화면 대체 | card-quiz·english가 각자 원했으나 = 셸 책임(계약 §5 이벤트 의도). 중복구현 방지 |
| **세션 목표·분량 설정** | 학습 시작 시 목표 문항수(5/10/20)·시간 설정 + 진행바. 목표 달성 메시지. 전 플러그인 공통 | english 이데이션 항목 → 셸이관. 빼기미학: 단순 카운터만(AI 스케줄링 v2) |
| **타이머/시험모드 (공통 옵션)** | settings에 시험모드 토글 → 문제당 제한시간·소요시간 기록(`plugin_extra` 또는 셸 진도). 기본 off | excel·(coding) 이데이션 → 셸이관(플러그인별 구현 시 동작 불일치). 컴활 실기 시간압박 대비 |

## low / 파킹 (셸)

| 기능 | 사유 |
|---|---|
| **IndexedDB 폴백 (대용량 진도)** | localStorage 5MB 한계 우회. 단 quota 가드+내보내기(core)로 1차 대응 후 필요시. `loadPluginProgress/savePluginProgress` 계약 내부 교체(플러그인 투명) | card-quiz가 독자구현하면 §5 진도계약 위반 → 셸 소유 [MDN IndexedDB 동적용량] |
| **다기기 동기화** | 현재 단일기기 localStorage. 클라우드=서비스화·비용 → 범위 밖 | 기획 §8 백로그 |
| **진도 기반 적응형 난이도 (공통)** | 전 플러그인 공통 ni즈(coding 난이도보정·excel 등). solve_ms 등 데이터 축적 선행 | v2, 데이터 전제 |

## 의존·주의
- BYO-key UI 자동생성은 **여러 플러그인 core의 공통 선행조건** — 우선 구현 확인 필요(이미 계약 §6에 설계됨, 실제 셸 구현 여부 미확인). 특히 **english `writing LLM 유연화` core의 effective 선행조건**(셸 UI 없으면 `ctx.getKey('llm_endpoint')` 항상 null → graceful pending만, 키 입력 경로 자체가 없음). 붕괴는 아니나 기능 무력화.
- quota 가드는 writing history·셀 스냅샷 등 **누적형 plugin_extra 기능의 안전 전제** — 해당 기능들보다 먼저.
- 세션관리(요약·목표·타이머)는 전부 `session-done`/settings를 통한 **셸-플러그인 단방향 계약** 위에서 — 플러그인 내부 구현 금지.
