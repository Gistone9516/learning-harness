# 적응형 깊이 스무고개 (생성 전 인테이크 spec)

> buildflow cycle-2 ②명세. **Claude가 세션에서 따르는 절차 spec**(앱 폼 아님 — 적응형·과목맞춤이라 정적 폼 불가).
> 목적: 학습물 생성 *전*, 과목 맞춤 스무고개로 학습자 프로파일 확정 → 플러그인 생성규칙에 주입.
> 원칙(메모리 [[learning-framework-design]]): 도메인 미리 안 박음(편향 회피), 사용자가 자기 수준/범위를 *생짜로는 모름* 전제.

---

## 핵심 원칙
1. **도메인 가정 금지(편향 차단).** 모델이 임베딩으로 "이 과목은 이럴 것" 단정하지 말 것 → **먼저 웹서치**로 그 과목의 실제 구조 파악(CLAUDE.md evidence-grounding 적용).
2. **진단형 > 자기보고형.** "수준이 어떻게 되세요?" ❌(사용자가 모름). → "다음 중 아는 것?/이 문제 풀 수 있나요?" 식 **진단 질문으로 모델이 수준을 추정**.
3. **적응형·유한.** 앞 답 보고 다음 질문 조정. 프로파일 채울 만큼 샤프해지면 즉시 정지(무한 질문 금지). 최대 ~5문항 목표.
4. **사용자 탈출 게이트.** "그냥 알아서" → 합리적 가정으로 채우고 *가정 명시* 후 진행.

## 절차
**STEP 0 — 과목 구조 조사 (웹서치, 필수)**
사용자가 과목 X를 말하면, 질문 전에 웹서치로 확보:
- X의 표준 학습 단계·레벨 체계(입문→숙련 경로)
- 하위 영역/토픽 분류
- 평가·자격·실무 기준(있으면)
- 흔한 학습 함정·선수지식
→ 임베딩 단정 금지. 출처 있는 구조 위에서 질문 설계.

**STEP 1 — 적응형 진단 스무고개 (과목 맞춤)**
STEP 0 구조 기반으로 *그 과목에 맞는* 질문 생성. 캐는 차원(고정 폼 아님, 과목 따라 가감):
- **현재 위치**(진단형): X의 핵심 개념 몇 개를 제시 → 아는지/풀 수 있는지로 수준 추정
- **목표**: 시험/자격 / 실무·프로젝트 / 교양·이해 / 특정 스킬 (X에 맞춰 구체 선택지)
- **범위**: 전체 커리큘럼 / 특정 토픽만 (STEP0 토픽 분류 제시)
- **깊이 비중**: 이론 ↔ 실습 어디에 무게
- **분량·기간**: 한 번에 볼 양 / 마감
- **선호 출력 환경**: 가용 플러그인 중(card-quiz/coding/excel/english/aws) X에 맞는 것

**STEP 2 — 프로파일 산출 → 생성규칙에 주입**
종료 시 `LearningProfile` 확정:
```
LearningProfile = {
  subject: string,
  subject_structure: {레벨체계, 토픽분류, 평가기준, 함정} (STEP0 웹조사, 출처포함),
  estimated_level: string (진단 기반 추정 + 근거),
  goal: enum,
  scope: string[] (대상 토픽),
  depth_ratio: {theory: 0..1, practice: 0..1},
  volume: string,
  target_plugins: plugin_id[]   ← 어느 실습환경으로 생성할지
}
```
→ 이 프로파일을 각 `target_plugins`의 **생성규칙**(플러그인별)에 입력으로 넘김 → 학습물 생성.

## 비용 주의
STEP0 웹조사·STEP1 질문은 *가볍게*(스무고개는 사용자 상호작용이라 opus 추론 적음). 무거운 건 STEP2 이후 *생성*인데, 그건 플러그인 생성규칙이 빡빡한 템플릿이라 추론 최소(생성규칙 spec 소관).

## 미정([OPEN], 추후 구체화 — 사용자 표시)
- 진단 질문의 구체 형태(객관식 vs 단답 vs 미니문제)는 과목·플러그인 따라 다름 → 플러그인별 생성규칙에서 진단 문항 템플릿 제공할지 검토.
- 재학습 시 기존 LearningProfile 재사용/갱신 흐름.
