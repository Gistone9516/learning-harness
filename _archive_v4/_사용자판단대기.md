## cycle-8 신규 실습 플러그인 (cad-print · robot-arm) — 파킹 5항목 (2026-06-09)

### P1. cad-print 슬라이싱/G-code 실습 (Kiri:Moto self-host)
- **맥락**: JSCAD 파라메트릭 모델링 이후 단계. 슬라이싱·G-code 생성을 브라우저 내에서 교육하려면 Kiri:Moto 정적 self-host 필요.
- **선행 조건**: `file://` 환경에서 WASM 로드 시 CORS 정책 충돌 여부 — 브라우저 실측 PoC 필요. 현재 MVP는 WASM 의존 금지 원칙(파일 CORS).
- **선택지**: ① Kiri:Moto CDN iframe 임베드(CSP·X-Frame 검토 필요) ② 로컬 서버(localhost) 전제로 infra=backend 전환 ③ 파킹 유지(슬라이싱 제외, 모델링만)
- **권고**: ③ 파킹 유지 — 비전공자 입문 MVP에 슬라이싱 불필요. JSCAD 모델링 루프 안정화 후 재평가.
- **회피 임시값**: cad-print MVP = JSCAD 파라메트릭 모델링만.

### P2. robot 경로계획/IK 실습
- **맥락**: FK 슬라이더 MVP 이후 확장. IK·경로계획은 비전공자에게 개념 체화 효과 있으나 직관적 인터페이스 필요.
- **선행 조건**: FK MVP 완성 후 학습효과 확인. 비전공자용 "코딩 없는" 인터페이스(블록 드래그·노드 연결) 설계 방향 결정 필요.
- **선택지**: ① 슬라이더 확장으로 IK 구현(수치 수렴, 직관성 낮음) ② 블록/노드 드래그 인터페이스(별도 UI 설계) ③ 파킹 유지
- **권고**: ③ 파킹 유지. FK 루프 완성 → 사용자 피드백 → IK 설계 방향 결정.
- **회피 임시값**: robot-arm MVP = FK 슬라이더만.

### P3. UAIbotJS · MuJoCo WASM · cannon-es 도입
- **맥락**: 로봇 물리 시뮬레이션 강화 목적으로 검토됐으나 모두 MVP 제외.
- **UAIbotJS**: 미성숙(API 불안정, 문서 부족). **MuJoCo WASM**: 브라우저 실험적, Windows `file://` CORS 충돌 미검증. **cannon-es**: FK 2D 평면 암에 물리엔진 불필요(과잉).
- **선택지**: 각 라이브러리 성숙도 재평가 후 필요시 도입. 현재는 three.js + 순수 JS 삼각함수로 충분.
- **권고**: 파킹 유지. 도입 시 브라우저 실측 PoC 선행 필수.
- **회피 임시값**: three.js 0.160.0 + 순수 JS FK 계산.

### P4. capabilities 신규 토큰 셸 부트 분기 영향
- **맥락**: cad-print · robot-arm 모두 `capabilities=['practice']`(범용 단일 토큰)로 선언. 신규 'cad'·'robotics' 토큰을 추가하면 셸(shell.js)이 capabilities 기반 하드 분기를 하는지 확인 필요.
- **선행 조건**: `shell.js` 구현 확인 — `capabilities` 값으로 분기하는 로직이 있으면 신규 토큰 추가 시 셸 변경 필요(계약 major).
- **선택지**: ① 'practice' 단일 토큰 유지(현재 권고) ② 신규 토큰 + 셸 무분기 확인 후 추가
- **권고**: ① 유지. 셸이 capabilities로 하드 분기하면 안 된다는 게 플러그인계약 §8 의도. 확인 후 문제없으면 나중에 추가 가능.
- **회피 임시값**: 'practice' 토큰 사용. 빌더가 shell.js 확인 후 결과 보고 필요.

### P5. 브라우저 시각검증 (JSCAD 렌더 · three.js 암 렌더 UI 실측)
- **맥락**: JSCAD regl-renderer 또는 three.js 기반 3D 미리보기, three.js FK 암 렌더 — 헤드리스 환경(Node.js)에서 WebGL 불가. 채점 로직은 Node 검증 가능하나 렌더·UI 상호작용은 브라우저 실측 필수.
- **선행 조건**: 사용자가 index.html(또는 subject HTML) 더블클릭 후 육안 확인.
- **검증 포인트**: ① JSCAD 코드 실행 후 3D 프리뷰 표시 ② robot-arm 슬라이더 조작 → 암 실시간 렌더 ③ 끝점 좌표 표시 ④ 목표점 마커 표시 ⑤ 채점 후 진단 피드백 렌더
- **권고**: MVP 구현 완료 후 사용자 실측 체크리스트 제공. 헤드리스 자동검증 범위 = 채점 로직만.
- **회피 임시값**: 채점/FK 계산은 Node 단위테스트로 커버, UI는 브라우저 확인 위임.
