## 2. 작업 1 — DB 구축 (25점)

### 2-1. 세부 유형 전수 (N = 8개 유형)

| # | 세부 유형 | 주요 요구 기능 | 정답 판정 산출물 |
|---|---|---|---|
| 1 | 테이블 필드 속성 설정 | 데이터 형식(텍스트·숫자·날짜/시간·예/아니오·OLE 등), 필드 크기, 형식, 캡션, 기본값, 필수, 빈 문자열 허용, IME 모드 | 디자인 보기 각 필드의 속성값 |
| 2 | 입력 마스크 | 0(필수 숫자)·9(선택 숫자)·L(필수 문자)·>(대문자 변환) 등 조합, 저장 문자 포함 여부 | 필드 속성 → 입력 마스크 값 |
| 3 | 유효성 검사 규칙 및 텍스트 | 비교·Between·Like·In 조건식, 오류 메시지 | 필드 속성 → 유효성 검사 규칙/텍스트 |
| 4 | 기본 키 및 인덱스 | 단일·복합 기본키 지정, 인덱스(아니오·중복 가능·중복 불가능) | 테이블 디자인 기본키 표시, 인덱스 창 |
| 5 | 조회 속성 (콤보·목록) | 행 원본 유형, 행 원본(SQL/값 목록), 열 수·너비, 목록 행 수, 목록 이외 값 허용 여부 | 필드 속성 → 조회 탭 속성값 |
| 6 | 외부 데이터 가져오기 | Excel·텍스트 파일 가져오기(구분기호·고정 너비), 첫 행 필드명 여부, 기존 테이블 추가 vs 새 테이블 | Access 내 테이블 레코드 및 구조 |
| 7 | 관계 설정 | 일대다 관계, 참조 무결성 적용, 연속 업데이트·삭제 옵션 | 관계 창(데이터베이스 도구) |
| 8 | 외래 키·테이블 연결/내보내기 | 연결 테이블 생성, 테이블 내보내기(Excel·텍스트) | 탐색 창 테이블 목록, 외부 파일 |

### 2-2. 배점 및 부분점수 규칙

- 테이블 필드 속성: 문항당 **3~5점**, 속성 항목별 부분점수 인정
- 기본키·관계: 각 **5점** 내외, 참조 무결성 옵션 미설정 시 부분 감점
- 외부 데이터: **5점** 내외, 레코드 수·필드명 일치 여부로 정오 판정

### 2-3. 자동채점 관점 정답 판정 산출물

- `테이블 디자인 보기` 각 필드의 속성 시트 값(문자열 비교)
- `관계 창` 관계선 유무, 참조 무결성 체크박스 상태
- 테이블 레코드 수 및 데이터 내용(외부 데이터 가져오기)

### 2-4. 자주 감점 포인트

- 입력 마스크 저장 문자(세미콜론 구분) 누락
- 유효성 텍스트 없이 규칙만 입력
- 조회 속성 열 너비 형식 오류(`0;3` → cm 단위 주의)
- 관계 창 연결 후 참조 무결성 체크 누락
- 기본값에 따옴표 처리 오류(텍스트형 기본값 = `"서울"`)

- 열 너비 `;`로 구분 시 숨길 열은 `0` 입력

### 4-2. 배점 및 부분점수 규칙

- 컨트롤 속성 문항: **3점/문항** × 5 = 15점
- 이벤트/하위 보고서 등 복합 문항: **5점** 내외
- 요약 함수 수식 문자열 일치로 채점

### 4-3. 자동채점 관점 정답 판정 산출물

- 보고서 `정렬 및 그룹화 창` 설정값

### 4-4. 자주 감점 포인트

## 5. 작업 4 — 처리 기능 구현 (35점)

### 5-1. 세부 유형 전수 (N = 14개 유형)

#### 5-1-A. 쿼리 유형 (출제 2~3문항)

| 2 | 선택 쿼리 (함수 활용) | IIf(), Format(), Left/Right/Mid(), DateDiff(), Year/Month(), Int() 등 계산 필드 | 계산 필드 수식 및 결과값 |
| 3 | 요약 쿼리 (Group By) | 묶는 방법(Group By), 집계(합계·평균·개수·최대·최소), 조건(Where·Having) | 쿼리 실행 결과 |
| 4 | 크로스탭 쿼리 | 행 머리글·열 머리글·값 지정, 크로스탭 마법사 또는 디자인 | 크로스탭 쿼리 결과셋 |
| 5 | 매개 변수 쿼리 | 대괄호 `[]` 매개변수 지정, 단일·다중 매개변수, 날짜 형식 | 매개변수 입력 시 필터 결과 |
| 6 | 불일치 검색 쿼리 | 두 테이블 비교, IS NULL 조건 또는 NOT IN 서브쿼리 | 불일치 레코드 결과셋 |
| 7 | 중복 데이터 검색 쿼리 | 동일값 그룹, COUNT > 1 조건 | 중복 레코드 결과셋 |
| 8 | 조인 속성 쿼리 | 내부 조인·왼쪽 외부 조인·오른쪽 외부 조인 선택 | 조인 유형별 결과셋 |
| 9 | 업데이트(갱신) 쿼리 | UPDATE 실행, 조건부 필드값 변경, IIf() 활용 | 실행 후 테이블 레코드 변경 결과 |
| 10 | 삭제 쿼리 | DELETE 실행, 조건에 맞는 레코드 제거 | 실행 후 테이블 레코드 수 |
| 11 | 추가 쿼리 | INSERT INTO, 특정 테이블에 조건 레코드 추가 | 대상 테이블 레코드 수 |
| 12 | 테이블 만들기 쿼리 | SELECT INTO로 새 테이블 생성 | 새 테이블 존재 및 레코드 |

#### 5-1-B. 이벤트 프로시저 / VBA (출제 1문항)

| # | 세부 유형 | 주요 요구 기능 | 정답 판정 산출물 |

| 14 | Filter / RecordSource / MsgBox | `Me.Filter = "..."` + `Me.FilterOn = True`, `Me.RecordSource = "SQL"`, `MsgBox` 조건 분기 | 코드 내 속성 할당 문자열 |

### 5-2. 배점 및 부분점수 규칙

- 쿼리 2문항: 각 **7~14점**, 조건식·필드 구성 단계별 부분점수
- 이벤트 프로시저 1문항: **7~14점**, 문법 오류 없이 실행 + 결과 정확 시 만점
- 실행 쿼리(업데이트·삭제·추가·테이블 만들기): **반드시 단 1회 실행** — 중복 실행 시 데이터 오염으로 0점 처리 위험

### 5-3. 자동채점 관점 정답 판정 산출물

- 선택/요약/크로스탭 쿼리: **실행 결과 레코드셋** (레코드 수, 필드명, 데이터값)
- 실행 쿼리: 실행 후 **대상 테이블 레코드 상태**
- VBA 프로시저: **코드 텍스트 문자열 비교** + 실제 실행 결과(폼·보고서 열기, 필터 적용 상태)

### 5-4. 자주 감점 포인트

- 크로스탭 쿼리: 열 머리글·행 머리글 혼동, 값 집계 함수 오류
- IIf() 중첩 시 True/False 값 순서 혼동
- DoCmd.OpenForm WhereCondition 문자열: 날짜 `#` 처리, 텍스트 `'` 처리 누락
- 실행 쿼리 저장 후 미실행(실행 전 저장만 하면 채점 미반영)
- VBA에서 컨트롤 참조 시 `Forms!폼명!컨트롤명` vs `Me.컨트롤명` 혼용 오류
- RecordsetClone + FindFirst 사용 시 `.NoMatch` 처리 누락
