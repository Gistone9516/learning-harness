# 학습 프레임워크 기획 v4 (B모드 수렴본)

> buildflow ① 기획 산출물. B모드 5라운드·적대렌즈 15회 → R4·R5 2연속 DRY 수렴.
> 상태: G1 통과(B모드 자율). 다음 = ② 명세(규격/ 계약 작성) → G2(적대 무결성 + 사용자 빨간펜).

---

## 1. 의도
컴활1급 **합격**용 학습 앱. 필기3 + 실기2 전 범위를 **개념서(이해) + 카드퀴즈(인출·회귀)**로 커버.
지금은 컴활1급 맞춤 UI/UX, 구조는 **타 과목 확장 가능**(config-driven).

## 2. 스코프 경계 (의식적 트레이드오프 — 구멍 아님)
- 실기 **자동채점 안 함**(웹시뮬·Office자동채점 타당성검토 후 폐기). 실제 손풀이 = 사용자 직접 Office.
- 실기 작업유형의 **절차·함수는 학습 콘텐츠**(절차/순서재현/판단형 카드)로 다룸.
- **procedural 갭**(빈 화면서 클릭경로 재현)은 앱이 측정 안 함 → 실기 체크리스트 실습반복이 전담.
- **합격확률 예측 안 함**(코호트 데이터 없음, 착각확신 방지). 대시보드는 proxy(인출성취율) 라벨 명시.
- **기출본문 미수집**(저작권/스코프) → 기출 CBT 불가, 자작문항 모의고사만(백로그).

## 3. 학습 모델
- **개념서** = 전체 레퍼런스(함수: 구문/인수/반환/예제/실수포인트/유사함수대조 · 절차: 단계/메뉴경로/단축키/주의).
- **카드** = 단서최소 인출(앞=최소단서, 뒤=상세). 개념서와 **양방향 링크**(섹션↔연결카드 바로풀기).
- **카드 5유형**: 함수 / 절차 / 순서재현 / cloze / 판단형(시나리오: 조건→적절 절차·함수 선택).
- **채점 4모드**: exact(구문, 정규화·복수정답) / keyword(절차 핵심키워드 포함) / cloze(빈칸) / self(자가 O·X).
- **오답 처리**: 세션내 "다 맞출때까지 재큐"(단기교정) + 세션간 **Leitner 간격회귀**(1/3/7일).
  - 박스 승급 = **세션간 cold 첫시도 정답만** 기준(세션내 재큐 warm 정답은 박스이동 무관).
  - **D-day 임박** 시 간격압축·전박스 강제소환("막판 총정리").
- **출제 우선순위**: Leitner due-first → 그 안에서 **빈출·배점 weight** 정렬. 신규카드는 weight 우선.

## 4. 진단·동기 (대시보드)
- **인출성취율**(proxy 라벨) + **약점진단**(단원 오답률) + **완주 진도바**(영역별 box 분포 = 정복률 box3+).
- **합격경로 진단**: 영역별 목표(필기 각40/평균60 · 실기70) 대비 커버리지×숙달도 → **미달위험 신호**.
- **학습플랜 라이트**: D-day + 영역별 잔여량 → 일일 권장량·필기/실기 비중 가이드(강제 아님).
- 진단3종 직교(단기성취 / 합격거리 / 완주상태). 기본진입 = **카드퀴즈 즉시시작**(대시보드 부차, 기능과다 회피).

## 5. 아키텍처
- **런타임 = 정적 HTML + localStorage**(서버 0, 더블클릭 실행).
- 카드 데이터 = generate.py가 **과목별 deck(네임스페이스) + manifest.js 레지스트리** 출력 → file:// fetch 회피(전역 script), 무로드 과목전환.
- **config-driven 엔진**: `엔진/` = 범용 채점·파서·Leitner 코어 / 과목특화(정규화 옵션·동의어·채점모드 매핑) = 과목폴더 config 주입 → 엔진 무오염, 2번째 과목 무재협상.
- **Python generate.py** = 콘텐츠 md → deck + manifest 빌드 + 오답/진도 export·백업.
- card_id = **작성자 명시 안정 id**(content-hash 아님 → 오탈자 수정에도 오답이력 보존). localStorage 스키마 = **버전필드 + 마이그레이션 shim**.
- 무의존(stdlib) 유지 — 단 채점기·정규화·cloze파서·Leitner는 **테스트셋 동반 필수**(④ 검증 게이트).

## 6. 파티션맵
```
학습 프레임워크 제작/
├ 기획_v4_수렴.md        ← 이 문서(지속상태)
├ 규격/                   ← 프레임워크 본체(②에서 작성)
│   ├ 카드규격.md         카드 스키마·5유형·4채점모드·정규화 규칙·card_id
│   ├ 개념서규격.md       함수/절차 레퍼런스 템플릿·카드 연결
│   ├ UI규격.md          진입동선·대시보드·디자인토큰(빼기·차분·유채색·둥근)·정보밀도 균형
│   ├ 생성규칙.md         md→deck+manifest 변환·config 스키마
│   └ 엔진규격.md         Leitner·출제큐·채점코어·localStorage 스키마·마이그레이션
├ 엔진/  (범용)           generate.py · app/[정적HTML·JS]
└ 과목/컴활1급/
    ├ 자료조사/  (9md 완료)
    ├ 콘텐츠/   (필기_컴퓨터일반·필기_스프레드시트·필기_데이터베이스·실기_엑셀·실기_액세스)
    ├ config/   (컴활 정규화·동의어·채점모드 매핑)
    └ 생성물/   (deck·manifest·빌드 HTML)
```

## 7. 얇은 슬라이스 (첫 증분)
필기 1단원 → 개념서뷰 + 카드(함수 exact + cloze + 판단형 1장) → Leitner·재큐·오답회귀·대시보드 최소판 → 정적 HTML 더블클릭 동작. 엔진 전 루프 + UI/UX 톤 동시 증명. 콘텐츠 대량생산은 그 후.

## 8. 백로그 (자율범위 밖 / 후순위)
- 자작문항 모의고사 타이머 모드(기출본문 미수집 → 자작만).
- 다기기 동기화(현재 단일기기 localStorage).
- FSRS 등 고급 SRS 차용(현재 Leitner 자체구현, 무의존).
- localStorage 5MB 쿼터 한계 → 대규모 deck 시 export 백업 권장(known tradeoff).

## 9. B모드 자율 fold 로그 (FYI — 사용자 빨간펜 대상)
원안 대비 적대평가로 바뀐 의사결정. **특히 ★는 사용자 명시기획/내 디폴트를 바꾼 것**:
- ★ 오답 = 원안 "다맞출때까지 재큐" **유지** + 그 위에 세션간 Leitner 간격회귀 **추가**(massed practice 결함 보완).
- ★ 영속화 = 내 디폴트였던 `python server.py`(http.server) **폐기** → 정적HTML+localStorage(비개발자 진입장벽 0). Python은 생성기로 잔류.
- 빈출·배점 가중 출제 / 과목별 합격경로 진단 / 완주 진도바 / 실기 체크리스트 / 판단형(시나리오) 카드 / D-day 모드 / 개념서↔카드 양방향링크 / config-driven 확장구조 / 카드 단서최소 분리(읽기자료화 방지) — 전부 적대평가 반영 추가.
