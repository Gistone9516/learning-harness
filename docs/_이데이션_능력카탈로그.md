# 이데이션 능력 카탈로그 (v5 ② 규격 입력)

> 출처: 멀티에이전트 이데이션 Workflow(7에이전트, 5렌즈 + 2비평, harness 58파일 전수).
> 무손실 원본 = `_이데이션_원본.json`. 본 문서 = opus가 source 대비 판정·dedup한 **4계층 합성**.
> 형식: 능력 — 학습가치 — `프리미티브` — [tier]. core=일상 핵심, extension=확장.
> ★ 경계: 학습 알고리즘(인터리빙·큐·Leitner)은 **엔진 코어**(파이썬), harness는 Discord I/O·진도 저장.

---

## 계층 1 — 엔진 코어 능력 (파이썬 포트, Discord 무관)

> v4 JS 코어(`_archive_v4/엔진/app/app.js`)에서 포팅. 전부 결정적·순수함수. 진도 영속만 `store.py`.

- **채점 4모드** — exact(복수정답·정규화) / keyword(필수키워드 전부) / cloze(빈칸 0-base) / self(자가 패스스루) + 순서(sequence 완전일치). 전부 이진 정/오, 부분점수 없음. — `score` — [core]
- **정규화** — 규칙 9종(nfkc/trim/collapse_space/strip_all_space/lower/fullwidth_to_halfwidth/unify_cell_dollar/unify_arg_sep/strip_trailing_paren) 순서적용 + lower 직후 synonyms 토큰치환. — `normalize` — [core]
- **Leitner 전이** — cold 첫시도 정답만 승급(box1→2→3), 오답 강등, warm/skip 무관, 졸업(BOX_MAX+cold정답). — `leitnerTransition`/`nextDueAt`/`isDue` — [core]
- **출제큐 buildQueue** — due-first → weight DESC → tie-break. **인터리빙**(단원 라운드로빈), D-day 전박스 소환, limit. — `buildQueue` — [core]
- **대시보드 집계** — 인출성취율(by_area) / 약점(단원 오답률) / 합격경로(커버리지×숙달, safe/watch/danger) / 완주분포(box3+). — `getDashboardData` — [core]
- **진도 영속 + 마이그레이션** — CardProgress/ProgressStore JSON 원자적 저장(localStorage 대체), schema_version shim, `.bak` 손상폴백. — `automation/store.py` + 포팅 `loadProgress`/`saveProgress` — [core]

## 계층 3 — AI 학습 능력 (`_invoke` stream-json 어댑터 복제)

> 전부 토큰 통제 동반: 짧은 preamble · `--effort low`/저렴모델 · 조건 미충족 시 미호출(토큰 0).

- **오픈엔드 채점**(서술·번역·단답, 결정적 불가 영역) → JSON verdict 강제, 실패 시 self 폴백 — `form`+`typing_indicator`+`cards`+`store` — [core]
- **소크라테스 대화**(설명→AI 후속질문, elaboration) — 슬라이딩 윈도우 4턴 — `threads`+`webhook`+`paginator`+`store` — [core]
- **힌트 생성**(레벨별 동적 단서) — 레벨 상한 3 — `buttons`+`typing_indicator`+`store` — [core]
- **항목/예문 생성**(시드 리스트→카드 초안 배치) — `driver`(1시드=1호출)+`progressbar`+`store` — [core]
- **개인화 피드백**(오답 이력 기반 교정) — 최근 3오답만 컨텍스트 — `cards`+`webhook`+`store` — [core]
- **오개념 탐지**(단원 단위 반복오답 패턴) — 대표 5카드 슬라이싱 — `scheduler`+`store`+`cards` — [extension]
- **적응형 난이도**(retrieval_rate→weight 재산출) — 규칙기반=토큰0, AI=전략제안만 — `scheduler`+`store`+`metricpush` — [core]
- **AI 세션 요약**(학습 일지, 메타인지) — 세션당 1호출 — `digest`+`store`+`cards` — [extension]
- **AI 스트리밍 렌더**(토큰 단위 실시간) — `live/livecard.py`+`typing_indicator` — [core]
- **변형질문 생성**(box3 카드, transfer) — box3에만→초기비용0 — `driver`+`store` — [extension]
- **페르소나 멀티스피커**(출제자/채점관/코치) — `channels/webhook.py`(토큰 무관 URL) — [extension]
- **프로액티브 리마인더**(due 기반 AI 독려) — due=0이면 미호출 — `scheduler`+`mention`+`cards` — [extension]
- **배치 AI 생성 드라이버**(크래시 재개) — `automation/driver.py` + `_invoke` — [core]

## 비평 반영 (이데이션 2비평)

**추가할 갭(렌즈가 놓친 능력):** 이벤트 기반 세션 자동진입 · 입력 게이팅 컴포지션 · heartbeat 생존알림 ·
ratelimit 코알레싱 공유 · logtail AI 스트림 · suppress 안전 카드 · presence 세션신호 · 권한 사전검증 ·
채널 카테고리 스캐폴드 · reaction 초저마찰 채점 · reply.forward 오답 아카이브 · pins 로테이션 ·
정교화질문(비AI) · preview-then-test · 지연피드백 게이트 · concept-ref 링크 · pre-testing · (인터리빙은 엔진).

**병합할 중복 6쌍:** LiveCard↔MetricDashboard(코드동일→push alias) · Due알림 3종(→채널 파라미터) ·
자유회상↔서술(→입력경로 2종) · 분산복습알림↔복습세션 · 자가갱신SRS↔복습세션(filedrop 제거) ·
시험모드↔시간차 자기퀴즈(→Delayed-Feedback Exam).

**결정적 교정(fact-cited):** "인터리빙 큐"를 `store.py`에 오귀속한 false-positive — `store.py`는 KV JSON뿐,
셔플 알고리즘 없음. **인터리빙·큐 로직은 파이썬 엔진 buildQueue 소관**(계층 1). harness는 I/O·저장만.

**tier 교정:** `persistviews`·`ratelimit`·`logtail`은 학습 코어가 아니라 **운영품질(infra)**. 계층 4로 분리.

**범위 외(갭 아님):** voice·soundboard·stickers·roles·monetization·invites·moderation·automod·emoji ·
audit(8~9파일) — voice/게이미피케이션/커뮤니티 비주류.

**완결성 판정(비평):** 핵심 5원리(인출연습·간격반복·즉각피드백·진행가시성·자기채점) harness 1:1 매핑 충족.
2차 학습과학(정교화·출처귀인·지연피드백·pre-testing)은 ② 규격에서 named capability로 보강.
