# 학습 프레임워크 기획 — 웹 프레임 (web/)

> buildflow ① 기획 산출물. 기획_v5_discord.md §4·§9에서 후순위로 예약했던 `web/`를 착수한다.
> 정체: Discord 컴포넌트로 못 덮는 학습을 다루는 frontend 작업공간. 정본 키트와 과목무관 주입, harness의 파츠+표준 규율을 그대로 계승한다.
> 상태: 핵심 결정 확정(첫 타겟·스택·호스팅·AI 경로·모델·진도). 다음은 ② 웹 명세.

---

## 1. 의도

Discord 네이티브 봇이 못 덮는 영역을 웹으로 보완한다. 연속 인터랙티브 실습과 장문 정독이 그 영역이고, 이번엔 두 모듈을 먼저 만든다.

- **인터랙티브 시트**: 수식 엔진을 갖춘 스프레드시트형 실습.
- **프로젝트 코드 독해**: 직접 A부터 Z까지 작성하는 시대가 지난 만큼, AI가 생성한 프로젝트를 **탑다운으로 읽고 이해**하는 학습. 전체 구조를 파악하고 "이걸 바꾸려면 어느 코드를 건드리나"를 익힌다.

장문 정독은 v4 선례가 있어 뒤로 미룬다. 봇과 별개 앱이 아니라 같은 정본 키트의 한 부분이며, 과목은 데이터와 config로만 주입한다.

## 2. 확정 결정

- **첫 타겟 2종**: ① 인터랙티브 시트(넓은 수식 엔진) ② 프로젝트 코드 독해(세션 적재·클릭 실시간 해설). 장문 정독은 후속.
- **스택**: Vite + vanilla TS(경량 빌드). 풀 프레임워크 미사용. 코드 뷰어는 CodeMirror를 파츠로 흡수(읽기 위주).
- **호스팅**: 로컬 우선(내 PC), 외부에서 볼 때만 터널(cloudflared). 공개 정적 호스팅은 백로그.
- **AI 경로(실측 완료)**: 웹은 claude를 직접 못 부르므로 `bot/ai_server.py`(게이티드 로컬 HTTP)를 경유해 `claude -p`(구독)를 돌린다. 별도 API 키·추가 과금 없음. 인터넷 왕복 검증됨.
- **모델**: `claude-opus-4-8[1m]`(opus 4.8, 1M 컨텍스트), effort `medium`. 1M 컨텍스트라 코드베이스를 세션에 통째 적재해도 여유.
- **진도**: v1은 브라우저 `localStorage` 단독(봇/engine과 디커플링). 하이브리드 공유는 백로그.
- **frontend-design 스킬**: 시각 디자인 단계에서 로드(지금은 아키텍처·파츠·표준 기획).

## 3. 설계 원칙 (불변 경계 계승)

봇 쪽 SoT(`specs/_interface-contract.md §7`)를 웹에도 그대로 적용한다.

- **과목 무관(§7.1)**: 웹 코드에 과목 리터럴 0. 영역·페르소나·문구·콘텐츠는 과목 config로 주입.
- **정본 키트(§7.11)**: 웹 코드도 정본 1벌. 과목별 fork 없음. 과목은 데이터·config만 다름.
- **비밀값과 게이팅(§7.12)**: 비밀값은 웹 번들에 절대 미포함. AI는 ai_server 경유, 토큰 게이팅. 개인용.
- **결정적은 토큰 0(§8)**: 시트의 수식 평가와 코드 모듈의 구조 탐색·정답 포인터 비교는 토큰 0. AI(코드 해설, 서술 채점)는 옵트인이며 ai_server 경유.

## 4. 웹 파츠 + 표준 (harness 규율의 웹 이식)

봇 harness가 "자기완결 capability 파일 카탈로그 + 공통 hard constraints"이듯, 웹도 동일 규율을 둔다.

**파츠 = 자기완결 웹 컴포넌트 카탈로그**
- 각 파츠는 독립 모듈(자기 TS와 CSS, 최소 의존). opt-in 등록: 앱 셸이 필요한 파츠만 켠다.
- 파츠 헤더 표준(harness의 WHAT/DEPS/PERMS/USAGE/SAFETY 대응): `WHAT`, `DEPS`(라이브러리·다른 파츠), `INPUT`(주입 데이터 형태), `EVENTS`(결과·진도 이벤트), `AI`(ai_server 사용 여부와 호출 형태), `CONSTRAINTS`, `DEMO`(오프라인 단독 데모).
- 인덱스: `web/parts/README.md` + 기계가독 `web/parts/index.json`(봇 harness index와 같은 모델).

**표준(모든 웹 파츠 공통 hard constraints)**
- 과목 리터럴 0(전부 주입). 비밀값 0(키는 클라이언트에 없음, AI는 ai_server 경유 토큰 게이팅).
- 결정적 결과는 봇과 같은 이진 채점으로 환원(부분점수 없음). 진도는 표준 이벤트로 store에 위임.
- 반응형(폰·PC)과 기본 접근성. 무거운 의존은 파츠 단위 격리.
- 파일명 충돌 회피, 자기완결 유지(harness 노트 계승).

## 5. 아키텍처

- **앱 셸**: Vite + vanilla TS. 활성 파츠 목록을 과목 config로 구성(capability_registry의 웹 대응).
- **AI 브리지(`bot/ai_server.py`, 구현·실측 완료)**: 봇과 별개 프로세스인 게이티드 로컬 HTTP 서버. `POST /ai`가 `ai.invoke`로 `claude -p`(구독)를 돌려 응답. `AI_SERVER_TOKEN` 게이팅, localhost 바인드 기본, 외부는 터널로만. **실측: 공개 터널 왕복 6초·PONG, 잘못된 토큰 401, Discord 비의존.** `session_id`를 받아 `--resume` 멀티턴 지원(검증됨).
- **세션 적재 코드 독해(핵심 메커니즘)**:
  1. 생성: 목표 규모로 AI가 프로젝트를 통째 생성(세션 turn 1). 이때 코드베이스가 그 claude 세션 문맥에 적재된다. 웹은 반환된 `session_id`를 보관.
  2. 표시: 좌측에 파일/폴더 트리, 파일 클릭 시 우측 코드 패널에 코드를 보여준다(읽기 위주).
  3. 클릭 실시간 해설(지연 생성): 사전 전체 생성은 토큰 과부하라 하지 않는다. 클릭한 파일·메서드·라인에 대해서만 `web → ai_server(session_id 동봉) → claude --resume → 해당 부분 해설`을 실시간 생성한다. 코드베이스가 이미 세션 문맥에 있으니 클릭 1번이 작은 호출이 된다. 해설 종류: 라이브러리·메서드 설명, 코드 흐름, 수정 팁.
  - 1M 컨텍스트에서 소~중 규모 프로젝트가 여유롭게 들어가며, 난이도(=프로젝트 규모)의 천장도 자연히 컨텍스트로 정해진다.
- **데이터 주입**: 시트 문제·정답 스펙은 과목 config 데이터로 주입(결정적). 코드 프로젝트는 데이터가 아니라 AI 온디맨드 생성(시드·규모·주제만 config/입력으로).
- **진도 store**: v1은 `localStorage`(이진 결과·이벤트). engine·`_state` 공유는 백로그.
- **딥링크**: Discord 카드의 링크 버튼이 웹의 특정 상태(모듈·과제)를 연다. 호스팅 위치에 따라 로컬 또는 터널 URL.

## 6. 파티션맵 (web/)

```
web/
├ index.html              앱 진입
├ src/
│   ├ shell/              앱 셸: 라우팅, 활성 파츠 조립, 진도 store
│   ├ inject/             과목 config·데이터 로더(주입 인터페이스 재사용)
│   ├ ai/                 ai_server 클라이언트(토큰·session_id 관리, /ai 호출)
│   └ store/              localStorage 진도(이진 결과·이벤트)
├ parts/                  자기완결 컴포넌트 카탈로그(+ README.md, index.json)
│   ├ sheet/              인터랙티브 시트(수식 엔진)
│   └ codeproj/           프로젝트 코드 독해(트리 + 코드 패널 + 세션 클릭 해설)
└ (백로그) reading/ 3d/ fk/
```

## 7. 첫 슬라이스 (얇게)

파츠+표준 골격과 ai_server 연동을 먼저 증명한다.

- **앱 셸 + 파츠 표준/인덱스 + ai_server 클라이언트**(토큰·session_id) 최소 구현.
- **시트 파츠 1종**: 주입된 문제로 셀·수식 입력 → 수식 엔진 평가 → 기대값 이진 채점 → store. 토큰 0(결정적).
- **코드 독해 파츠 1종**: 시드/규모 입력 → AI 프로젝트 생성(세션 적재) → 좌측 트리·우측 코드 패널 → 파일·메서드 클릭 시 `--resume` 실시간 해설. opus 4.8 1M, medium.
- 채점형 과제("X를 바꾸려면 어느 파일/함수"를 사용자가 지목 → 결정적 포인터 비교)는 탐색 UI 위 fast-follow.
- 목업/소규모 시드로 동작 증명(과목무관 가드 유지). frontend-design은 이 시점에 로드해 시각 다듬기.

## 8. 백로그 / 후순위

- 코드 모듈 채점형 과제(수정 지점 식별·흐름 추적 점수화), 더 큰 규모 프로젝트.
- 장문 개념 정독(v4 선례 포팅), 3D 모델링·로봇팔 FK.
- engine·`_state` 공유 하이브리드(ai_server 경유 진도 동기화), 공개 정적 호스팅 + 게이팅.
- 시트 함수 커버리지 확대, (필요 시) 코드 실행 샌드박스.

## 9. 다음 단계 + 잔여 red-pen

**확정 정리**
- 첫 타겟 = 인터랙티브 시트(넓은 수식 엔진) + 프로젝트 코드 독해(세션 적재·클릭 해설).
- AI 경로 = `bot/ai_server.py` + 터널(실측). 모델 = opus 4.8 1M, effort medium.
- 진도 = localStorage 단독(v1). 스택 = Vite + vanilla TS.

**② 웹 명세에서 확정할 것**
- 파츠 헤더 표준 스키마, `index.json` 형식.
- ai_server 클라이언트 계약: `/ai` 요청·응답, 토큰 주입 경로(번들 비포함), `session_id` 수명·재개.
- 코드 독해 계약: 프로젝트 생성 프롬프트·반환 형태(파일 트리 + 파일 내용), 클릭 해설 요청 형태, 세션 종료 시점.
- 시트 계약: 지원 함수 집합(v1 범위), 셀·범위 참조 모델, 정답 스펙·이진 채점.
- 진도 store 이벤트 스키마(localStorage).

**남은 소수 결정(red-pen)**
- 시트 v1 함수 커버리지(어디까지: 산술·집계·논리·참조 중 우선순위).
- 코드 프로젝트 "규모" 파라미터화(파일 수·줄 수 기준 난이도 단계).
