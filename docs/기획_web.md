# 학습 프레임워크 기획 — 웹 프레임 (web/)

> buildflow ① 기획 산출물(초안, G1 검토 대기). 기획_v5_discord.md §4·§9에서 후순위로 예약했던 `web/`를 착수한다.
> 정체: Discord 컴포넌트로 못 덮는 학습을 다루는 frontend 작업공간. 정본 키트와 과목무관 주입, harness의 파츠+표준 규율을 그대로 계승한다.
> 상태: 핵심 결정 확정(스택·호스팅·AI 경로·첫 타겟). 다음은 사용자 red-pen 후 ② 웹 명세.

---

## 1. 의도

Discord 네이티브 봇이 못 덮는 영역을 웹으로 보완한다. 연속 인터랙티브 실습과 장문 정독이 그 영역이고, 이번에 **인터랙티브 시트와 라이브 코드 에디터**를 먼저 만든다(장문 정독은 v4 선례가 있어 뒤로). 봇과 별개 앱이 아니라 같은 정본 키트의 한 부분으로, 과목은 데이터와 config로만 주입한다.

## 2. 확정 결정 (이번 라운드)

- **첫 타겟**: 인터랙티브 시트 1종, 라이브 코드 에디터 1종. 장문 정독은 후속(선례 있음).
- **스택**: Vite + vanilla TS(경량 빌드). 풀 프레임워크 미사용. 코드 에디터는 CodeMirror 같은 라이브러리를 파츠로 흡수.
- **호스팅**: 로컬 우선(내 PC), 외부에서 볼 때만 터널(Cloudflare Tunnel 또는 ngrok). 공개 정적 호스팅은 백로그.
- **AI 경로**: 웹은 claude를 직접 못 부르므로 로컬 봇을 경유한다. 웹에서 로컬 봇으로 요청하고 봇이 `claude -p`(구독)를 돌려 돌려준다. 별도 API 키와 추가 과금 없음. 봇의 `_invoke` 모델과 일관.
- **frontend-design 스킬**: 시각 디자인 단계에서 로드(지금은 아키텍처·파츠·표준 기획만).

## 3. 설계 원칙 (불변 경계 계승)

봇 쪽 SoT(`specs/_interface-contract.md §7`)를 웹에도 그대로 적용한다.

- **과목 무관(§7.1)**: 웹 코드에 과목 리터럴 0. 영역·페르소나·문구·콘텐츠는 과목 폴더 config로 주입.
- **정본 키트(§7.11)**: 웹 코드도 정본 1벌. 과목별 fork 없음. 과목은 데이터와 config만 다름.
- **비밀값과 게이팅(§7.12)**: 토큰 등 비밀값은 웹 번들에 절대 포함하지 않는다(클라이언트 노출 금지). 개인용, 접근 게이팅.
- **결정적은 토큰 0(§8)**: 시트 조작 채점과 코드 결정적 채점(기대 출력 비교)은 토큰 0. AI는 옵트인이며 로컬 봇 경유.

## 4. 웹 파츠 + 표준 (harness 규율의 웹 이식)

봇 harness가 "자기완결 capability 파일 카탈로그 + 공통 hard constraints"이듯, 웹도 동일 규율을 둔다.

**파츠 = 자기완결 웹 컴포넌트 카탈로그**
- 각 파츠는 독립 모듈(자기 TS와 CSS, 최소 의존). opt-in 복사: 앱 셸이 필요한 파츠만 등록.
- 파츠 헤더 표준(harness의 WHAT/DEPS/PERMS/USAGE/SAFETY 대응): `WHAT`(무엇), `DEPS`(의존 라이브러리·다른 파츠), `INPUT`(주입 데이터 형태), `EVENTS`(내보내는 결과·진도 이벤트), `CONSTRAINTS`(지켜야 할 표준), `DEMO`(오프라인 단독 데모).
- 인덱스: `web/parts/README.md` + 기계가독 `web/parts/index.json`(봇 harness index와 같은 모델).

**표준(모든 웹 파츠 공통 hard constraints)**
- 과목 리터럴 0(전부 주입). 비밀값 0(키는 클라이언트에 없음, AI는 로컬 봇 경유).
- 개인용 접근 게이팅(공개 호스팅 시 진도 데이터는 클라이언트 보관 또는 게이팅).
- 결과는 봇과 같은 이진 채점 모델로 환원(부분점수 없음). 진도는 표준 이벤트로 store에 위임.
- 반응형(폰·PC)과 기본 접근성. 무거운 의존은 파츠 단위 격리(3D 등).
- 파일명 충돌 회피, 자기완결 유지(harness 노트 계승).

## 5. 아키텍처

- **앱 셸**: Vite + vanilla TS. 라우팅은 과목 config가 정의한 활성 파츠 목록으로 구성(capability_registry의 웹 대응).
- **데이터 주입**: 과목 폴더의 manifest·config를 웹이 로드(봇과 같은 주입 인터페이스 재사용). 시트 문제, 코드 과제, 정답 스펙은 데이터로 온다.
- **진도 store**: v1은 클라이언트 `localStorage`(봇 `_state`와 분리). 추후 로컬 봇 경유로 engine·`_state` 공유하는 하이브리드를 백로그에 둔다.
- **AI 브리지(`bot/ai_server.py`, 구현·실측 완료)**: 봇과 **별개 프로세스**인 게이티드 로컬 HTTP 서버. `POST /ai`가 검증된 `ai.invoke`로 `claude -p`(구독)를 돌려 응답한다. 토큰 게이팅(`AI_SERVER_TOKEN`, 미설정 시 기동 거부)과 localhost 바인드 기본, 외부는 터널로만. **실측: cloudflared 공개 URL 왕복 6초·PONG, 잘못된 토큰 401, Discord 비의존.** 추가 의존성 0(stdlib).
- **딥링크**: Discord 카드의 링크 버튼이 웹의 특정 상태(과목·과제)를 연다. 호스팅 위치에 따라 로컬 또는 터널 URL.

## 6. 파티션맵 (web/)

```
web/
├ index.html              앱 진입
├ src/
│   ├ shell/              앱 셸: 라우팅, 활성 파츠 조립, 진도 store
│   ├ inject/             과목 config·데이터 로더(주입 인터페이스 재사용)
│   └ store/              localStorage 진도(이진 결과·이벤트)
├ parts/                  자기완결 컴포넌트 카탈로그(+ README.md, index.json)
│   ├ sheet/              인터랙티브 시트 파츠
│   └ codeedit/           라이브 코드 에디터 파츠(CodeMirror)
├ parts/README.md         파츠 인덱스 + 표준(hard constraints)
└ (백로그) reading/ 3d/ fk/
```
- 봇 쪽 신규: `bot/`에 로컬 AI HTTP 엔드포인트(웹 전용, allowlist·로컬 바인드).

## 7. 첫 슬라이스 (얇게)

파츠+표준 골격을 먼저 증명한다.
- **파츠 표준·인덱스·앱 셸** 최소 구현 + **시트 파츠 1종**(주입된 문제로 셀 입력→결정적 채점→이진 결과→store) + **코드 에디터 파츠 1종**(과제 코드 입력→실행 또는 기대출력 비교→이진 결과).
- 둘 다 **토큰 0 결정적**으로 먼저. AI(설명·서술 채점)는 로컬 봇 경유로 그다음.
- 목업 과목 데이터로 동작 증명(과목무관 가드 유지). frontend-design은 이 시점에 로드해 시각 다듬기.

## 8. 백로그 / 후순위

- 장문 개념 정독(v4 선례 포팅), 3D 모델링·로봇팔 FK.
- engine·`_state` 공유 하이브리드(로컬 봇 경유 진도 동기화).
- 공개 정적 호스팅(GitHub Pages 등) + 진도 게이팅.
- 코드 실행 샌드박스 고도화(보안 격리), 시트 함수 커버리지 확대.
