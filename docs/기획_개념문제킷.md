# 학습 프레임워크 기획 — 개념·문제 공부 킷 (web 모듈)

> buildflow ① 기획 산출물. 웹 프레임(`기획_web.md`)의 세 번째 모듈. 미뤘던 "장문 개념 정독"을 되살려 문제풀이와 잇는다.
> 정체: 개념을 먼저 학습(정독)하고 그 개념에 연결된 문제를 푸는 교과서형 흐름의 웹 모듈.
> 상태: 핵심 결정 확정(웹 모듈·개념→문제 흐름·개념 AI 생성·문제 엔진 결정적 채점). 다음은 ② 명세.

---

## 1. 의도

봇의 카드 인출은 "문제"는 잘 다루지만 "개념을 차분히 읽고 이해"하는 정독은 Discord 컴포넌트로 약하다. 이 모듈은 **개념 학습 → 연결된 문제 풀이**를 한 흐름으로 묶는다. 개념은 AI가 생성하고, 문제 채점은 결정적(토큰0)이다.

## 2. 확정 결정

- **위치/형태**: 웹 모듈(`web/parts/conceptprob/`). 웹 프레임의 한 파츠.
- **흐름**: 개념 → 문제(교과서형 연결). 개념을 학습한 뒤 그 개념에 연결된 문제를 푼다.
- **구조(3단)**: 영역(area) / 개념(concept) / 문제(problem). 영역과 개념은 인덱스로 **좌측 토글 패널**(영역 → 개념 아코디언), 본문은 **상단 개념 정독, 하단 문제 블럭**. 영역 taxonomy는 과목 config로 주입(과목무관, `bot/subject.py` areas와 동형).
- **개념 출처**: AI 온디맨드 생성. 주제·범위를 주면 ai_server가 장문 개념 설명을 생성한다(코드 독해 모듈과 같은 세션 적재 방식).
- **문제 채점**: 엔진 결정적 재사용(exact/keyword/cloze/judge). 토큰 0, 안정. AI가 문제를 생성하되 채점은 엔진이 한다.
- **진도**: localStorage 단독(v1). 봇 Leitner/SRS 연동은 백로그.

## 3. 웹 프레임에서 상속 (중복 안 함)

`기획_web.md`의 결정·표준을 그대로 따른다.

- 파츠+표준 규율(자기완결·헤더 표준·index), 불변 경계(과목무관·정본·비밀값·게이팅).
- AI 경로 = `bot/ai_server.py`(게이티드 로컬 HTTP + 터널, 실측 완료), `session_id` `--resume`.
- 모델 = `claude-opus-4-8[1m]`, effort `medium`.
- 스택 = Vite + vanilla TS. 진도 = localStorage 이진 결과.

## 4. 학습 흐름

좌측 토글 패널에서 **영역 → 개념**을 고르면 본문이 로드된다(상단 개념 정독, 하단 문제 블럭).

1. **개념 생성·적재**: 주제·범위(시드)를 받아 AI가 개념 장문과 그 개념에 연결된 문제 세트를 한 번에 생성한다(세션 turn 1, 개념·문제가 세션 문맥에 적재). 웹은 `session_id`를 보관.
2. **개념 정독**: 생성된 개념 장문을 읽는다. 읽는 중 모르는 부분은 클릭·질문 → `--resume`로 그 부분 심화 해설을 실시간 생성(사전 전체생성 회피, 코드 모듈과 동일 패턴).
3. **연결 문제 풀이**: 그 개념에 묶인 문제를 푼다. 각 문제는 생성 시 `answer_spec`(accepted/keywords/blanks 등)을 함께 받았으므로, 학습자 답을 엔진 `score()`로 **결정적·이진 채점**(토큰0). 틀리면 개념의 해당 지점으로 되짚는 링크 제공.
4. **진도**: 개념별 문제 정답률을 localStorage에 이진 결과로 기록. 다음 개념으로.

## 5. 아키텍처 / 데이터

- **생성물 형태**: ai_server가 `{concept: 장문텍스트, problems: [CardDef-호환(answer_spec 포함)]}`를 반환(force_json). CardDef 스키마를 웹이 그대로 소비(주입 인터페이스 재사용).
- **결정적 채점 위치(설계 결정 필요)**: 웹 프레임 원칙은 "결정적 채점 = 웹사이드 토큰0". 따라서 엔진 `score()`/`normalize()`(순수)를 **TS로 포팅**하되, 엔진의 골든 테스트 벡터로 **Python 엔진과 패리티 가드**(드리프트 방지). 대안은 봇에 결정적 grade 엔드포인트(토큰0)를 두는 것. 기본은 TS 포팅(웹 단독성), ②에서 확정.
- **세션 적재**: 개념·문제가 한 세션에 들어가므로, 정독 중 심화 해설과 (선택) 문제 힌트가 같은 세션 `--resume`로 싸게 나온다.
- **데이터 주입**: 주제·범위·난이도는 과목 config/입력으로(과목무관). 개념·문제 자체는 데이터가 아니라 AI 생성.
- **진도 store**: 개념 id별 문제 결과(이진)·정독 완료 플래그. localStorage.

## 6. 파츠 (web/parts/conceptprob/)

```
web/parts/conceptprob/
├ conceptprob.ts          파츠 진입 + 레이아웃(좌측 인덱스 / 본문 상단 개념·하단 문제)
├ indexpanel.ts           좌측 토글 패널: 영역 → 개념 아코디언 인덱스(선택 시 본문 로드)
├ reader.ts               (본문 상단) 개념 장문 정독 뷰(클릭·질문 → 세션 심화 해설)
├ problems.ts             (본문 하단) 연결 문제 블럭 + 엔진 결정적 채점 호출
├ grade.ts                score()/normalize() TS 포트(엔진 패리티, 골든 벡터 테스트)
└ conceptprob.css
```
- 공유: `src/ai/`(ai_server 클라이언트·session), `src/store/`(localStorage 진도) 재사용.

## 7. 첫 슬라이스 (얇게)

- ai_server로 **개념 1개 + 연결 문제 N개 생성**(answer_spec 포함) → 세션 적재.
- **정독 뷰**: 개념 장문 표시 + 클릭 심화 해설(`--resume`).
- **문제 뷰**: 문제 제시 → 학습자 답 → `grade.ts`(엔진 포트) 결정적 채점 → 이진 결과·피드백 → store.
- 소규모 시드로 동작 증명(과목무관 가드). 정독↔문제 되짚기 링크 최소 1개.

## 8. 백로그 / 후순위

- 개념별 숙달·SRS 연동(봇 engine `_state` 공유 하이브리드), 개념 그래프(선후행).
- 문제 유형 확장(AI 서술 채점 옵션 추가), 개념 난이도·분량 파라미터화.
- 큐레이션 개념 주입 옵션(AI 생성 외 큐레이션 혼용).
